# 阶段5C：管理员审核写入与不可变审计

## 基线与范围

- 基线：`main@32bdab33464f2851dd223502da873dc04100e0f7`。
- 分支：`agent/stage5c-admin-review-mutations`。
- 阶段5A提供管理员登录、15分钟同源HTTPS会话与登录限流。
- 阶段5B提供鉴权后的合成审核队列与详情只读投影。
- 阶段5C只增加三项管理员动作：批准、拒绝、编辑普通单价后批准。
- 可信设备、封禁、回滚和导出仍不在本阶段范围内。
- 普通用户页面、正式用户价格库和冻结的码单器8.2.25不修改。

## 默认关闭与硬锁作用域

仓库默认值必须保持：

```text
CLOUD_ADMIN_REVIEW_PREVIEW_ENABLED=0
CLOUD_ADMIN_REVIEW_MUTATION_PREVIEW_ENABLED=0
CLOUD_ADMIN_REVIEW_BLOB_STORE_NAME=cloud-collab-preview-v1
CLOUD_ADMIN_REVIEW_ALLOWED_GROUP_ID=group_fixture
CLOUD_ADMIN_REVIEW_ALLOWED_LIBRARY_ID=lib_receive_fixture
CLOUD_WRITE_PREVIEW_ENABLED=0
CLOUD_AUTO_APPROVAL_PREVIEW_ENABLED=0
```

管理员审核写入必须同时满足：

1. 阶段5A管理员登录预览已开启，且会话有效；
2. 阶段5B只读审核开关已开启；
3. 阶段5C审核写入开关已开启；
4. 公共候选写入和自动批准预览开关均为`0`；
5. Blob、groupId和libraryId与上述合成值完全一致；
6. 请求为同源HTTPS `POST`，携带有效同源Cookie、`Origin`和JSON内容类型。

任一条件不满足时，在初始化审核Blob Store之前失败。

请求体严格白名单：

```json

```

```json

```

```json

```

- 页面每次动作都要求浏览器二次确认。
- 拒绝原因只能从固定枚举选择，不接收自由文本备注。
- 编辑后批准只允许更改普通单价；服务名称、结算方式和businessKey保持不变。
- 单价继续使用阶段4A协议：大于0、不超过1,000,000、最多3位小数。
- 页面不读取、显示或存储管理员密码、会话密钥、限流盐值或EdgeOne预览密钥。
- 页面不使用LocalStorage、SessionStorage或IndexedDB。

## 不可变状态机

每次动作按以下顺序执行：

1. 强一致重读待审核Marker、全部设备确认、不可变候选和公共基线；
2. 服务端重算候选协议Hash并确认reviewId对应真实Blob链路；
3. 对批准类动作先占用该businessKey与基线版本的唯一批准周期；
4. 写入只增不改的决策声明，固定动作、目标、证据、管理员不可逆标签和服务器时间；
5. 批准类动作复用阶段4D的公共事件、迁移索引、批准索引与快照发布链；
6. 写入只增不改的审计记录；
7. 写入只增不改的审核归档记录；
8. 最后写入只增不改的完成记录。

待审核Marker、设备确认和原始候选均不覆盖、不删除。队列通过强一致读取归档记录排除已经完成的项目。

## 三种动作

### 批准

- 目标候选必须仍基于当前公共基线。
- 创建`admin_approved`公共事件。
- 同一审核周期的其他冲突候选归档为`superseded`。
- 公共版本增加一次。

### 拒绝

- 只归档当前候选为`rejected`。
- 同周期其他候选仍可继续审核。
- 不创建公共事件，不改变公共版本。

### 编辑后批准

- 只从原候选派生新的普通单价payload和contentHash。
- 原候选保持不可变。
- 创建`admin_edit_and_approved`公共事件。
- 同周期其他候选归档为`superseded`。
- 公共版本增加一次。

## 并发、幂等与恢复

- reviewId对应一个不可变决策Key；不同动作并发时只有第一个声明可以成立。
- 批准类动作还有基线周期级唯一声明；同一businessKey和基线不可能批准两个候选值。
- 相同请求重试会复用同一decisionId、approvalId和审计记录。
- 若公共事件已经发布、但审计或归档写入中断，同一请求可继续补齐；不会再次增加公共版本。
- 若公共基线在操作前已经变化，批准和编辑后批准在写决策声明前返回冲突。
- 任一不可变对象内容与重算结果不一致时失败关闭，不返回部分可信成功。

## 隐私与权限投影

管理员HTTP响应可返回：

- 不透明reviewId、decisionId、auditId和approvalId；
- 动作与完成状态；
- 目标contentHash；
- 公共版本、事件版本、是否实际发布、归档数量与幂等重放标记。

管理员HTTP响应禁止返回：

- 完整deviceId、submissionId或idempotencyKey；
- Blob Key、审核目录、提交目录或审计路径；
- 管理员密码、Cookie、会话签名、密钥和盐值；
- 历史、订单、聊天、备注、自定义比例、布局或使用记录。

阶段5C响应能力中只有审核写入为开启；设备管理、回滚和导出继续为`false`。`publicMutationAllowed=true`只存在于阶段5C管理员隔离接口响应，不会进入普通用户客户端。

## 验收策略

阶段5B不再单独建立EdgeOne验收。阶段5C代码和CI完成后，从当时主线建立一次性、临时、`[DO NOT MERGE]`的5B/5C合并验收分支，至少验证：

1. 登录、会话、刷新恢复和退出；
2. 5B队列与详情强一致读取；
3. 5C批准、拒绝和编辑后批准；
4. 公共版本、快照、增量与跨设备读取一致；
5. 中断后的同请求恢复与不重复版本；
6. 所有合成Blob最终清理至0并二次强一致复查；
7. 关闭5B与5C预览开关，删除临时凭据、盐值、设备数据、部署和EdgeOne项目。

该临时验收分支、部署和凭据不得合并或复用。阶段5C主PR在合并验收完成前不得合并。
