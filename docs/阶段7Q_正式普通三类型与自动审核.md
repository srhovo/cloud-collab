# 阶段7Q：正式普通三类型与自动审核

## 目标

在不部署EdgeOne、不访问真实Blob、不生成真实密钥和不晋升稳定版的前提下，把已经通过阶段5G验证的普通共享能力接入正式生产运行时：

- 精确价格 `exact_price`；
- 已确认陪玩名字 `playable_name`；
- 老板普通资料 `boss_profile`；
- 两台不同设备一致确认；
- 可信设备普通批准能力继续由既有引擎保留；
- 冲突及不满足自动批准规则的候选进入人工审核标记。

本阶段不接入敏感类型。区间、加价、礼物、删除及老板敏感变化仍必须使用阶段6人工审核链。

## 路由

继续使用既有公共路径：

```text
POST /api/submissions/create
```

`CLOUD_PRODUCTION_ENABLED`仍决定进入预览或正式处理器，客户端不需要修改URL。

## 正式普通处理器

`src/server/production_ordinary_runtime_v1.js`复用：

- `normalizeOrdinarySubmission`；
- `acceptOrdinarySubmission`；
- 设备Bearer令牌鉴权；
- 正式作用域`group_see / lib_see_cz`；
- 不可变候选与幂等索引；
- 生产限流；
- `reviewOrdinaryCandidate`；
- 普通公共事件和公共快照引擎。

自动审核关闭时：

```text
候选：写入不可变队列
公共版本：不变化
公共事件：不生成
```

自动审核开启时：

```text
第一台设备一致候选 → waiting_confirmation
第二台不同设备相同内容 → auto_approved
冲突内容或不满足规则 → pending_review
```

## 安全边界

- 只接受协议作用域`group_see / lib_see_cz`；
- Authorization设备必须与正文deviceId一致；
- 新候选每台设备每5秒一个生产限流槽；
- 精确幂等重放不消耗新限流槽；
- 候选和公共事件继续使用onlyIfNew不可变写入；
- 只允许精确配置的正式HTTPS来源；
- 不允许通配CORS；
- 不返回密钥、盐值或内部错误详情；
- `stablePromotionAuthorized=false`；
- 默认环境模板中全部生产开关保持0。

## 复用而非重写

阶段7Q没有建立新的普通共享协议。所有业务Key、内容Hash、幂等键、老板身份、审核证据、公共事件和快照格式均复用阶段5G已验证实现。因此以后创建真实生产Blob时，可以继续使用同一迁移、维护、回滚与完整性检查链。
