# 阶段4E：隔离预览路由接入

## 阶段目标

在阶段4D自动审核基础之上，新增一套不会覆盖现有正式接口的隔离预览路由，用于真实设备验证：

```text
/api/preview/submissions/create
/api/preview/public-version
/api/preview/public-snapshot
/api/preview/public-changes
```

现有路由保持不变：

```text
/api/submissions/create
/api/public-version
/api/public-snapshot
/api/public-changes
```

因此本阶段不会改变8.2.28现有用户页面、正式只读目录或正式提交行为。

## 双重门禁

必须同时满足：

```text
CLOUD_WRITE_PREVIEW_ENABLED=1
CLOUD_AUTO_APPROVAL_PREVIEW_ENABLED=1
```

任一开关不为1，自动审核预览均返回503，并且不会初始化Blob Store。

所有预览读写还必须携带：

```text
X-Cloud-Collab-Preview-Key
```

密钥沿用隔离预览访问密钥，长度32至256字节，只保存在EdgeOne环境变量与临时验收页面内存中，不写入仓库、HTML持久内容、LocalStorage或聊天。

## 作用域

代码硬锁：

```text
group_fixture
lib_receive_fixture
```

环境变量改成其他团或价格库会拒绝启动；请求其他作用域返回403。

## 提交流程

预览提交路由执行：

1. 校验双开关；
2. 校验预览访问密钥；
3. 校验请求体大小与冻结提交协议；
4. 校验设备令牌和测试作用域；
5. 不可变保存候选；
6. 强一致读回候选；
7. 执行阶段4D自动审核；
8. 返回等待、自动批准或待审核结果。

返回中允许出现：

```text
previewMutationApplied
previewPublicVersion
previewEventVersion
previewAutoApprovalEnabled
```

但正式能力标志始终保持：

```text
writeEnabled=false
publicMutationAllowed=false
autoApprovalEnabled=false
```

这样8.2.28客户端不会把隔离预览能力误认为正式公共写入能力。

## 动态读取

动态版本、快照和增量都只读取批准索引明确采用的公共事件。

增量接口只投影以下公开字段：

```text
version
approvedAt
businessKey
contentHash
dataType
operation
payload
```

不会向客户端暴露批准ID、设备ID、提交ID、迁移索引或内部审核证据。

## 本阶段验证范围

自动化测试覆盖：

- 双开关和访问密钥在Blob初始化前阻断；
- 新价格首设备等待；
- 两个不同设备相同内容自动批准；
- 幂等重放不新增事件；
- 已有价格恰好±10%双设备批准；
- 超过±10%进入审核；
- 同一公共基线不同候选进入审核；
- 动态公共版本、快照和增量一致；
- 增量响应不泄露内部批准证据；
- 现有正式路由未被替换。

## 真实预览验收顺序

1. 本PR完整CI通过后再建立临时验收页面分支；
2. EdgeOne预览环境同时开启两个预览开关；
3. 使用全新独立预览访问密钥和限流盐；
4. 两台真实设备分别提交同一新增价格；
5. 验证第一台等待、第二台批准、公共版本增加；
6. 两台设备提交±10%以内同一改价并验证再次批准；
7. 提交超过±10%或不同候选，验证只进入审核且公共版本不增加；
8. 验证动态快照和增量被另一台设备接收；
9. 清理所有合成对象并确认剩余0；
10. 将两个开关恢复为0，删除临时密钥、限流盐和预览部署；
11. 临时验收页面PR关闭且禁止合并。

## 禁止事项

- 不得把预览路由替换成正式路由；
- 不得允许非测试作用域；
- 不得复用正式公共数据命名空间；
- 不得把预览密钥写进仓库；
- 不得开放管理员审核、可信设备管理或人工批准页面；
- 不得在未完成真实清理前合并任何临时验收工具。
