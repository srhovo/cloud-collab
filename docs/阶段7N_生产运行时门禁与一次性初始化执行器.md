# 阶段7N：生产运行时门禁与一次性初始化执行器

## 目标

在没有自定义域名、真实生产Blob和真实密钥的情况下，完成可由代码和CI闭环的生产运行时与空库初始化能力。阶段7N不部署EdgeOne、不生成真实密钥、不触碰真实Blob，也不晋升稳定版。

## 生产运行时门禁

`src/server/production_runtime_config_v1.js`统一校验：

- 所有开关必须显式为`0`或`1`；
- 总开关关闭时，任何子能力都不能单独开启；
- 普通提交必须在只读同步之后；
- 普通自动审核必须在只读同步和普通提交之后；
- 管理员审核必须同时具备只读同步和管理员身份能力；
- 敏感提交必须在管理员人工审核已经就绪后；
- 一次性初始化只能在所有生产能力关闭时运行；
- 初始化必须使用精确确认词`INITIALIZE-see-see_cz-V1`；
- 用户可见`see / see_cz`必须映射为协议`group_see / lib_see_cz`；
- 公共与管理员Blob必须使用两个固定且不同的生产命名空间；
- 生产来源必须为不含路径、查询或凭据的纯HTTPS origin；
- 已配置的密钥必须为32至256字节且彼此不同；
- 返回状态投影永远不包含真实密钥值；
- 稳定晋升授权保持false。

## 一次性初始化执行器

`src/server/production_bootstrap_v1.js`生成并执行10个冻结资源：

### 公共生产Store

1. `group_see`元数据；
2. `lib_see_cz`元数据；
3. 公共版本0指针；
4. 空公共快照；
5. 普通提交队列标记；
6. 敏感审核队列标记；
7. 每日快照维护标记；
8. 迁移导出维护标记。

### 管理员生产Store

9. 管理员安全初始化记录；
10. 不可变审计创世记录。

执行顺序：

1. 对全部目标Key做强一致预检；
2. 任一既有对象内容冲突时，在新增写入前失败；
3. 对缺失对象执行`onlyIfNew`不可变写入；
4. 并发占用时只接受内容完全相同的对象；
5. 对全部对象再次强一致复核；
6. 返回资源清单、SHA-256及实际`get/setJSON/delete`调用计数。

精确重放不产生新写入。执行器没有物理删除路径，返回的删除计数必须为0。

## CI边界

`.github/workflows/stage7n-production-runtime.yml`只运行：

- 默认关闭配置审计；
- 冻结初始化计划生成；
- 内存Store首次初始化；
- 精确重放；
- 冲突失败关闭；
- 密钥、来源和分阶段依赖校验。

CI报告明确：

```text
真实Blob读取：0
真实Blob写入：0
真实Blob删除：0
真实密钥生成：false
生产能力启用：false
稳定晋升：false
```

## 仍需外部条件

代码准备完成后，真实执行仍需要：

- EdgeOne控制台创建两个Blob Store；
- 本地生成并录入8项随机密钥；
- 一个可长期匿名访问的自定义域名；
- 在正式环境按只读到敏感人工审核的顺序完成L4；
- 项目负责人单独授权8.3.0晋升。
