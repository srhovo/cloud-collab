# 阶段6A：敏感规则协议范围冻结

## 基线

- 仓库：`srhovo/cloud-collab`

- 本阶段不创建 EdgeOne 项目、部署、凭据、设备或 Blob 对象

## 唯一主目标

冻结并实现“敏感公共变更”的统一提交协议、服务器白名单校验、业务键/内容 Hash 规则和人工审核分类底座。

本阶段所有有效敏感候选都只能进入 `pending_review`。不同设备重复提交、可信设备、首次绑定或任何自动审核条件，都不得把敏感候选自动批准。

## 本阶段覆盖的数据类型与操作

### 1. 区间规则 `rank_range_rule / upsert`

只允许公共业务投影：

- `rangeLabel`
- `alias`
- `rankType`
- `minStar`
- `maxStar`
- `namedRanks`
- `prices.normal.round/hour`
- `prices.carry.round/hour`
- `prices.starGuarantee.round/hour`

必须重新构造白名单对象；本地规则 ID、使用次数、排序、时间、来源和界面状态不得进入提交。

### 2. 加价规则 `surcharge_rule / upsert`

只允许公共业务投影：

- `name`
- `keywords`
- `prices.round/hour`
- `enabled`

关键词必须经过 NFKC、空格和重复项规范化。不得上传本地规则 ID、时间、使用记录、备注或触发上下文。

### 3. 礼物规则 `gift_rule / upsert`

只允许公共业务投影：

- `serviceName`
- `mode`
- `unitPrice`

约束：

- `mode=fixed` 时 `unitPrice` 必须是合法正数；
- `mode=variable` 时 `unitPrice` 必须为 `null`；
- 不上传 `id`、`serviceKey`、使用次数、最近使用时间或订单来源。

### 4. 显式删除 `operation=delete`

删除只能来自明确用户操作，不能根据“本地对象消失”“导入后缺少”“迁移差异”“云端拉取”“回滚”或整库比较推断。

本阶段协议允许为已支持公共类型构造删除候选，但必须满足：

- 明确目标数据类型和稳定业务键；
- `payload=null`；
- 目标公共基线存在；
- 始终进入人工审核；
- 不删除历史事件，不覆盖旧批准记录；
- 批准后仅发布新的不可变墓碑事件。

整库覆盖、库重命名和重新绑定不属于普通删除，不在阶段6A开放。

### 5. 老板敏感变化 `boss_profile / upsert`

继续沿用阶段5G的 `bossId`、`bossName`、`paiDan`、`discount`公共模型。以下变化全部归类为敏感候选并人工审核：

- 直属/派单变化；
- 折数升高；
- 异常大幅折数下降；
- 老板名称变化或身份映射不明确。

折数仍必须位于 `0.8–1`。超出范围属于无效提交，不进入可批准审核队列。

## 统一协议要求

1. 继续使用冻结的提交顶层 envelope、schemaVersion、payloadSchemaVersion、clientContext、幂等键和请求大小限制。
2. 客户端给出的 `businessKey`、`contentHash`、`bossId`均不可信，服务端必须重新规范化并重算。
3. 仅允许 `origin=user` 或用户明确选择参与协作后产生的 `origin=initialBinding`。
4. `import`、`migration`、`cloudPull`、`rollback`、`system`永不生成敏感候选。
5. 同一幂等键与相同请求体返回同一结果；同一幂等键出现不同请求体必须冲突失败。
6. 所有敏感候选固定返回：
   - `decision=pending_review`
   - `autoApprovalEnabled=false`
   - `publicMutationAllowed=false`
   - 明确、稳定的审核原因码
7. 客户端过滤不是安全边界；服务器必须递归拒绝永不上传字段、多余字段、控制字符、联系方式、异常数字和超限结构。
8. 普通价格、陪玩名字、新老板和合理折数下降的阶段5G行为不得改变。

## 永不上传清单继续生效

完整历史、订单、项目明细、原始聊天、备注、金额与收益、最近老板、使用次数、最近使用时间、自定义比例、锁定字段、DIY布局、布局模板、名字修正上下文、本地ID、本地时间、备份元数据、撤销快照、设备令牌、管理员凭据、API Token和服务器密钥。

## 环境与作用域门禁

阶段6A新增独立预览门禁，默认关闭：

```text
CLOUD_SENSITIVE_RULES_PREVIEW_ENABLED=0
CLOUD_SENSITIVE_RULES_BLOB_STORE_NAME=cloud-collab-preview-v1
CLOUD_SENSITIVE_RULES_ALLOWED_GROUP_ID=group_fixture
CLOUD_SENSITIVE_RULES_ALLOWED_LIBRARY_ID=lib_receive_fixture
```

即使门禁开启，也只允许合成 fixture 作用域。正式公共写入、正式自动批准和正式管理员发布仍保持关闭。

## 本阶段实施顺序

1. 冻结敏感 payload、operation、身份与 Hash 规范；
2. 实现服务器规范化、递归隐私阻断和请求大小校验；
3. 实现敏感候选统一人工审核分类；
4. 增加确定性 Hash、幂等、删除墓碑和老板敏感变化测试；
5. 增加 Stage5G 兼容回归，证明普通类型行为不变；
6. 完成静态隐私门禁与完整 CI。

## 明确排除

阶段6A不包含：

- 普通用户页面的敏感规则保存监听和上传入口；
- 管理员敏感审核页面与真实批准写入；
- 敏感公共事件、快照和同步合并的最终接入；
- 整库导入覆盖、库重命名、重新绑定；
- 正式公共库开关；
- EdgeOne真实部署或验收；
- 稳定版 `码单器8.2.25.html` 修改；
- 阶段7发布与维护工作。
