# 阶段7T：正式管理员身份隔离加固

## 背景

阶段7S的正式敏感候选入队和正式管理员登录/会话由两条并发主线进入仓库。对已合并管理员身份实现进行独立安全审计后，确认四项需要失败关闭的隔离边界：

1. 管理员处理器不能只依据管理员子开关选择，否则生产项目关闭管理员能力时可能回退阶段5A预览处理器；
2. 正式会话不能继续使用预览issuer；
3. 正式登录限流不能继续使用`admin-preview-rate/login/`前缀；
4. 管理员能力关闭后仍应允许精确同源退出清除旧Cookie。

本阶段只加固上述边界，不新增接口、不启用生产环境、不改变8.2.31候选或稳定版。

## 模式分发

管理员身份路由继续使用：

```text
POST /api/admin/auth/login
GET  /api/admin/auth/session
POST /api/admin/auth/logout
```

处理器选择改为由生产总开关决定：

```text
CLOUD_PRODUCTION_ENABLED=0或未配置 → 阶段5A隔离预览处理器
CLOUD_PRODUCTION_ENABLED=1          → 正式处理器
其他值                              → 503 PRODUCTION_FLAG_INVALID
```

正式处理器内部再验证`CLOUD_ADMIN_PRODUCTION_ENABLED=1`。因此生产项目中管理员子开关为0时返回503，不会回退预览处理器，也不会解析登录正文或创建Blob Store。

## 会话域隔离

新增`src/server/production_admin_auth_v1.js`，正式会话使用独立issuer：

```text
cloud-collab-admin-production
```

阶段5A预览会话继续使用：

```text
cloud-collab-admin-preview
```

即使测试中人为复用相同会话密钥，两类令牌也不能交叉验证。正式Cookie仍保持：

```text
15分钟有效
HttpOnly
Secure
SameSite=Strict
Path=/api/admin
```

## 登录限流隔离

正式登录限流Key固定为：

```text
admin-production-rate/login/<salted-hmac>/<slot>.json
```

阶段5A预览前缀保持`admin-preview-rate/login/`。正式限流对象只写入`cloud-collab-admin-production-v1`，Key不包含用户名、客户端地址、密码或盐值明文。

## 退出清理

登录和会话查询仍要求完整正式管理员配置。退出只需要有效的`CLOUD_ADMIN_PUBLIC_ORIGIN`并通过精确同源校验，因此管理员子开关关闭或管理员秘密移除后，浏览器仍可清理旧的API作用域Cookie。

跨站退出仍返回403，不清除Cookie。

## 响应边界

正式身份响应只返回脱敏身份、外部/协议作用域和能力标志：

```text
productionAdmin=true
reviewQueueRead=false
reviewMutation=false
deviceMutation=false
rollback=false
export=false
publicMutationAllowed=false
stablePromotionAuthorized=false
realSecretValuesExposed=false
```

不会返回会话令牌正文、密码、密钥、盐值或Blob对象。

## 自动化验证

更新`tests/stage7s-production-admin-auth.test.mjs`与既有专项工作流，覆盖：

- 生产总开关决定处理器；
- 生产管理员关闭不回退预览；
- 正式与预览issuer不能交叉验证；
- 正式限流使用独立前缀；
- 错误来源和关闭状态在正文解析、Store创建前阻断；
- 正式管理员关闭后同源退出仍清除Cookie；
- 弱密钥、错误Store、错误用户名失败关闭；
- 阶段5A预览身份回归；
- Cloud Function相对导入回归；
- 完整Node与浏览器矩阵。
