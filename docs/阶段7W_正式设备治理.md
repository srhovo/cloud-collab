# 阶段7W：正式设备治理

## 目标

把阶段5D已经验证的设备列表、详情、可信、撤销可信、封禁和解封能力接入正式生产运行时，同时保留阶段5D隔离预览兼容。

本阶段不部署EdgeOne、不访问真实Blob、不生成或保存真实密钥、不修改8.2.31候选，也不晋升8.3.0。

## 正式路由

```text
GET  /api/admin/devices
GET  /api/admin/devices/detail?id=<device-ref>
POST /api/admin/devices/trust
POST /api/admin/devices/revoke-trust
POST /api/admin/devices/block
POST /api/admin/devices/unblock
```

六个Cloud Function只调用`admin_device_governance_mode_dispatch_v1.js`。

## 模式分发

```text
CLOUD_PRODUCTION_ENABLED=0或未配置 → 阶段5D隔离预览处理器
CLOUD_PRODUCTION_ENABLED=1          → 正式处理器
其他值                              → 503失败关闭
```

进入生产模式后，再校验管理员身份和人工审核子开关。生产项目中子开关关闭只能返回503，不得回退预览处理器。

## 正式门禁

正式设备治理要求：

```text
CLOUD_PRODUCTION_ENABLED=1
CLOUD_PRODUCTION_READ_SYNC_ENABLED=1
CLOUD_ADMIN_PRODUCTION_ENABLED=1
CLOUD_PRODUCTION_ADMIN_REVIEW_ENABLED=1
```

并固定：

```text
公共Store：cloud-collab-production-v1
外部作用域：see / see_cz
协议作用域：group_see / lib_see_cz
管理员来源：CLOUD_ADMIN_PUBLIC_ORIGIN
设备引用盐值：CLOUD_ADMIN_DEVICE_REF_SALT
```

治理状态与设备档案位于公共生产Store；管理员登录限流仍使用独立管理员Store。

## 身份与隐私

正式设备治理只接受`cloud-collab-admin-production` issuer会话。

- GET要求同站请求；
- POST要求精确同源`Origin`；
- 跨站POST在正文解析和Store创建前阻断；
- 对外只返回不可逆`deviceRef`；
- 原始`deviceId`、设备令牌、请求摘要、事件ID、Blob Key和盐值不得进入响应；
- 响应声明`realSecretValuesExposed=false`和`stablePromotionAuthorized=false`。

## 治理动作

```text
trust         原因：verified_operator
revoke_trust  原因：trust_withdrawn
block         原因：abuse | credential_compromise | rate_abuse | manual_safety
unblock       原因：manual_review_cleared
```

正文固定为：

```json
{
  "schemaVersion": 1,
  "deviceRef": "devref_v1_...",
  "requestId": "dgrq_v1_...",
  "reasonCode": "..."
}
```

所有治理事件、请求索引和审计不可变；同一请求精确重放返回原结果。封禁会撤销可信，解封不会自动恢复可信。

## 数据边界

设备治理会修改设备的信任/封禁状态，但不会直接修改公共价格、老板资料、规则或墓碑，因此响应保持：

```text
publicMutationAllowed=false
syntheticFixtureOnly=false
stablePromotionAuthorized=false
```

## 自动验证

专项工作流覆盖：

- 正式生产配置与作用域；
- production issuer会话；
- 公共生产Store；
- 不可逆`deviceRef`投影；
- 设为可信真实协议转发；
- 跨站前置阻断；
- 生产子开关关闭不回退预览；
- 六个Cloud Function仅依赖模式分发器；
- 阶段5D、阶段7S、阶段7T与生产运行时继承回归；
- Cloud Function相对导入审计。
