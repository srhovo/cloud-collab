# 阶段8A及后续负责人操作清单

## 当前阶段

阶段8A代码、测试和CI期间，负责人不需要操作EdgeOne，不需要购买域名，不需要创建Blob，不需要生成密钥，也不需要重新部署8.2.31候选。

## 将来具备长期HTTPS入口后

以下操作必须由拥有EdgeOne控制台权限的负责人执行。开始前应先确认已经拥有可长期使用的管理员HTTPS来源；带`eo_token`的临时预览网址不能填写为正式来源。

### 1. 创建公共生产Blob

控制台路径：

```text
EdgeOne Pages控制台
→ cloud-collab项目
→ 存储 / Blob
→ 创建存储空间
```

名称：

```text
cloud-collab-production-v1
```

预期结果：控制台显示存储空间创建成功并可被当前项目绑定。

失败处理：保留错误截图，不要改仓库中的固定名称，不要使用`cloud-collab-preview-v1`代替。

### 2. 创建管理员生产Blob

同一路径再创建：

```text
cloud-collab-admin-production-v1
```

预期结果：两个存储空间同时存在且名称不同。

失败处理：不得把管理员会话、登录限流或管理元数据放入公共生产Blob。

### 3. 生成真实密钥

在受信任电脑的本地安全目录运行：

```bash
npm ci --ignore-scripts
npm run production:secrets:generate -- --output /安全路径/cloud-collab-production-secrets.env
```

生成后：

- 立即保存到密码管理器；
- 不上传GitHub；
- 不发送到聊天；
- 不截图展示；
- 不使用聊天中出现过的旧管理员密码。

### 4. 录入环境变量

控制台路径：

```text
EdgeOne Pages控制台
→ cloud-collab项目
→ 设置
→ 环境变量
```

首次录入时，所有能力开关保持：

```text
0
```

作用域和Store名称按`config/production.env.template`逐项录入。管理员用户名固定：

```text
xiaxue
```

管理员和普通用户来源必须填写纯HTTPS Origin，不带路径、查询参数、片段或临时令牌。

预期结果：变量列表存在，但所有生产能力仍关闭。

失败处理：变量值不符合32至256字节、重复使用、来源不是纯HTTPS或名称不一致时停止，不要尝试放宽代码门禁。

### 5. 执行一次性空库初始化

仅在两个Blob和全部变量准备好后执行。初始化阶段必须保持全部生产能力关闭，只临时设置：

```text
CLOUD_PRODUCTION_BOOTSTRAP_ENABLED=1
CLOUD_PRODUCTION_BOOTSTRAP_CONFIRMATION=INITIALIZE-see-see_cz-V1
```

完成后立即恢复：

```text
CLOUD_PRODUCTION_BOOTSTRAP_ENABLED=0
CLOUD_PRODUCTION_BOOTSTRAP_CONFIRMATION=
```

预期结果：初始化报告显示固定资源全部创建或精确重放，无删除、无能力开启。

失败处理：不要手工补写部分Blob对象；保留报告并停止。

### 6. 分阶段开启与验收

顺序：

```text
1. CLOUD_PRODUCTION_ENABLED=1
2. CLOUD_PRODUCTION_READ_SYNC_ENABLED=1
3. CLOUD_ADMIN_PRODUCTION_ENABLED=1
4. CLOUD_PRODUCTION_EXPORT_ENABLED=1
5. 真实生成一次迁移ZIP并离线核验
6. 其余普通提交、自动审核、人工审核、设备治理和敏感提交按发布计划继续
```

迁移导出不要求提前开启写入能力。

### 7. 迁移导出真实验收

管理员入口登录后：

```text
GET  /api/admin/exports/summary
POST /api/admin/exports/download
```

下载请求确认词：

```text
EXPORT_PRODUCTION_MIGRATION_PACKAGE
```

预期结果：

- ZIP可正常解压；
- 包含manifest、schema、library、prices、playable-names、bosses、rules、tombstones和audit摘要；
- manifest文件Hash全部匹配；
- 同一requestId重放得到相同packageId；
- 换新requestId可导出新的公共版本；
- 公共版本不因导出而变化。

失败处理：立即把`CLOUD_PRODUCTION_EXPORT_ENABLED`恢复为0，保留错误码和不含密钥的响应头截图，不删除Blob对象。
