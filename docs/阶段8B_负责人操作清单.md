# 阶段8B：负责人操作清单

## 当前结论

阶段8B代码和自动化验证期间，负责人**无需进行任何操作**。

当前不要做：

```text
不要重新部署普通用户8.2.31候选
不要创建管理员真实站点
不要填写聊天中出现过的旧密码
不要把真实密钥发送到聊天或GitHub
不要开启任何生产开关
不要手工写入Blob
不要授权8.3.0稳定晋升
```

管理员真实部署要等你拥有可控制的长期HTTPS来源后再执行。带`eo_token`的3小时临时链接不能作为`CLOUD_ADMIN_PUBLIC_ORIGIN`。

---

## 将来需要你亲自执行的步骤

### 步骤1：准备长期管理员HTTPS来源

可接受形式：

```text
https://admin.你的域名
```

要求：

- 你能控制该域名DNS；
- 只填写Origin，不带路径、查询参数或片段；
- 管理员来源与普通用户来源可以是同一主域名下的不同子域名；
- 中国大陆节点按EdgeOne当时规则完成备案要求。

预期结果：浏览器直接打开管理员域名，不需要临时令牌。

失败处理：没有长期来源时停止在本步骤，继续保留代码与CI成果，不使用平台临时预览链接冒充正式来源。

### 步骤2：创建公共生产Blob

控制台路径：

```text
EdgeOne Pages控制台
→ cloud-collab项目
→ 存储 / Blob
→ 创建存储空间
```

名称必须为：

```text
cloud-collab-production-v1
```

预期结果：存储空间显示可用。

失败处理：保留错误截图；不要改代码中的固定名称，不要使用`cloud-collab-preview-v1`代替。

### 步骤3：创建管理员生产Blob

同一路径再次创建：

```text
cloud-collab-admin-production-v1
```

预期结果：公共与管理员两个存储空间同时存在且相互独立。

失败处理：不要把管理员登录限流、会话相关元数据或管理请求索引放进公共Blob。

### 步骤4：在受信任电脑生成真实密钥

在仓库根目录执行：

```bash
npm ci --ignore-scripts
npm run production:secrets:generate -- --output /你的安全目录/cloud-collab-production-secrets.env
```

执行后：

1. 把文件导入密码管理器；
2. 不上传云盘公开链接；
3. 不提交GitHub；
4. 不发送到聊天；
5. 不截图展示；
6. 不使用此前聊天中出现过的管理员密码。

预期结果：生成8项彼此不同、至少32字节的值。

失败处理：删除不完整输出，检查Node版本和目录权限后重新生成；不要人工用重复字符串补齐长度。

### 步骤5：录入EdgeOne私密环境变量

控制台路径：

```text
EdgeOne Pages控制台
→ 管理员项目（将来单独创建）
→ 设置
→ 环境变量
```

按`config/production.env.template`逐项录入。

首次录入时所有开关保持：

```text
CLOUD_PRODUCTION_ENABLED=0
CLOUD_PRODUCTION_READ_SYNC_ENABLED=0
CLOUD_PRODUCTION_ORDINARY_SUBMISSION_ENABLED=0
CLOUD_PRODUCTION_AUTO_APPROVAL_ENABLED=0
CLOUD_PRODUCTION_SENSITIVE_SUBMISSION_ENABLED=0
CLOUD_PRODUCTION_ADMIN_REVIEW_ENABLED=0
CLOUD_PRODUCTION_DEVICE_GOVERNANCE_ENABLED=0
CLOUD_PRODUCTION_EXPORT_ENABLED=0
CLOUD_ADMIN_PRODUCTION_ENABLED=0
CLOUD_PRODUCTION_BOOTSTRAP_ENABLED=0
```

正式回滚没有独立环境变量；其生产处理器要求只读同步、管理员身份和管理员审核三项能力同时开启。

固定非秘密配置：

```text
CLOUD_PRODUCTION_EXTERNAL_CLUB_ID=see
CLOUD_PRODUCTION_EXTERNAL_LIBRARY_ID=see_cz
CLOUD_PRODUCTION_GROUP_ID=group_see
CLOUD_PRODUCTION_LIBRARY_ID=lib_see_cz
CLOUD_PRODUCTION_BLOB_STORE_NAME=cloud-collab-production-v1
CLOUD_ADMIN_PRODUCTION_BLOB_STORE_NAME=cloud-collab-admin-production-v1
CLOUD_ADMIN_USERNAME=xiaxue
CLOUD_ADMIN_PUBLIC_ORIGIN=https://admin.你的域名
```

预期结果：环境变量全部存在，但生产能力仍未开启。

失败处理：出现变量名不匹配、密钥长度不足、密钥重复、Origin不是纯HTTPS或Store名称错误时停止，不要放宽代码门禁。

### 步骤6：创建独立管理员EdgeOne项目

建议不要把管理员页面加入现有普通用户项目。

控制台路径：

```text
EdgeOne Pages控制台
→ 新建项目
→ 导入GitHub仓库 srhovo/cloud-collab
```

构建设置：

```text
分支：main
Node.js：22.11.0
安装命令：npm ci --ignore-scripts
构建命令：npm run edgeone:admin:build
输出目录：.edgeone-admin-artifact
```

绑定管理员子域名后，填写与该域名完全一致的`CLOUD_ADMIN_PUBLIC_ORIGIN`。

预期结果：部署产物只有：

```text
index.html
admin-release.json
```

失败处理：若产物出现`build-manifest.json`、`pages-release.json`或普通码单器主页，立即停止部署并保留构建日志。

### 步骤7：执行一次性空库初始化

只有两个Blob、全部变量和长期HTTPS来源准备完成后执行。

初始化时所有生产能力仍保持0，只临时设置：

```text
CLOUD_PRODUCTION_BOOTSTRAP_ENABLED=1
CLOUD_PRODUCTION_BOOTSTRAP_CONFIRMATION=INITIALIZE-see-see_cz-V1
```

执行完成后立即恢复：

```text
CLOUD_PRODUCTION_BOOTSTRAP_ENABLED=0
CLOUD_PRODUCTION_BOOTSTRAP_CONFIRMATION=
```

预期结果：初始化报告显示固定对象全部创建或精确重放，无删除、无能力开启。

失败处理：不要手工补写部分对象；保留初始化报告，恢复开关为0后停止。

### 步骤8：按顺序开启并验收

严格顺序：

```text
1. 生产总开关
2. 只读同步
3. 管理员身份
4. 完整导出
5. 普通提交
6. 普通自动审核
7. 管理员人工审核（此时正式回滚条件才具备）
8. 设备治理
9. 公共数据回滚真实验收
10. 敏感提交与敏感人工审核
```

每次只改一个真实存在的开关，重新部署并完成对应真实设备测试后再继续。第9项回滚验收不需要新增开关，而是在第7项门禁已开启后执行。

失败处理：任何阶段失败，立即把刚开启的真实开关恢复为0；不要删除Blob，不要同时调整多个密钥或开关。

### 步骤9：管理员控制台L4验收

需要你在真实浏览器完成：

```text
桌面Chromium
Android浏览器
iPhone Safari
```

逐项确认：

1. 登录后密码输入框为空；
2. 刷新页面可恢复HttpOnly会话；
3. 退出后所有模块数据清空；
4. 精确价格批准/拒绝/编辑后批准；
5. 普通资料审核；
6. 敏感批准、拒绝、编辑后批准和墓碑；
7. 设备可信、撤销、封禁、解封；
8. 回滚只追加新版本；
9. 完整ZIP导出和离线解压；
10. 会话过期后自动返回登录页并清空业务数据。

失败处理：先关闭对应生产子开关；回滚异常则先关闭`CLOUD_PRODUCTION_ADMIN_REVIEW_ENABLED`。保存不含密钥的错误码、时间、浏览器和截图，再由我分析。
