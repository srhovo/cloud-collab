# 阶段7G：8.2.30候选生成与最终发布清单

## 一、授权与目标

项目负责人已于2026-07-21明确：

1. 候选版本采用`8.2.30`；
2. 接受清理精确删除数量和两次独立零对象数字缺失的单独证据豁免；
3. 继续生成候选单文件和最终发布清单；
4. 暂不晋升稳定版。

本阶段属于发布候选命名、构建和证据封装，不新增业务功能。

## 二、目标文件与基线

- 冻结稳定基线：`码单器8.2.25_现.html`；
- 稳定基线SHA-256：`d34a436d5910ab027ad466309c44c6607fc8b60d2b21cf4b1cc4bf5a188bd6d3`；
- 上一兼容候选：`8.2.28`；
- 本阶段候选：`8.2.30`；
- 候选单文件：`release/candidates/码单器8.2.30_候选.html`；
- 最终发布清单：`release/final-release-manifest-v1.json`。

稳定版8.2.25不进入候选构建输入，也不会被候选构建覆盖。

## 三、构建方式

`scripts/build-stage7g-release-candidate.mjs`复用已经完成阶段5G、6B和7A至7F验证的兼容构建链路：

```text
阶段5G/6B模块化源码
→ build-stage6b-compatible.mjs
→ 已验证8.2.28兼容候选
→ 受控版本迁移
→ 8.2.30完整候选单文件
```

版本迁移同时核对：

- 页面标题为`码单器8.2.30（公共协作完整候选版）`；
- `APP_VERSION`为`8.2.30`；
- 候选中不残留`8.2.28`；
- 候选单文件与`dist/index.html`字节完全一致；
- 构建清单中的版本、大小和SHA-256与候选一致。

这属于纯候选命名迁移和发布封装，不宣称新增8.2.30独有业务能力。

## 四、最终发布清单

构建生成的`release/final-release-manifest-v1.json`记录：

- 稳定基线版本、文件名、字节数和SHA-256；
- 8.2.30候选文件名、仓库路径、标题、字节数和SHA-256；
- 项目负责人版本决定；
- 最终人工重跑豁免；
- 清理精确证据缺失豁免；
- 临时EdgeOne资源销毁状态；
- 正式公共写入未开启；
- EdgeOne部署和Blob变更未执行；
- 稳定版晋升未授权、未执行。

发布状态固定为：

```text
candidate_ready_not_promoted
```

## 五、CI与验证

阶段7G属于发布节点，采用L4工程自动化门禁：

- Node完整测试；
- 兼容候选结构验证；
- 阶段5F、5G、6A、6B专项验证；
- 核心比较和浏览器集成；
- 管理员阶段5A至5G浏览器回归；
- 阶段6B敏感候选用户端和管理员端浏览器回归；
- 阶段7G候选文件、发布清单和稳定边界专项验证。

GitHub Actions会上传`stage7g-8.2.30-candidate`产物包，用于把CI实际生成并验证过的候选和清单回填仓库。

## 六、明确不执行

- 不覆盖或改名稳定版8.2.25；
- 不把8.2.30标记为稳定版；
- 不创建EdgeOne正式部署；
- 不开启正式公共写入；
- 不执行Blob写入或删除；
- 不修改存储key、备份结构、导入导出结构或业务算法；
