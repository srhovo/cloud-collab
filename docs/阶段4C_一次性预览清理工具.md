# 阶段4C：一次性预览 Blob 清理工具

## 目标

该工具只用于清理 EdgeOne Blob 命名空间 `cloud-collab-preview-v1` 中由阶段4B.2真实联调生成的合成测试对象。它不是通用删除接口，不得用于正式公共数据，也不得长期保留。

## 固定安全边界

- `CLOUD_WRITE_PREVIEW_ENABLED` 必须保持 `0`；清理能力与预览写入不能同时开启。
- `CLOUD_BLOB_STORE_NAME` 必须严格等于 `cloud-collab-preview-v1`。
- group/library 必须严格等于 `group_fixture / lib_receive_fixture`。
- 清理功能另设 `CLOUD_PREVIEW_CLEANUP_ENABLED`，默认值必须为 `0`。
- 需要服务器侧固定确认词和现有预览访问凭据；真实值不得提交到仓库、HTML、日志、截图或聊天。
- 仅接受 POST/OPTIONS；配置门禁和访问凭据校验均在读取请求体、初始化 Blob Store 之前完成。
- 先强一致列举全部对象，再对每个 Key 做白名单校验；只要出现一个未知 Key，整次操作在首次删除前中止。
- 仅允许阶段4B.2已知合成对象族：设备档案、令牌 Hash 索引、测试库 pending submission、注册/提交限流槽。
- 删除后再次强一致列举，只有剩余对象数为0才判定完成。
- 响应只返回数量和 Key 集合摘要，不返回原始 Key、令牌、盐值或访问凭据。
- 最大处理对象数为10000；超过上限时拒绝运行。

## 一次性运行顺序

1. 独立分支完成代码、单元测试、静态边界检查和 PR 审核。
2. 部署该分支到受控预览环境；确认普通预览写入开关仍为 `0`。
3. 仅在清理窗口由部署端临时设置清理开关和固定确认词，不把值发送到聊天或写入仓库。
4. 由具备现有预览访问凭据的操作者调用一次清理路由。
5. 仅记录响应中的 `deletedCount`、`remainingCount` 和 `keySetDigest`；不得记录请求凭据或完整对象名。
6. 核验 `completed=true`、`remainingCount=0`。
7. 立即把清理开关恢复为 `0`。
8. 立即从仓库删除 `cloud-functions/one-shot/cleanup-preview.js`、`src/server/preview_cleanup_v1.js`、`src/server/preview_cleanup_http_v1.js` 及相应临时环境变量；再次通过 CI 和 PR 审核后重新部署。

## 失败与回滚

Blob 删除不可逆，因此这里的“回滚”不是恢复测试对象，而是阻止错误删除：

- 配置、作用域、命名空间、确认词或凭据任一不匹配时，不初始化 Blob Store。
- 发现未知对象时，不删除任何对象。
- 删除中途失败或删除后仍有对象时，工具必须保留并保持清理开关关闭，先核验错误再重新运行；不得在结果未确认前删除工具代码。
- 被允许删除的对象仅是可重新生成的合成测试对象，不包含正式用户数据。

## 与客户端8.2.28的关系

清理工具生命周期与客户端开发使用不同分支和 PR。客户端分支不得依赖该路由，不得内置预览访问密钥，也不得因为清理工具存在而开启正式公共库写入、自动批准或管理员审核。
