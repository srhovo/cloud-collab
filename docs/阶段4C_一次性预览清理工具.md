# 阶段4C：一次性预览 Blob 清理工具

## 目标

该工具只用于清理 EdgeOne Blob 命名空间 `cloud-collab-preview-v1` 中由阶段4B.2真实联调生成的合成测试对象。它不是通用删除接口，不得用于正式公共数据，也不得长期保留。

## 固定安全边界

- `CLOUD_WRITE_PREVIEW_ENABLED` 必须保持 `0`；清理能力与预览写入不能同时开启。
- `CLOUD_BLOB_STORE_NAME` 必须严格等于 `cloud-collab-preview-v1`。
- group/library 必须严格等于 `group_fixture / lib_receive_fixture`。
- 清理功能另设 `CLOUD_PREVIEW_CLEANUP_ENABLED`，默认值必须为 `0`。
- 必须使用独立的 `CLOUD_PREVIEW_CLEANUP_KEY`；不得复用 `CLOUD_WRITE_PREVIEW_KEY` 或 `CLOUD_RATE_LIMIT_SALT`。
- 真实清理密钥和固定确认词只保存在部署环境中，不得提交到仓库、HTML、日志、截图或聊天。
- 仅接受同源 POST；不返回跨域许可，响应使用 `Cross-Origin-Resource-Policy: same-origin`。
- 配置门禁和独立清理凭据校验均在读取请求体、初始化 Blob Store 之前完成。
- 清理分为两步：先 `inspect`，再使用检查结果中的 `keySetDigest` 执行 `execute`。
- `execute` 会重新强一致列举并核对对象集合摘要；对象集合发生任何变化时，在首次删除前中止。
- 先强一致列举全部对象，再对每个 Key 做白名单校验；只要出现一个未知 Key，整次操作在首次删除前中止。
- 仅允许阶段4B.2已知合成对象族：设备档案、令牌 Hash 索引、测试库 pending submission、注册/提交限流槽。
- 删除后再次强一致列举，只有剩余对象数为0才判定完成。
- 响应只返回数量和 Key 集合摘要，不返回原始 Key、令牌、盐值或访问凭据。
- 最大处理对象数为500；超过上限时拒绝运行并要求人工核查，避免一次性函数超时导致部分删除。

## 按钮式操作页

构建产物包含一次性页面：

```text
/cleanup-preview.html
```

该页面：

- 只向同源 `/one-shot/cleanup-preview` 发出 POST；
- 自动把当前 EdgeOne 预览地址中的 `eo_token`、`eo_time` 传给接口；
- 不把临时清理密码写入 localStorage、sessionStorage、IndexedDB、日志或页面结果；
- 必须先点击“检查数量（不会删除）”；
- 检查成功后才保存对象集合摘要并显示数量；
- 只有勾选人工确认后，红色删除按钮才会启用；
- 点击删除后还会显示浏览器二次确认；
- 只有接口返回 `completed=true` 且 `remainingCount=0` 才显示完成。

## 一次性运行顺序

1. 独立分支完成代码、单元测试、静态边界检查和 PR 审核。
2. 确认 `CLOUD_WRITE_PREVIEW_ENABLED=0`。
3. 临时设置：
   - `CLOUD_PREVIEW_CLEANUP_ENABLED=1`
   - 新生成且未复用的 `CLOUD_PREVIEW_CLEANUP_KEY`
   - `CLOUD_PREVIEW_CLEANUP_CONFIRMATION=DELETE_SYNTHETIC_PREVIEW_V1`
4. 部署 `stage4c-preview-cleanup` 到受控预览环境。
5. 从该部署的 EdgeOne“预览”入口打开预览地址；在同一地址中访问 `/cleanup-preview.html`，必须保留 EdgeOne 自动生成的预览访问参数。
6. 在页面中粘贴临时清理密码，点击“检查数量（不会删除）”。
7. 人工核对对象数量是否符合预期；数量异常时立即停止，不勾选确认，不点击删除。
8. 数量合理时勾选确认，点击红色“删除这些测试对象”，再确认一次。
9. 核验页面显示：已删除数量、剩余 `0` 个。
10. 立即把 `CLOUD_PREVIEW_CLEANUP_ENABLED` 恢复为 `0`，并删除部署环境中的 `CLOUD_PREVIEW_CLEANUP_KEY` 与 `CLOUD_PREVIEW_CLEANUP_CONFIRMATION`。
11. 合并专用移除 PR 到清理分支并重新部署；确认 `/cleanup-preview.html` 与 `/one-shot/cleanup-preview` 均已不存在。
12. 关闭清理 PR，不合并到 `main`。

## 请求结构

检查：

```json
{
  "schemaVersion": 1,
  "action": "inspect",
  "confirmation": "DELETE_SYNTHETIC_PREVIEW_V1"
}
```

执行：

```json
{
  "schemaVersion": 1,
  "action": "execute",
  "confirmation": "DELETE_SYNTHETIC_PREVIEW_V1",
  "expectedKeySetDigest": "inspect返回的摘要"
}
```

请求头使用 `X-Cloud-Collab-Cleanup-Key`。文档不提供、示例不填充任何真实密钥。

## 失败与回滚

Blob 删除不可逆，因此这里的“回滚”不是恢复测试对象，而是阻止错误删除：

- 配置、作用域、命名空间、确认词或独立凭据任一不匹配时，不初始化 Blob Store。
- 清理密钥与预览写入密钥或限流盐值相同时，拒绝启动。
- `inspect` 阶段只读不删。
- 发现未知对象时，不删除任何对象。
- `execute` 的摘要与当前对象集合不一致时，不删除任何对象，必须重新检查。
- 删除中途失败或删除后仍有对象时，工具必须保留并保持清理开关关闭，先核验错误再重新运行；不得在结果未确认前删除工具代码。
- 被允许删除的对象仅是可重新生成的合成测试对象，不包含正式用户数据。

## 与客户端8.2.28的关系

清理工具生命周期与客户端开发使用不同分支和 PR。客户端分支不得依赖该路由，不得内置预览访问密钥或清理密钥，也不得因为清理工具存在而开启正式公共库写入、自动批准或管理员审核。
