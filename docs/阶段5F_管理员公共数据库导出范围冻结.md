# 阶段5F：管理员公共数据库导出范围冻结（默认关闭）

## 基线

- 正式开发基线：`main@ecd7b513f1dc226bf443d7101314f4adf2dc8bbe`。
- 该基线已经包含阶段5A管理员登录、阶段5B只读审核、阶段5C审核写入、阶段5D设备治理和阶段5E公共数据回滚。
- 本阶段独立分支：`agent/stage5f-admin-export`。

## 唯一主目标

建立管理员公共数据库导出能力，为后续迁移、备份和平台替换提供不依赖EdgeOne专有结构的标准包。

本阶段只导出已经通过既有公共事件、批准索引和快照校验的合成公共数据，不导出用户本地数据、设备凭据、原始提交正文或管理员秘密。

## 默认关闭与作用域

仓库模板新增独立门禁并保持：

```text
CLOUD_ADMIN_EXPORT_PREVIEW_ENABLED=0
```

导出只允许同时满足：

- 阶段5A管理员会话有效；
- 同项目HTTPS与同源校验通过；
- Blob Store固定为`cloud-collab-preview-v1`；
- groupId固定为`group_fixture`；
- libraryId固定为`lib_receive_fixture`；
- 只处理合成公共数据库；
- 不创建正式公共库、不修改正式公共版本。

阶段5F是读取型管理员能力，可以在未来联合验收中与阶段5D设备治理和阶段5E回滚同时开启。导出下载本身不得修改公共事件、公共快照或批准索引。

## 标准导出结构

下载文件为ZIP，根目录固定为：

```text
码单器公共数据库导出/
├─ manifest.json
├─ schema.json
├─ groups.json
├─ libraries/
│  └─ lib_receive_fixture.json
├─ bosses/
│  └─ index.json
├─ playable-names/
│  └─ index.json
├─ rules/
│  └─ index.json
└─ audit/
   ├─ public-events.json
   └─ rollbacks.json
```

首轮真实内容仅包含当前已经实现并验证的普通精确价格。老板、陪玩名字和敏感规则目录保留稳定空结构，为后续协议扩展预留，但不得伪造尚未开放的数据。

## 文件语义

### `manifest.json`

至少包含：

- 导出协议版本；
- 包格式版本；
- 生成时间；
- group/library作用域；
- 当前公共版本；
- 有效公共事件数量；
- 当前快照记录数量；
- 文件清单、UTF-8字节数和SHA-256；
- 与EdgeOne无关的内容包标识。

### `schema.json`

固定描述导出协议、字段类型、当前支持的数据类型和兼容边界。不得包含运行时环境变量、Blob名称或内部路径。

### `groups.json`

只记录导出作用域及其库引用；不包含管理员身份或设备信息。

### `libraries/*.json`

记录当前公共快照和通过验证的普通精确价格。保留迁移所需的公共版本、businessKey、contentHash、批准版本、payload和时间；不得包含设备证据或内部Blob Key。

### `audit/public-events.json`

记录通过批准索引验证的有效公共事件时间线。可包含公共版本、业务Hash、基线摘要、payload、批准模式和证据数量；不得包含deviceId、submissionId、approvalId、eventKey或内部请求Hash。

### `audit/rollbacks.json`

只输出经过严格校验的脱敏回滚摘要：操作时间、来源版本、恢复版本、结果公共版本和事件版本。不得输出actorTag、rollbackId、auditId、请求索引、设备或提交身份。

## 隐私与禁止字段

导出包不得包含：

- deviceId、设备令牌、tokenHash或设备治理内部引用；
- submissionId、idempotencyKey、原始提交正文或待审核队列；
- 管理员用户名、Cookie、会话ID、密码、密钥、盐值或actorTag；
- rollbackId、auditId、requestId、requestHash、approvalId、eventKey、snapshotKey或Blob路径；
- 原始聊天、订单、备注、历史、自定义比例、布局、使用次数、最近老板或本地时间记录；
- EdgeOne项目ID、部署地址、Blob命名空间或平台专有迁移参数。

## 生成与完整性

- 所有JSON使用UTF-8和稳定键排序；
- 每个文件使用SHA-256校验；
- 内容包标识只由文件内容与作用域推导，不依赖EdgeOne对象Key；
- ZIP使用标准格式，解压后无需EdgeOne SDK即可读取；
- 导出生成在请求内完成，不创建公开下载URL，不把ZIP永久保存在Blob；
- 导出包上限固定为10MB，事件对象上限固定为10000；超过上限失败关闭；
- 任何无效公共事件、批准索引、回滚审计或摘要不一致均停止导出，不生成“部分成功”包。

## 管理员接口与明确确认

计划接口：

```text
GET  /api/admin/exports/summary
POST /api/admin/exports/download
```

下载POST正文固定为：

```json
{
  "schemaVersion": 1,
  "requestId": "<unique-request-id>",
  "confirmation": "EXPORT_SYNTHETIC_PUBLIC_DATABASE"
}
```

规则：

- GET只返回脱敏摘要，不返回文件内容；
- POST必须通过管理员会话、同源Origin和固定确认词校验；
- 响应必须使用`application/zip`和安全的固定文件名；
- 错误响应仍为无缓存JSON；
- 页面不得把ZIP内容、管理员凭据或requestId写入LocalStorage或SessionStorage；
- 下载失败不得退化为返回内部对象、堆栈或Blob路径。

## 审计边界

管理员下载需要写入一条服务端私有不可变导出审计摘要，记录脱敏管理员标签、生成时间、内容包标识、公共版本和文件数量。该审计不得进入ZIP，也不得修改公共数据。

相同requestId和相同正文重放应关联同一审计请求；相同requestId对应不同正文或另一管理员时失败关闭。

## 明确不包含

- 用户本地备份导出功能修改；
- 正式公共库或正式管理员导出启用；
- 自动定时导出、每日任务或对象保留策略；
- 导入、恢复或跨平台写回；
- 敏感规则、老板或陪玩名字的新提交协议；
- 阶段6删除和直属变化；
- 修改码单器8.2.25稳定文件或普通用户页面。

## 最低验证门槛

阶段5F至少需要：

- 默认关闭、固定Blob和fixture作用域测试；
- 管理员会话、同项目HTTPS和同源校验测试；
- 固定确认词、请求大小和请求字段测试；
- ZIP目录、CRC、文件名和解压测试；
- 文件SHA-256、内容包标识和稳定JSON测试；
- 当前快照、有效公共事件和导出库一致性测试；
- 回滚后导出结果与最新公共值一致测试；
- 无效事件、无效批准索引和无效回滚审计失败关闭测试；
- 隐私禁止字段扫描；
- 10MB与10000事件硬上限测试；
- 同requestId幂等和不同正文冲突测试；
- 页面下载、会话恢复、最终状态、浏览器存储为空和精确按钮名称回归；
- 完整Node、静态隐私门禁、冻结核心计算对比和Chromium回归。

## 联合EdgeOne验收策略

阶段5F代码和CI独立闭环后，不单独做EdgeOne验收。随后创建一次管理员能力联合EdgeOne验收，覆盖：

- 阶段5D设备治理；
- 阶段5E公共数据回滚；
- 阶段5F公共数据库导出；
- 管理员会话恢复、同源HTTPS、真实Blob、稳定错误码、下载完整性、验收数据清理和资源销毁。

合并验收不等于合并代码。阶段5F正式PR仍需单独获得Ready和合并授权。
