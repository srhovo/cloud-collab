# 阶段6B：敏感候选审核与发布链路范围冻结

## 基线

- 仓库：`srhovo/cloud-collab`
- 基线：`main@80578d59a571241cc8f8b5c76bfec648970f3729`
- 来源：PR #31 阶段6A squash 合并后的新 main
- 开发分支：`agent/stage6b-sensitive-review-publish`
- 稳定文件：`码单器8.2.25.html`继续冻结，不修改、不作为构建输入
- 本阶段不创建 EdgeOne 项目、部署、凭据、设备或 Blob 对象

## 唯一主目标

把阶段6A已经冻结的敏感提交协议接入完整闭环：

1. 普通用户显式产生敏感候选；
2. 敏感候选进入管理员人工审核队列；
3. 管理员明确批准、拒绝或编辑后批准；
4. 批准后发布不可变公共事件；
5. 删除批准发布不可变墓碑事件；
6. 公共快照、增量与本地三方合并识别敏感规则及墓碑。

所有敏感变更始终人工审核。可信设备、两设备一致、首次绑定和管理员设备信任均不得绕过审核。

## 客户端显式入队

### 可入队事件

仅以下明确用户动作可以生成敏感候选：

- 新增或修改区间规则；
- 新增或修改加价规则；
- 新增或修改礼物规则；
- 明确删除公共价格、陪玩名字、老板、区间、加价或礼物规则；
- 修改老板直属/派单；
- 提高老板折数；
- 提交异常大幅折数下降；
- 需要人工确认的老板身份变化。

### 永不入队来源

以下来源不得产生敏感候选：

- `import`
- `migration`
- `cloudPull`
- `rollback`
- `system`
- 仅接收模式
- 页面初始化、候选预览、撤销、恢复、备份导入
- 根据“本地对象消失”或整库差异推断的删除

删除必须由用户在明确删除交互中产生，并绑定当时已知的公共业务键和公共基线 Hash。

### 本地队列要求

- 使用独立敏感候选状态，不与普通待确认候选混淆；
- 保留失败重试、幂等键和稳定 submissionId；
- 不保存管理员凭据、设备令牌明文或原始隐私数据；
- 云端拉取、审核结果应用和墓碑合并不得反向生成新候选；
- 仅投影阶段6A白名单字段。

## 管理员人工审核

### 队列与详情

管理员页面必须支持：

- 按状态、数据类型和时间查看敏感候选；
- 显示候选值、当前公共基线和差异；
- 清晰标识新增、修改、删除、直属变化、折数升高和异常下降；
- 删除候选明确显示目标业务键、目标类型和基线版本；
- 不展示设备令牌、原始聊天、订单、备注或其他永不上传数据。

### 审核动作

允许：

- 批准；
- 拒绝；
- 在严格白名单内编辑后批准。

每个写入动作必须：

- 通过阶段5A管理员会话；
- HTTPS 且同源；
- 使用明确确认词；
- 校验请求幂等；
- 校验候选仍为待审核状态；
- 校验公共基线未过期；
- 写入不可变管理员审计。

禁止批量无确认批准、自动批准、可信设备直通和后台静默修改。

## 公共事件与墓碑发布

### 普通敏感更新

批准后发布新的不可变公共事件，事件包含：

- 数据类型；
- 操作类型；
- 业务键；
- 内容 Hash；
- 严格白名单 payload；
- 前一公共版本和新公共版本；
- 不可逆审核引用；
- 服务器时间。

旧事件和旧快照不得覆盖或删除。

### 显式删除

删除批准后发布墓碑事件：

- `operation=delete`；
- `payload=null`；
- 保留目标业务键、数据类型和必要老板身份；
- 记录被删除公共内容 Hash；
- 公共版本递增一次；
- 重放不得再次递增；
- 不物理删除历史事件、审核记录或旧快照。

目标公共基线不存在、业务键变化或版本陈旧时必须失败关闭并要求重新审核。

## 快照、增量与本地合并

公共读取链路扩展支持：

- `rank_range_rule`
- `surcharge_rule`
- `gift_rule`
- 已支持类型的删除墓碑
- 老板敏感变化

本地接收继续使用三方合并：

- 本地未修改：应用云端更新或墓碑；
- 本地已修改且云端未变：保留本地；
- 本地和云端都变：进入本地冲突状态，不静默覆盖；
- 墓碑与本地修改冲突：保留本地数据并提示冲突，不自动删除；
- 同一事件重放不重复应用。

价格、名字、老板、区间、加价、礼物、同步游标、公共版本和基线 Hash 必须组合提交；失败时整体回滚。

## 安全与环境门禁

阶段6B使用独立预览门禁，默认关闭：

```text
CLOUD_SENSITIVE_REVIEW_PREVIEW_ENABLED=0
CLOUD_SENSITIVE_REVIEW_BLOB_STORE_NAME=cloud-collab-preview-v1
CLOUD_SENSITIVE_REVIEW_ALLOWED_GROUP_ID=group_fixture
CLOUD_SENSITIVE_REVIEW_ALLOWED_LIBRARY_ID=lib_receive_fixture
```

同时要求：

- 阶段5A管理员预览会话门禁；
- 阶段6A敏感规则协议门禁；
- Blob硬锁 `cloud-collab-preview-v1`；
- 作用域硬锁 `group_fixture / lib_receive_fixture`；
- 正式公共写入与正式自动批准保持关闭；
- 稳定版`码单器8.2.25.html`不修改。

## 实施顺序

1. 客户端敏感候选投影与显式入队；
2. 服务端候选列表、详情和差异读取；
3. 管理员批准、拒绝、编辑后批准与不可变审计；
4. 公共事件与删除墓碑发布；
5. 快照、增量和客户端三方合并；
6. 管理员与普通用户 Chromium 回归；
7. Stage5G、Stage6A兼容与隐私回归；
8. 完整 CI 闭环。
