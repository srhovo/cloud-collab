# 阶段7W及后续需要负责人亲自操作的事项

当前阶段7W代码和CI验证不需要负责人操作。以下事项只在进入真实EdgeOne生产资源时才需要执行。

## 1. 创建生产Blob

控制台路径以EdgeOne当前界面为准：

```text
EdgeOne Pages控制台
→ cloud-collab项目
→ 存储 / Blob
→ 创建存储空间
```

需要两个相互独立的存储空间：

```text
cloud-collab-production-v1
cloud-collab-admin-production-v1
```

预期结果：两个名称均显示可用，且不是同一个存储空间。

失败处理：不要改代码中的固定名称，也不要用预览Blob代替；保留错误截图后停止。

## 2. 录入生产环境变量

路径：

```text
EdgeOne Pages控制台
→ cloud-collab项目
→ 设置
→ 环境变量
```

真实密钥必须使用仓库生成器在本地安全路径生成，并逐项复制到控制台；不得发到聊天或提交GitHub。

所有功能开关首次录入时保持`0`。管理员用户名为`xiaxue`；聊天中出现过的旧密码不得使用。

## 3. 配置长期管理员来源

`CLOUD_ADMIN_PUBLIC_ORIGIN`必须是管理员页面最终使用的纯HTTPS来源，不带路径、查询或片段。没有可长期使用的自定义域名时，不执行此步骤，不开启管理员生产能力。

## 4. 一次性初始化

只有两个Blob和全部环境变量准备好后，按专门初始化流程执行。初始化确认词固定为：

```text
INITIALIZE-see-see_cz-V1
```

不得手工在Blob中创建部分对象，不得在生产能力开启时运行初始化。

## 5. 分阶段开启

必须按顺序：

```text
只读同步
→ 普通提交
→ 普通自动审核
→ 管理员身份与人工审核
→ 敏感提交与敏感人工审核
```

设备治理随管理员人工审核阶段启用。每阶段完成真实设备验收后才能开启下一阶段。
