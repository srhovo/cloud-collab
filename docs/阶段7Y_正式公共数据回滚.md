# 阶段7Y：正式公共数据回滚

## 目标

把阶段5E已经通过自动化和浏览器回归的普通精确价格回滚能力接到正式管理员会话与生产公共Blob，同时保留预览模式和全部历史数据。

本阶段不部署EdgeOne、不访问真实Blob、不填写真实密钥，也不晋升稳定版。

## 独立生产门禁

新增：

```text
CLOUD_PRODUCTION_ROLLBACK_ENABLED=0
```

该开关默认关闭并与人工审核、设备治理相互独立。正式回滚要求：

```text
CLOUD_PRODUCTION_ENABLED=1
CLOUD_PRODUCTION_READ_SYNC_ENABLED=1
CLOUD_ADMIN_PRODUCTION_ENABLED=1
CLOUD_PRODUCTION_ROLLBACK_ENABLED=1
```

总开关关闭或初始化模式执行时，回滚开关必须为0。开启正式回滚时，`CLOUD_ADMIN_ROLLBACK_REF_SALT`必须为32至256字节且不得复用其他生产密钥。

## 作用域与存储

```text
外部作用域：see / see_cz
协议作用域：group_see / lib_see_cz
公共生产Blob：cloud-collab-production-v1
管理员身份Blob：cloud-collab-admin-production-v1
```

回滚候选、公共事件、请求索引、决策、完成记录和审计均位于公共生产Blob；管理员密码、登录限流和会话仍只使用管理员身份Blob。

## API与模式分发

现有URL保持不变：

```text
GET  /api/admin/rollbacks
POST /api/admin/rollbacks/execute
```

`CLOUD_PRODUCTION_ENABLED`决定正式或阶段5E预览处理器。生产总开关为1时，即使回滚子开关为0，也只会由正式处理器返回503，不会回退到预览环境。

## 安全边界

- 只接受issuer为`cloud-collab-admin-production`的正式管理员会话；
- POST要求精确同源；
- 跨站请求在读取正文和创建公共Blob Store前失败关闭；
- 响应不暴露业务键、内容Hash、事件Key、设备ID、提交ID、内部Blob路径、密钥或盐值；
- `rollbackRef`由独立盐值对作用域、业务键、当前版本和上一版本进行HMAC绑定。

## 回滚语义

阶段7Y继续复用唯一的`admin_rollback_v1.js`核心，不复制第二套算法。核心事件校验由传入配置决定作用域：阶段5E预览配置仍硬锁`group_fixture / lib_receive_fixture`，正式配置使用`group_see / lib_see_cz`。

正式回滚只支持普通精确价格，并且：

1. 只列出拥有上一批准值的当前项目；
2. 首次新增项目不能通过回滚变成删除；
3. 必须提交固定确认词`ROLLBACK_TO_PREVIOUS_APPROVED_VALUE`；
4. 通过追加补偿公共事件恢复上一批准值；
5. 不覆盖或删除任何旧事件、审核、决策和审计；
6. 同一请求精确重放只返回已有完成记录；
7. 基线已变化时返回冲突，不覆盖其他批准或回滚。

## 自动化证据

阶段7Y专项测试使用`group_see / lib_see_cz`真实生产协议作用域构造两个已批准版本，验证：

- 候选显示120恢复到100；
- 执行后新增第三个公共事件；
- 前两个事件逐字节保持不变；
- 公共快照恢复为100；
- 决策、请求索引、完成和审计均为不可变记录；
- 重放不创建第四个事件；
- 正式issuer、同源、脱敏和公共Blob绑定；
- 回滚子开关关闭不回退预览。

专项Actions同时继承阶段5E、7X、7W、正式管理员身份与生产运行时回归。通用CI继续执行退出码和Node最终摘要双重门禁，以及完整浏览器矩阵。
