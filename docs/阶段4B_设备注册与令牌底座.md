# 阶段4B：设备注册与令牌底座

## 1. 本阶段范围

本阶段只建立匿名设备的首次注册、私有设备记录、令牌签发与服务端鉴权底座。

当前仍然明确关闭：

- 用户候选提交接口
- 本地待上传队列派发
- 自动批准
- 公共快照修改
- 公共版本增加
- 管理员写入

阶段3B只读同步接口和8.2.27候选页面保持不变。

## 2. 接口

```text
POST /api/device/register
```

默认部署状态返回：

```json
{
  "ok": false,
  "error": {
    "code": "DEVICE_REGISTRATION_DISABLED"
  }
}
```

只有显式配置以下环境变量后才允许注册：

```text
CLOUD_COLLAB_DEVICE_REGISTRATION_ENABLED=true
CLOUD_COLLAB_DEVICE_TOKEN_SECRET=<至少32字节的随机秘密>
```

可选配置：

```text
CLOUD_COLLAB_DEVICE_STORE_NAME=cloud-collab-private
```

真实秘密只能保存于EdgeOne项目环境变量，不得写入GitHub、HTML、普通备份、日志或聊天记录。

## 3. 注册请求

```json
{
  "schemaVersion": 1,
  "deviceId": "dev_01JABCDEF0123456789XYZABCD",
  "nickname": "小雪",
  "clientContext": {
    "appVersion": "8.2.28",
    "protocolVersion": 1
  }
}
```

边界：

- 顶层和`clientContext`严格字段白名单。
- `deviceId`必须是冻结协议中的Crockford ULID。
- 昵称可为空；非空时经NFKC和空格归一化，最多24字符。
- 注册请求最大8KB。
- 不接受`trusted`、`role`、管理员身份、设备令牌或其他客户端自报权限。

## 4. 设备令牌

令牌格式：

```text
dt_v1.<规范化载荷Base64URL>.<HMAC-SHA256签名Base64URL>
```

载荷只包含：

- 令牌结构版本
- deviceId
- tokenVersion
- issuedAt
- expiresAt
- 128位随机nonce

默认有效期30天，代码硬上限90天。

服务端验证必须同时通过：

1. HMAC签名正确。
2. 令牌未过期。
3. 私有设备记录存在且为active。
4. tokenVersion与私有记录一致。
5. 完整令牌SHA-256摘要与私有记录一致。
6. 设备未被封禁。

仅验证签名不足以构成有效鉴权。

## 5. 私有存储

设备记录存入EdgeOne Blob私有命名空间。

采用：

- `onlyIfNew:true`：首次注册条件写，禁止覆盖现有设备。
- 强一致读取：鉴权、封禁和令牌撤销读取最新记录。
- Blob对象只保存令牌SHA-256摘要，不保存令牌原文。

令牌原文只在首次注册成功响应中返回一次。客户端后续应保存至独立`cloudDeviceCredential`，不得进入提交正文、普通备份或公共导出。

## 6. 身份接管防护

无凭据重复注册同一deviceId会返回：

```text
DEVICE_ALREADY_REGISTERED
```

因此攻击者不能仅凭deviceId覆盖设备令牌。

本阶段尚未开放令牌轮换接口。匿名用户丢失令牌时，当前恢复方式是清除该设备身份并生成新的deviceId；不得通过公开注册接口夺回旧deviceId。

## 7. 当前安全门禁

即使设备注册成功，响应仍固定声明：

```text
publicMutationAllowed=false
submissionEnabled=false
autoApprovalEnabled=false
```

设备注册成功不代表可信设备，不赋予审批、发布或管理员权限。
