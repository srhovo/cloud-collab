# 阶段7V：正式普通审核会话验证修复

## 背景

阶段7T正式管理员身份隔离后，正式会话使用独立issuer：

```text
cloud-collab-admin-production
```

但随后合并的正式普通审核处理器仍调用阶段5A预览会话验证器；其专项测试也使用预览会话签发器构造Cookie，因此测试与实现形成一致但错误的假阳性。

## 修复

- 正式审核处理器直接从`production_admin_auth_v1.js`读取正式管理员配置；
- Cookie使用`verifyProductionAdminSessionToken`验证；
- 不再从正式管理员HTTP层转引配置；
- 不再导入或调用预览`verifyAdminSessionToken`；
- 测试使用`createProductionAdminSessionToken`生成真实正式issuer令牌；
- 增加正式令牌无法被预览验证器接受的隔离断言；
- 增加无效令牌在公共Store创建前返回401并清理Cookie的回归；
- 专项Actions加入静态门禁，禁止正式审核重新导入预览验证器。

## 不变边界

- API路径不变；
- 精确价格、陪玩名字和老板资料的审核算法不变；
- 管理员Store与公共Store隔离不变；
- 生产开关全部默认0；
- 不部署EdgeOne；
- 不访问真实Blob；
- 不生成或保存真实密钥；
- 候选8.2.31及其摘要不变；
- 稳定版仍为8.2.25；
- 8.3.0稳定晋升仍未授权。
