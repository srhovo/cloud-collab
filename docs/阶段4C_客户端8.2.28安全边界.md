# 阶段4C：码单器8.2.28客户端安全边界

## 本阶段实现

码单器8.2.28在8.2.27只读同步候选之上增加以下客户端基础能力：

- 沿用阶段2C已冻结的本地 `deviceId` 生成与稳定保存。
- 设备注册协议客户端。
- 明文设备令牌仅写入 `cloudDeviceCredential` 专用本地凭据区；界面与状态只读取脱敏摘要。
- 普通精确价格首次绑定候选生成。
- `pendingCloudChanges` 到期记录派发。
- Bearer设备身份、同正文幂等重试、队列状态回写。
- 网络/超时、429、401、403、409和5xx分类。
- 断网、隐藏页面、上传门禁关闭时安全降级。

## 默认关闭

构建产物固定包含：

```html
<meta name="cloud-collab-write-enabled" content="0">
```

因此当前8.2.28候选不会主动注册设备或上传队列。模块可通过单元测试的注入传输层完成协议验收，但不得把阶段4B.2预览访问密钥写入客户端。当前服务端预览路由仍要求服务器侧预览密钥，普通客户端没有也不应获得该密钥；在出现不依赖客户端秘密的受控提交网关之前，不得把该meta改为1。

## 模式边界

- `local`：不接收，不上传，不生成候选。
- `receive`：只接收公共普通精确价格；即使本地存在旧队列记录，也会在网络请求前阻断。
- `collaborate`：只有该模式允许生成普通精确价格待上传候选。

首次绑定参与协作时，客户端必须先成功完成公共库比较，再逐条投影本地普通精确价格：

- 与公共Hash相同：忽略。
- 存在开放冲突：暂不提交。
- 已存在相同业务键与内容Hash的队列记录：忽略。
- 其他合格项：拆成原子 `exact_price / upsert / initialBinding` 候选。
- 无效项：跳过并计数，不把私人字段带入候选。

## 永不上传

候选生成器只读取和复制：

- `serviceType -> serviceName`
- `settleType`
- `unitPrice`

历史、订单正文、原聊天、备注、自定义比例、最近老板、布局、使用次数、最后使用时间、个人排序、锁定状态、凭据、盐值和管理信息都不会进入网络请求体。

## 队列状态与错误处理

- 成功接收或相同正文幂等重复：`acknowledged`。
- 网络错误、超时、429、5xx：`retry_wait`，使用固定退避；429的 `Retry-After` 可延长等待时间。
- 401：清除失效的专用设备凭据并把当前记录设为 `blocked`。
- 403、409和其他非重试4xx：`blocked`，等待人工处理或后续协议阶段。
- 只接收/本地模式的遗留队列：网络请求前 `blocked`。

所有云端失败只影响协作上传状态，不影响正常码单、查价、价格库本地保存或历史记录。

## 本阶段明确不做

- 不修改或覆盖码单器8.2.25稳定基线。
- 不开启正式公共快照写入。
- 不自动批准候选。
- 不开放管理员审核或管理员写入。
- 不实现两设备自动匹配发布。
- 不上传敏感数据。
- 不让用户自主创建正式公共库。
- 不在客户端内置预览密钥、设备令牌样例、限流盐或完整受保护预览地址。

## 验收顺序

CI先构建并验证8.2.27基线，运行全部单元测试和阶段4B.2静态门禁；随后注入阶段4C模块生成8.2.28单文件候选，再运行8.2.28语法、隐私边界、模式门禁、错误分类和构建清单检查。只有全部通过并完成PR审核后，代码才可进入下一步；真实上传仍保持关闭。
