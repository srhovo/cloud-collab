# 阶段4C：码单器8.2.28设备注册与候选队列派发边界

## 目标

在阶段4B.2服务端隔离预览写入底座之上，为码单器候选客户端增加设备注册、本地凭据管理、`pendingCloudChanges` 派发、幂等重试、错误分类、状态回写和断网降级。所有功能均服从“本地优先、默认拒绝、只写候选、不改正式公共库”。

## 固定边界

1. 构建源仍是冻结的阶段2C候选源；`码单器8.2.25_现.html` 不修改、不覆盖、不作为构建输入。
2. 客户端只允许 `group_fixture / lib_receive_fixture`、`exact_price`、`upsert`。
3. `local` 与 `receive` 模式既不生成候选，也不派发已有候选。
4. 正式公共写入、自动批准、管理员审核始终关闭；客户端会拒绝宣告这些能力的服务端响应。
5. 预览访问凭据不由页面输入，不写入 `localStorage`、`sessionStorage`、队列、日志或普通备份；仅由受控运行时以内存函数提供。
6. 设备令牌只写入既有 `cloudDeviceCredential`，只通过 `Authorization: Bearer` 发送，不显示、不导出。

## 入队来源

- 首次从非协作模式切换到 `collaborate`：先完成只读同步和基线 Hash 更新，再把本地与公共基线不同的普通精确价格逐条规划为候选。
- 用户在价格库界面新增或修改普通精确价格：保存成功后异步投影单条候选。
- 云端拉取、导入、迁移、回滚、系统自动记忆、礼物、区间规则、加价规则及任何私人字段不会触发本阶段上传。

重复保存同一协作绑定不会重新批量规划；队列层继续用冻结的请求体与幂等键约束去重。

## 派发状态机

- `queued -> sending`
- 成功或服务端确认同一幂等请求：`sending -> acknowledged`
- 网络、超时、429、5xx：`sending -> retry_wait`，沿用本地 Store 的退避与抖动
- 凭据、权限、作用域、协议或幂等冲突：`sending -> blocked`
- 手动重试只重排明确可恢复的阻断项

遇到 429、凭据失败或临时服务故障时停止当前批次，避免请求风暴。离线或运行时门禁关闭时不修改队列状态、不注册设备、不发出 POST。

## 错误分类

- `network / transient / rate_limit`：可重试
- `credential / credential_recovery`：清理失效本地凭据并阻断，避免注册循环
- `write_gate / service_disabled`：默认拒绝，不自动循环
- `forbidden / validation / protocol / conflict`：阻断并保留错误码供本地状态展示

## 隐私防护

出站构建器仅包含冻结协议字段和普通精确价格投影：服务名、结算类型、单价及其 Hash。禁止历史记录、订单、聊天、备注、布局、自定义比例、锁定字段、使用次数、最后使用时间、原始文本、老板记忆及令牌进入载荷。

## 测试与回滚

- Node 单元测试覆盖路由、门禁、令牌位置、Hash/幂等、模式隔离、离线、限流、凭据失效、冲突与批次停止。
- 静态验证锁定源文件 SHA、旧 schema、单脚本输出、隐私字段、正式写入关闭、无清理路由和可重复构建。
- Playwright 核心计算对比与浏览器同步回归继续执行。
- 所有改动位于独立分支并通过 PR；回滚只需关闭/撤销该 PR 或回退候选提交，main 与稳定8.2.25不受影响。
