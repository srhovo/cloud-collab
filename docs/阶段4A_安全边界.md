# 阶段4A安全边界

## 明确允许

- 普通精确价格候选协议
- 冻结原子提交信封与严格字段白名单
- 名称、数字和ID标准化
- 服务端重算 `businessKey`、`contentHash`、`idempotencyKey`
- `ik_v1` 设备范围幂等算法
- 16KB请求大小门禁
- 两个不同设备一致、可信设备和冲突的纯资格判断
- 已有公共值发生变化时进入审核资格

## 明确关闭

- EdgeOne POST写入路由
- 设备注册和设备令牌签发
- KV、Blob或数据库写入
- 自动批准
- 公共快照修改
- 公共版本增加
- 管理员审核动作
- 客户端自动上传
- 本地待上传队列派发

## 当前拒绝

- 删除价格
- 区间规则、加价规则、礼物规则
- 陪玩名字
- 老板资料与折数
- 整库上传、覆盖、重命名或重新绑定
- `import`、`migration`、`cloudPull`、`rollback`、`system`来源
- 历史、订单、聊天、备注、布局、自定义比例、使用次数、最近使用时间
- 设备令牌、管理员令牌或Authorization内容
- 非Crockford ULID设备ID和提交ID
- 客户端Hash与服务端重算不一致的请求

## 不可变原则

1. 前端白名单不能替代服务端白名单。
2. 客户端自报的业务键、内容Hash和幂等键都不可信。
3. 每次请求只能描述一个原子业务变更。
4. 两设备一致或可信设备只产生资格，不等于阶段4A允许发布。
5. `publicMutationAllowed` 与 `autoApprovalEnabled` 始终为 `false`。
6. “价格变化±10%”不是冻结协议规则，只能由后续可版本化审核策略决定。
7. 阶段3B只读接口、公共快照和8.2.27页面保持不变。
8. 提交模块异常不得影响本地码单或只接收同步。
