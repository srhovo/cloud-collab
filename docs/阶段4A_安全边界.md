# 阶段4A安全边界

## 明确允许

- 普通精确价格候选协议
- 严格字段白名单
- 名称与数字标准化
- 服务端重算业务键和内容 Hash
- 设备范围幂等键
- 64KB请求大小门禁
- 新增价格的两设备一致、可信设备与冲突资格判断
- 已有公共价格发生变化时，默认进入人工审核

## 明确关闭

- EdgeOne写入路由
- 设备注册和设备令牌签发
- KV、Blob或数据库写入
- 自动批准
- 公共快照修改
- 公共版本增加
- 管理员审核动作
- 客户端自动上传
- 本地待上传队列派发

## 当前拒绝的数据

- 删除价格
- 区间规则
- 加价规则
- 礼物规则
- 陪玩名字
- 老板资料及折数
- 整库上传或覆盖
- 历史、订单、聊天、备注、布局、自定义比例、使用次数、最近使用时间
- 设备令牌、管理员令牌或Authorization内容

## 安全原则

1. 前端白名单不能替代服务端白名单。
2. 客户端提供的业务键和Hash一律不可信。
3. 两设备一致只是候选资格，不等于本阶段允许发布。
4. “价格变化±10%”不得写入不可变协议；后续只能作为独立、可版本化且可回滚的审核策略讨论。
5. 已有公共价格的任何变化在当前默认策略下均进入人工审核，不因设备可信或两设备相同而自动放行。
6. 本阶段模块不得被Edge Functions直接暴露为POST接口。
7. 阶段3B只读接口与8.2.27候选页面不得修改。
8. 云端故障、提交模块故障均不得影响本地码单和只接收同步。
