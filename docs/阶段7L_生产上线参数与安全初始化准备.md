## 二、永久访问入口结论

当前EdgeOne项目使用“全球可用区（含中国大陆）”。在该区域下，平台项目域名和部署域名必须通过系统生成的临时预览链接访问，链接有效期约3小时，不能作为长期公开网址。

GitHub仓库地址只用于查看源码，不是普通用户应用入口。GitHub Pages可以免费托管静态HTML备用入口，但不能承载本项目的EdgeOne Cloud Functions和Blob正式写入，并且中国大陆网络连通性不能作为稳定承诺。

### 可选路径

1. **推荐正式路径**：购买并控制一个域名，完成ICP备案后绑定当前EdgeOne项目，获得中国大陆节点稳定入口。
2. **无备案过渡路径**：购买并控制一个域名，新建“全球可用区（不含中国大陆）”EdgeOne项目并绑定该域名；无需ICP备案，但只使用境外节点，必须额外实测中国大陆网络。
3. **完全免费静态备用**：部署GitHub Pages，只提供本地模式和静态候选下载；公共协作API不能依赖该入口。
4. **暂不购买域名**：继续使用3小时预览链接完成生产后端和L4验收，但不能对外宣布长期正式上线。

当前计划选择第4项作为技术推进方式，同时保留第3项作为可选免费静态备用。正式公共入口仍阻断于“负责人可控制的自定义域名”。

## 三、管理员密钥处理

聊天中出现过的管理员密码已经暴露，且可预测，必须视为失效，禁止配置到任何环境。

仓库只保存：

- 管理员用户名`xiaxue`；
- 密钥变量名；
- 长度、独立性和默认关闭规则；
- 本地生成器。

真实密钥必须：

- 至少32字节随机值；
- 每个用途使用不同值；
- 只保存于密码管理器和EdgeOne私密环境变量；
- 不进入GitHub、Actions日志、Artifact、截图或聊天；
- 不复用任何预览环境凭据。

本地生成命令：

```bash
npm run production:secrets:generate -- --output /安全路径/cloud-collab-production-secrets.env
```

生成器拒绝在CI中运行，拒绝覆盖现有文件，也不支持把密钥打印到stdout。录入EdgeOne后应安全删除本地文件。

## 四、生产环境模板

变量模板位于：

```text
config/production.env.template
```

所有生产能力默认关闭。正式启用顺序固定为：

```text
1. 创建独立生产Blob与管理员安全Blob
2. 一次性空库初始化和完整性核验
3. 只读同步
4. 少量设备普通提交
5. 普通数据自动审核
6. 敏感提交与管理员人工审核
7. 完整L4验收
8. 单独授权晋升8.3.0
```

禁止一次性把所有开关同时改为1。
