# 阶段5E：管理员公共数据回滚范围冻结（默认关闭）

阶段5E基线为`main@ae02c867d1beabbc4d4fab10b19e16274756e0e4`，独立分支为`agent/stage5e-admin-rollback`。

## 范围

阶段5E仅处理合成普通精确价格回滚：管理员通过不可逆`rollbackRef`，把当前生效值恢复到紧邻的上一份已批准值。回滚通过追加新的不可变补偿事件完成，不删除、不覆盖、不改写旧公共事件、审核或审计。

明确不包含首次新增回滚为删除、批量或整库回滚、敏感规则、导出、正式公共库写入、普通用户页面以及码单器8.2.25修改。

## 事件与兼容策略

公共补偿事件继续使用既有schemaVersion 1和`exact_price/upsert`格式，并由既有批准发布器负责版本预留、基线迁移、批准索引和快照重建。服务端私有不可变回滚决策与审计记录`admin_rollback`、当前版本、恢复版本、请求关联及脱敏管理员标签。

该策略保持普通增量接口和旧客户端兼容，回滚专属内部信息不进入普通用户投影。

## 幂等、并发与中断恢复

请求正文严格包含：

```json
{
  "schemaVersion": 1,
  "rollbackRef": "<opaque-ref>",
  "requestId": "<unique-id>",
  "confirmation": "ROLLBACK_TO_PREVIOUS_APPROVED_VALUE"
}
```

服务端先占用不可变请求索引。相同请求并发到达时共享同一请求与决策链，不因服务器时间差异误判冲突；相同requestId绑定不同正文或管理员时失败关闭。历史目标、无上一值、目标不存在和基线占用使用独立稳定错误码。

事件预留竞争产生但未获得批准索引的对象不进入有效公共链。公共事件已发布但私有审计或完成记录中断时，重放同一请求从请求索引、不可变决策、迁移声明和批准索引恢复，不产生第二个有效公共版本。

## 隐私边界

管理员响应不返回设备令牌、tokenHash、deviceId、submissionId、idempotencyKey、businessKey、contentHash、eventKey、snapshotKey、Blob路径、approvalId、requestHash、rollbackId、auditId、管理员秘密或私人业务数据。页面不使用LocalStorage或SessionStorage保存凭据或回滚状态。
