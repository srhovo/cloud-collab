# 阶段5E：管理员公共数据回滚范围冻结（默认关闭）

## 基线

- 正式开发基线：`main@ae02c867d1beabbc4d4fab10b19e16274756e0e4`。
- 该基线已经包含阶段5A管理员登录、阶段5B只读审核、阶段5C审核写入和阶段5D设备治理。
- 阶段5D已经按授权 squash 合并；阶段5D不单独执行EdgeOne验收。
- 本阶段分支：`agent/stage5e-admin-rollback`，必须从上述新main独立推进。

## 本阶段唯一主目标

建立管理员公共数据回滚能力。回滚不是删除或重写旧记录，而是追加新的不可变补偿事件，使公共快照恢复到目标业务键的上一份已批准值。

阶段5E只处理：

1. 管理员查看可回滚的合成普通精确价格事件；
2. 查看当前值、上一已批准值、影响业务键和公共版本；
3. 对“当前仍生效”的事件执行一次明确确认的回滚；
4. 生成新的不可变公共事件、批准索引、迁移声明、请求索引和审计记录；
5. 重新生成强一致可验证的最新公共快照；
6. 同一请求重放返回同一回滚结果；
7. 并发审批或回滚导致基线变化时失败关闭，不覆盖较新的公共值。

## 明确不包含

- 回滚到“项目不存在”或通过回滚删除首次新增项目；
- 普通删除、老板删除或其他阶段6敏感删除能力；
- 区间、加价、礼物、直属变化等敏感规则；
- 批量回滚、整库回滚或跨库回滚；
- 导出、迁移包和平台迁移；
- 正式公共库写入或正式管理员能力启用；
- 修改码单器8.2.25稳定文件或普通用户页面。

首次新增项目没有上一已批准值时，本阶段必须返回稳定错误码 `ADMIN_ROLLBACK_NO_PREVIOUS_VALUE`，不得偷偷转换成删除。

## 默认关闭与作用域

仓库模板新增独立门禁并保持：

```text
CLOUD_ADMIN_ROLLBACK_PREVIEW_ENABLED=0
```

回滚只允许同时满足：

- 阶段5A管理员会话有效；
- 请求通过既有同项目HTTPS与同源校验；
- Blob Store固定为`cloud-collab-preview-v1`；
- groupId固定为`group_fixture`；
- libraryId固定为`lib_receive_fixture`；
- 普通用户写入、自动审核和阶段5C审核写入均保持关闭；
- 只处理合成普通精确价格公共事件。

阶段5D设备治理门禁可以在未来联合验收中同时开启，因为两者写入不同前缀；阶段5E不得要求重做阶段5D独立验收。

## 回滚语义

允许回滚的目标必须同时满足：

1. 目标是有效公共事件；
2. 数据类型为`exact_price`，操作为`upsert`；
3. 目标事件仍是该businessKey当前生效版本；
4. 同一businessKey存在更早的有效已批准事件；
5. 上一事件的payload和contentHash可重新校验；
6. 当前公共快照与目标事件基线一致；
7. 当前基线未被另一个批准或回滚占用。

回滚完成后：

- 原事件、原提交、原审核和原审计全部保留；
- 新事件的公共版本严格递增；
- 新事件payload等于上一已批准值；
- 新事件必须明确记录`admin_rollback`模式、回滚来源版本、恢复目标版本和脱敏管理员标签；
- 公共快照显示恢复后的值；
- 增量接口可把新补偿事件作为普通新版本下发；
- 不允许修改旧事件、旧快照或旧批准索引。

## 事件兼容策略

既有阶段4D/5C事件继续保持schemaVersion 1，不批量迁移。

阶段5E为管理员回滚引入兼容读取的新事件版本，只扩展回滚事件，不改变既有自动批准与管理员审核事件的写入格式。公共事件读取器必须同时验证旧事件与回滚事件；普通用户投影仍只返回版本、业务键、内容Hash、数据类型、操作和payload，不暴露管理员身份、请求Hash、Blob Key或内部索引。

## 幂等与并发

- 客户端提交独立`requestId`；服务端生成确定性requestHash和rollbackId；
- 同一requestId和同一正文重放返回同一结果；
- 同一requestId对应不同正文返回409；
- 同一businessKey同一基线只允许一个迁移声明；
- 若目标事件已不再是当前值，返回`ADMIN_ROLLBACK_TARGET_STALE`；
- 若另一个请求先占用当前基线，返回`ADMIN_ROLLBACK_TRANSITION_CONFLICT`；
- 任何中断恢复都必须从不可变事件、请求索引和批准索引重建结果。

## 隐私与响应投影

管理员页面和接口不得返回：

- 完整deviceId、设备令牌或tokenHash；
- submissionId、idempotencyKey和原请求Hash；
- eventKey、snapshotKey、Blob路径或存储内部错误；
- 管理员密码、Cookie、会话密钥、限流盐值和回滚引用盐值；
- 原始聊天、订单、备注、历史、自定义比例或私人使用信息。

页面使用不可逆`rollbackRef`定位目标。可显示：服务名称、结算方式、当前单价、上一单价、当前公共版本、恢复目标版本和批准时间。
