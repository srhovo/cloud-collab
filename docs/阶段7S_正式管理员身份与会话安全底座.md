# 阶段7S：正式管理员身份与会话安全底座

## 目标

在阶段7R完成正式普通三类提交与自动审核接线后，先建立正式管理员登录、会话验证和退出底座。

本阶段只接入身份层，不接入审核队列、批准、拒绝、编辑后批准、设备治理、回滚、导出或敏感提交；不部署EdgeOne、不访问真实Blob、不生成真实密钥、不晋升稳定版。

## 路由

继续使用阶段5A已经冻结的路径：

```text
POST /api/admin/auth/login
GET  /api/admin/auth/session
POST /api/admin/auth/logout
```

三个Cloud Function路由统一调用`src/server/admin_auth_mode_dispatch_v1.js`。

分发规则：

```text
CLOUD_PRODUCTION_ENABLED=1 → 正式管理员处理器
CLOUD_PRODUCTION_ENABLED=0 → 阶段5A隔离预览处理器
其他值                     → 503 PRODUCTION_FLAG_INVALID
```

生产总开关为1时，即使正式管理员开关为0，也不会回退到预览处理器。

## 正式配置

`readProductionAdminAuthConfig`复用`readProductionRuntimeConfig`，要求：

```text
CLOUD_PRODUCTION_ENABLED=1
CLOUD_ADMIN_PRODUCTION_ENABLED=1
CLOUD_ADMIN_USERNAME=xiaxue
CLOUD_ADMIN_PRODUCTION_BLOB_STORE_NAME=cloud-collab-admin-production-v1
CLOUD_ADMIN_PUBLIC_ORIGIN=<纯HTTPS来源>
```

同时要求下列真实秘密均为32至256字节、彼此不同，并且只存在于EdgeOne私密环境变量：

```text
CLOUD_ADMIN_PASSWORD
CLOUD_ADMIN_SESSION_SECRET
CLOUD_ADMIN_RATE_LIMIT_SALT
```

聊天中曾出现的密码不得使用。

## 会话隔离

正式会话使用独立issuer：

```text
cloud-collab-admin-production
```

隔离预览继续使用阶段5A既有issuer。即使测试中人为复用相同会话密钥，生产令牌和预览令牌也因issuer不同而不能互相验证。

会话保持：

```text
HS256
15分钟有效
HttpOnly
Secure
SameSite=Strict
Path=/api/admin
```

响应不返回令牌正文、密码、会话密钥、限流盐值或真实Blob内容。

## 登录限流

正式管理员登录限流固定写入独立管理员Store，Key前缀为：

```text
admin-production-rate/login/
```

Key只包含用户名和客户端地址的加盐HMAC摘要及时间槽，不包含用户名、IP、盐值或密码明文。

相同用户名与客户端地址在同一10秒槽内只允许一次尝试。限流对象使用`onlyIfNew`和强一致复读。

## 同源与失败关闭

登录和退出必须带精确同源`Origin`；会话查询只接受配置的HTTPS来源。继续复用阶段5A的EdgeOne同项目滚动部署域名识别规则。

正式管理员关闭、生产总开关错误、弱密钥、复用密钥、错误Store、错误用户名或非HTTPS来源时，均在业务处理前失败关闭。

正式管理员关闭时，登录和会话查询返回503；退出接口仍可在精确同源校验后清除旧Cookie，避免失效配置导致浏览器残留会话无法清理。

## 能力边界

本阶段身份响应继续声明：

```text
reviewQueueRead=false
reviewMutation=false
deviceMutation=false
rollback=false
export=false
publicMutationAllowed=false
stablePromotionAuthorized=false
```

阶段7S不赋予任何审核或公共数据修改能力。

## 自动化验证

专项工作流：

```text
.github/workflows/stage7s-production-admin-auth.yml
```

覆盖：

- 生产运行时门禁和固定管理员Store；
- 生产/预览会话issuer隔离；
- 生产登录限流摘要与前缀；
- 登录、会话、退出Cookie闭环；
- 正式管理员关闭时正文解析和Store创建前阻断；
- 生产模式不回退预览；
- 预览阶段5A兼容回归；
- 三个Cloud Function仅调用模式分发器；
- Cloud Function相对导入解析；
- 全量Node和浏览器回归。
