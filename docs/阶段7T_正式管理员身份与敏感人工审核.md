# 阶段7T：正式管理员身份与敏感人工审核

## 目标

在正式敏感候选入口已经进入主线后，完成无需自定义域名和真实EdgeOne资源即可验证的管理员控制面代码：正式登录、短会话、敏感审核队列、详情、批准、拒绝和修改后批准。

阶段7T只完成代码和内存Store验证，不创建真实管理员密码、不部署管理员网址、不访问真实Blob，也不晋升稳定版。

## 两Store隔离

```text
管理员生产Store：cloud-collab-admin-production-v1
公共生产Store：cloud-collab-production-v1
```

管理员Store仅承载登录限流和会话请求限流。公共Store承载候选、审核决策、公共事件、墓碑、审计和公共版本。管理员密码、会话密钥和盐值只存在于EdgeOne私密环境变量。

## 管理员认证路由

```text
POST /api/admin/auth/login
GET  /api/admin/auth/session
POST /api/admin/auth/logout
```

公共路径通过`CLOUD_PRODUCTION_ENABLED`显式分发：

- `1`：正式管理员处理器；
- `0`或未配置：原阶段5A预览处理器；
- 其他值：503失败关闭。

正式认证要求：

- 生产、只读、管理员身份和管理员审核开关均开启；
- 请求URL、Origin、Referer和Sec-Fetch-Site满足同源规则；
- 用户名与密码常量时间比较；
- 登录限流使用管理员Store；
- 会话为15分钟HS256令牌；
- Cookie为`HttpOnly; Secure; SameSite=Strict; Path=/api/admin`；
- 会话不进入LocalStorage、SessionStorage或IndexedDB；
- 响应永远不包含真实密钥。

## 敏感审核路由

```text
GET  /api/admin/sensitive-reviews
GET  /api/admin/sensitive-reviews/detail?id=...
POST /api/admin/sensitive-reviews/approve
POST /api/admin/sensitive-reviews/reject
POST /api/admin/sensitive-reviews/edit-and-approve
```

所有审核路由先验证正式管理员会话，再使用公共生产Store。它们复用阶段6B已经验收的：

- 待审核队列和脱敏详情；
- 候选摘要和公共版本并发检查；
- 精确确认词；
- 批准、拒绝、修改后批准；
- 敏感公共事件和显式删除墓碑；
- 不可变审核与审计；
- 幂等重放与冲突失败关闭。

## 独立管理员页面

源码：

```text
public/admin-production-sensitive-reviews.html
```

页面提供登录、会话检查、队列、详情和三类人工决策。它不保存密码或会话到浏览器存储，使用严格CSP和同源请求。

该页面明确不进入普通用户候选白名单。普通用户入口仍只有：

```text
index.html
build-manifest.json
pages-release.json
```

未来必须单独部署管理员入口，并配置独立管理员HTTPS来源。
