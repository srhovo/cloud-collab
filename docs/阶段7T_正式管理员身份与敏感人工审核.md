# 阶段7T：正式管理员身份与敏感人工审核

## 目标

在正式敏感候选入口已经进入主线后，完成仍可由代码和自动化闭环的管理员控制面：正式登录、短会话、敏感审核队列、详情、批准、拒绝和修改后批准。

本阶段不创建真实管理员密码、不部署管理员网址、不访问真实EdgeOne Blob，也不晋升稳定版。

## 两个生产Store隔离

```text
管理员生产Store：cloud-collab-admin-production-v1
公共生产Store：cloud-collab-production-v1
```

管理员Store只承载登录限流与会话请求限流；公共Store承载候选、审核决策、公共事件、墓碑、审计和公共版本。真实密码、会话密钥与盐值只能进入EdgeOne私密环境变量。

## 管理员认证路由

```text
POST /api/admin/auth/login
GET  /api/admin/auth/session
POST /api/admin/auth/logout
```

公共路径通过`CLOUD_PRODUCTION_ENABLED`显式分发：

- `1`：正式管理员处理器；
- `0`或未配置：原阶段5A预览处理器；
- 其他值：503失败关闭。

正式认证要求：

- 生产、只读、管理员身份与管理员审核开关均开启；
- URL、Origin、Referer和Sec-Fetch-Site满足同源规则；
- 用户名和密码使用常量时间比较；
- 登录限流使用管理员Store；
- 会话为15分钟HS256令牌；
- Cookie为`HttpOnly; Secure; SameSite=Strict; Path=/api/admin`；
- 会话不进入LocalStorage、SessionStorage或IndexedDB；
- 响应不包含真实密钥。

## 敏感审核路由

```text
GET  /api/admin/sensitive-reviews
GET  /api/admin/sensitive-reviews/detail?id=...
POST /api/admin/sensitive-reviews/approve
POST /api/admin/sensitive-reviews/reject
POST /api/admin/sensitive-reviews/edit-and-approve
```

每个审核路由先验证正式管理员会话，再使用公共生产Store。它们复用阶段6B已验收的能力：

- 待审核队列和脱敏详情；
- 候选摘要与公共版本并发检查；
- 精确确认词；
- 批准、拒绝、修改后批准；
- 敏感公共事件与显式删除墓碑；
- 不可变审核和审计；
- 幂等重放与冲突失败关闭。

## 历史预览兼容

现有Cloud Function文件保留阶段5A和6B历史处理器名称，但这些名称只是新模式分发器的别名。默认或预览模式仍进入原预览处理器；只有生产总开关明确为1时才进入正式处理器。

## 独立管理员页面

源码：

```text
public/admin-production-sensitive-reviews.html
```

页面提供登录、会话检查、审核队列、详情和三类人工决策。页面使用严格CSP和同源请求，不保存密码或会话到浏览器本地存储。

该页面不会进入普通用户发布包。普通用户公开入口仍只有：

```text
index.html
build-manifest.json
pages-release.json
```

未来需把管理员页面单独部署到管理员HTTPS来源。
