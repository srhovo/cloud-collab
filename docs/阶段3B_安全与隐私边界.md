# 阶段3B安全与隐私边界

1. 所有服务器路由只接受 GET、HEAD、OPTIONS。
2. 客户端请求使用 `credentials: omit`，不发送 Cookie、Authorization 或请求正文。
3. 请求参数只有公共 `groupId`、`libraryId`、版本和分页上限。
4. 不发送 deviceId、昵称、设备令牌、本地价格、老板、陪玩名字、订单、历史、备注、布局或使用统计。
5. 服务端快照仅包含字段白名单中的公共投影。
6. 客户端不信任服务端 Hash，必须重新计算并核对。
7. 未知字段、未知数据类型、重复业务键、超限快照和不兼容 schema 会阻断整次同步。
8. 只有 `receive` 与 `collaborate` 绑定允许接收；`local` 模式不请求对应公共库。
9. 云端拉取不会产生 submission，也不会监听底层 localStorage 写入。
10. 普通备份继续排除六个云协作 key。
11. 正式初始公共库保持空；合成 fixture 使用独立 group/library，不与真实库混用。
12. 仓库、HTML和接口中没有管理员密码、API Token或设备令牌。
