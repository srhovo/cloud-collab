# 阶段7R：正式陪玩名字与老板资料自动审核

## 目标

在阶段7Q已经完成正式精确价格自动审核的基础上，继续接入两类普通共享数据：

- `playable_name`：已确认陪玩名字；
- `boss_profile`：老板名、直属/派单和普通折数。

本阶段复用阶段5G已经验证的普通类型协议、内容Hash、业务Key、幂等键、候选结构、两设备确认、可信设备批准、冲突标记、公共事件和快照引擎。

## 路由保持不变

```text
POST /api/submissions/create
```

统一正式处理器按`dataType`分发：

```text
exact_price   → 阶段7Q精确价格处理器
playable_name → 阶段5G普通类型处理器
boss_profile  → 阶段5G普通类型处理器
```

客户端不需要新增接口，也不能通过不同数据类型绕过生产限流；三类普通数据共用`submission-create`限流作用域。

## 自动审核关闭

当`CLOUD_PRODUCTION_AUTO_APPROVAL_ENABLED=0`时：

- 设备令牌和协议作用域仍必须通过；
- 候选使用onlyIfNew不可变写入；
- 幂等重放返回已有候选；
- 不创建公共事件；
- 不改变公共版本；
- `publicMutationAllowed=false`。

## 自动审核开启

当`CLOUD_PRODUCTION_AUTO_APPROVAL_ENABLED=1`时：

- 第一台普通设备提交一致候选：等待确认；
- 第二台不同设备提交相同内容：自动批准；
- 可信设备可按既有阶段5G规则批准普通数据；
- 同一业务Key出现冲突内容：进入人工审核标记；
- 老板直属变化、折数增加等敏感变化继续进入人工审核；
- 自动批准使用不可变公共事件和快照，不覆盖历史对象。

## 与阶段7Q兼容

精确价格仍直接调用`acceptProductionExactSubmission`，不会复制或重写阶段7Q实现。HTTP响应继续使用同一语义：

```text
autoApprovalEnabled
publicMutationAllowed
publicMutationApplied
autoApprovalResult
stablePromotionAuthorized=false
```

## 明确边界

- 敏感区间、加价、礼物、删除和老板敏感变化尚未接入正式提交路由；
- 管理员正式控制面尚未部署；
- 所有生产环境变量默认仍为0或空；
- 不创建、读取、写入或删除真实EdgeOne Blob；
- 不生成真实生产密钥；
- 不修改8.2.31候选HTML和冻结摘要；
- 不授权或执行8.3.0稳定晋升。

下一阶段应建立独立管理员正式控制面，然后接入阶段6B敏感候选与人工审核。
