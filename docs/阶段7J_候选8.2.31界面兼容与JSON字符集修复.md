# 阶段7J：候选8.2.31界面兼容与JSON字符集修复

## 一、项目负责人授权

2026-07-21，项目负责人明确授权：

- 候选版本采用`8.2.31`；
- 设备昵称示例由“例如：小雪”改为“例如：下雪”；
- 第一个价格库绑定输入框显示为`club`，示例为`club_id`；
- 内部继续使用`groupId`协议，兼容既有本地绑定、请求、审核和迁移数据；
- `club ID`与`library ID`不支持中文，只允许小写英文字母、数字和下划线；
- 修复公开JSON在Safari中的中文字符集显示；
- 暂不晋升稳定版。

## 二、兼容方案

本阶段只修改显示层和输入校验，不把内部字段`groupId`整体重命名为`clubId`，原因是后者会影响：

- 本地持久化绑定；
- 提交与接收协议；
- 管理员审核作用域；
- 公共快照与事件链；
- 备份、迁移和回滚数据。

因此采用：

```text
界面显示：club
界面示例：club_id
DOM与内部字段：cloudGroupIdInput / groupId
允许字符：^[a-z0-9_]+$
```

既有`group_xxx`继续有效，新建ID可使用`club_xxx`。中文名称未来应作为独立显示名称，而不是机器ID。

## 三、界面修改

候选构建器在最终单HTML中执行受控替换：

- `placeholder="例如：小雪"` → `placeholder="例如：下雪"`；
- `groupId`标签 → `club`；
- `group_xiacijian`示例 → `club_id`；
- 两个ID输入框增加`pattern="[a-z0-9_]+"`；
- 增加可见格式提示；
- 保存绑定前再次执行JavaScript校验，中文、大写字母、连字符或空白均失败关闭；
- 校验代码合并到原有唯一内联脚本，最终仍只有一个`<script>`。

## 四、JSON字符集修复

EdgeOne配置对以下两个公开文件显式设置：

```text
/build-manifest.json
/pages-release.json
Content-Type: application/json; charset=utf-8
```

线上验证器对EdgeOne主入口强制要求UTF-8字符集；缺失时以`PUBLIC_DEPLOYMENT_CONTENT_TYPE_INVALID`失败。
