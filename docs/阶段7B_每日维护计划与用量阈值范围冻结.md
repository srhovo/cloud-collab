# 阶段7B：每日维护计划与用量阈值范围冻结

## 目标

阶段7A已经建立只读完整性核查核心。阶段7B只负责把阶段7A报告转换为确定性的每日维护计划，覆盖总方案中的每日快照、用量监控和迁移导出调度判断。

本阶段不直接执行任何维护写入，不部署定时任务，也不要求人工 EdgeOne 验收。

## 输入

`buildDailyMaintenancePlan` 接收：

- 阶段7A完整性报告；
- 上一次每日快照完成标记；
- 上一次迁移导出完成标记；
- 可选的项目维护阈值；
- 当前服务器时间。

维护标记只包含公共版本和完成时间，不包含 Blob Key、业务正文、设备或提交身份。

## 输出

计划固定输出：

- 当前公共版本；
- 每日快照是否到期及原因；
- 迁移导出是否到期及原因；
- 候选库存健康、警告或严重等级；
- 需要执行的维护动作列表；
- 下一次建议运行时间；
- `readOnly=true`；
- `mutationsPerformed=0`。

## 默认策略

- 每日快照间隔：24小时；
- 迁移导出间隔：24小时；
- 候选库存警告：100个；
- 候选库存严重：500个；
- 最老候选警告：24小时；
- 最老候选严重：72小时。

这些是项目维护策略，不代表 EdgeOne 套餐或平台配额。后续正式用量上限必须根据实际平台账户和运行数据单独配置，不能在客户端或公共源码中写入私人账户信息。

## 到期规则

每日快照或迁移导出符合任一条件即到期：

1. 从未生成完成标记；
2. 标记公共版本落后于当前公共版本；
3. 相同公共版本但距离上次完成已达到维护间隔。

候选库存符合任一条件即要求关注：

- 数量达到警告或严重阈值；
- 最老候选达到年龄阈值；
- 存在无效 `receivedAt`，该情况直接视为严重。

## 失败关闭

以下情况不生成计划：

- 阶段7A报告不是只读报告；
- 阶段7A任一完整性检查未通过；
- 报告时间或维护标记来自未来；
- 快照版本与公共版本不一致；
- 完成标记版本领先于当前公共版本；
- 候选库存统计自相矛盾；
- 警告阈值不低于严重阈值。

## 明确排除

- 不写入每日快照；
- 不生成实际 ZIP 导出包；
- 不删除过期候选；
- 不重建索引；
- 不发送通知；
- 不创建 GitHub Actions 定时任务；
- 不新增环境变量、HTTP 路由或管理员页面；
- 不启用正式公共写入、自动批准或管理员发布；
- 不修改普通用户 HTML 和稳定版 `码单器8.2.25`。

实际执行器、幂等维护写入、保留期清理和通知属于后续独立阶段，必须继续默认关闭并单独审核。
