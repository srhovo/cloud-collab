# 阶段7R：正式普通陪玩名字与老板资料接线

## 目标

在阶段7Q正式普通精确价格自动审核已经接线的基础上，把阶段5G已经完成并验证的`playable_name`与`boss_profile`协议接入同一个正式提交端点。

本阶段不部署EdgeOne、不访问真实Blob、不生成真实密钥、不接入管理员正式控制面、不接入阶段6敏感类型、不晋升稳定版。

## 统一端点

```text
POST /api/submissions/create
```

按正文`dataType`严格分发：

```text
exact_price   → 阶段7Q精确价格接收与自动审核
playable_name → 阶段5G普通类型接收与统一公共事件引擎
boss_profile  → 阶段5G普通类型接收与统一公共事件引擎
其他类型      → 400 UNSUPPORTED_PRODUCTION_ORDINARY_DATA_TYPE
```

设备注册响应与提交响应会公开冻结的支持类型列表，但不会暴露密钥、内部Blob对象或管理员信息。

## 安全顺序

每个合法普通提交依次执行：

1. 严格协议与字段白名单校验；
2. 链接、邮箱、电话、微信号等联系方式拦截；
3. `group_see / lib_see_cz`生产作用域校验；
4. 设备令牌认证并核对正文`deviceId`；
5. 新请求消耗生产限流槽，精确幂等重放绕过新限流槽；
6. 候选以`onlyIfNew`不可变写入；
7. 自动审核开启时强一致读回候选；
8. 使用既有确认标记、审核标记、公共事件、批准索引和快照协议作出决定。

验证失败发生在尽可能早的位置。联系方式或作用域不合法时，不调用设备认证、不写限流槽、不写候选。

## 普通陪玩名字

新名字满足以下任一条件可自动批准：

- 一台可信设备提交；
- 两台不同设备提交完全相同的规范化内容。

第一台普通设备提交时等待第二台确认。候选冲突或与既有公共记录冲突时进入人工审核，不自动覆盖。

## 老板资料

新老板资料满足可信设备或双设备相同确认时可自动批准。

已有老板资料发生以下变化时必须进入人工审核：

- 老板名变化；
- 直属/派单变化；
- 折数升高；
- 折数一次下降超过0.05；
- 同一公共基线出现冲突候选。

在安全范围内的小幅降折仍要求两个不同设备确认，不能仅凭一台可信设备自动更新既有记录。

## 响应语义

```text
supportedDataTypes=[exact_price, playable_name, boss_profile]
dataType=exact_price|playable_name|boss_profile
autoApprovalEnabled=true|false
publicMutationAllowed=true|false
publicMutationApplied=true|false
autoApprovalResult.status=waiting_confirmation|pending_review|auto_approved
stablePromotionAuthorized=false
```

`publicMutationAllowed`表示运行时能力已开启，`publicMutationApplied`只表示本次请求实际创建了新的公共事件。精确重放、公共值相同或转人工审核时，后者为`false`。

## 复用范围

正式链直接复用：

- `ordinary_types_policy_v1.js`；
- `ordinary_submission_acceptance_v1.js`；
- `ordinary_public_engine_v1.js`；
- 阶段7Q精确价格链；
- 现有设备身份、Blob仓储、幂等键和生产限流。

没有新增普通类型专用密码、盐值、Store或公开路由。

## 明确边界

本阶段仍不实现：

- 管理员正式登录、审核队列、批准、拒绝、编辑后批准；
- 区间规则、加价规则、礼物规则、删除墓碑等阶段6敏感提交；
- EdgeOne环境变量写入和真实生产开关开启；
- 真实Blob初始化、读取、写入或删除；
- 候选HTML和8.2.31摘要修改；
- 8.3.0稳定版晋升。

## 自动化验证

专项测试覆盖：

- 不支持类型在读取配置和Blob前失败；
- 陪玩名字首次等待、双设备相同确认和唯一公共事件；
- 可信设备批准新老板资料；
- 直属变化、折数升高和异常大幅降折转人工审核；
- 自动审核关闭时只入队；
- 联系方式和越界作用域在认证及限流写入前阻断；
- HTTP统一分发和支持类型投影；
- 阶段7P、7Q、5G和生产运行时回归。

所有专项与完整CI只使用内存Store。真实EdgeOne部署、环境变量和Blob操作均为0。
