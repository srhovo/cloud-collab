# 阶段8B：正式管理员统一控制台

## 目标

把已经分别通过自动测试的正式管理员能力收口为一个独立单HTML控制面：

```text
登录、会话检查与退出
精确价格审核
陪玩名字与老板普通资料审核
敏感变更与删除墓碑审核
设备可信、撤销可信、封禁与解封
公共数据回滚
完整公共数据库迁移导出
```

本阶段只建立页面、独立产物、自动测试和部署边界，不创建真实管理员站点，不填写真实密钥，不访问真实Blob，不启用任何生产能力。

## 页面与产物

源文件：

```text
dist/admin-production-console.html
```

独立构建命令：

```bash
npm run edgeone:admin:build
```

独立输出目录：

```text
.edgeone-admin-artifact/
├── index.html
└── admin-release.json
```

普通用户候选产物继续保持：

```text
.edgeone-artifact/
├── index.html
├── build-manifest.json
└── pages-release.json
```

两套产物互相排斥：

- 普通用户产物不包含管理员HTML或`admin-release.json`；
- 管理员产物不包含码单器候选主页、构建清单或普通发布清单；
- 管理员页面不会进入GitHub Pages静态备用入口；
- 管理员产物必须部署到受控的独立管理员HTTPS来源。

## 身份与会话

控制台复用现有API：

```text
POST /api/admin/auth/login
GET  /api/admin/auth/session
POST /api/admin/auth/logout
```

安全边界：

- 密码只保留在一次登录请求的局部变量中；
- 请求发出前立即清空密码输入框；
- 会话只使用服务端设置的HttpOnly、Secure、SameSite=Strict Cookie；
- 页面不调用LocalStorage、SessionStorage或IndexedDB；
- 页面隐藏、退出或会话401时清空所有队列、详情、不可逆引用、回滚候选和导出摘要；
- 不传播EdgeOne临时预览链接中的`eo_token`或`eo_time`；
- 所有请求同源、`credentials=same-origin`、`cache=no-store`、`redirect=error`、`referrerPolicy=no-referrer`。

## 模块API

### 精确价格审核

```text
GET  /api/admin/reviews
POST /api/admin/reviews/approve
POST /api/admin/reviews/reject
POST /api/admin/reviews/edit-and-approve
```

确认词：

```text
APPROVE
REJECT
EDIT_AND_APPROVE
```

### 普通资料审核

```text
GET  /api/admin/ordinary-reviews
POST /api/admin/ordinary-reviews/approve
POST /api/admin/ordinary-reviews/reject
POST /api/admin/ordinary-reviews/edit-and-approve
```

确认词：

```text
APPROVE_ORDINARY
REJECT_ORDINARY
EDIT_AND_APPROVE_ORDINARY
```

老板身份、直属、升折和异常降折仍由服务端强制转入敏感审核，页面不能绕过。

### 敏感人工审核

```text
GET  /api/admin/sensitive-reviews
GET  /api/admin/sensitive-reviews/detail?id=...
POST /api/admin/sensitive-reviews/approve
POST /api/admin/sensitive-reviews/reject
POST /api/admin/sensitive-reviews/edit-and-approve
```

确认词：

```text
APPROVE_SENSITIVE
REJECT_SENSITIVE
EDIT_AND_APPROVE_SENSITIVE
```

删除批准发布不可变墓碑，不物理删除历史事件。敏感候选不能由可信设备或双设备一致自动批准。

### 设备治理

```text
GET  /api/admin/devices
GET  /api/admin/devices/detail?id=...
POST /api/admin/devices/trust
POST /api/admin/devices/revoke-trust
POST /api/admin/devices/block
POST /api/admin/devices/unblock
```

页面只显示不可逆`deviceRef`。封禁立即撤销可信并令设备认证返回`DEVICE_BLOCKED`；解封不恢复可信。

### 公共数据回滚

```text
GET  /api/admin/rollbacks
POST /api/admin/rollbacks/execute
```

固定确认词：

```text
ROLLBACK_TO_PREVIOUS_APPROVED_VALUE
```

回滚追加补偿事件，只恢复紧邻的上一批准值，不覆盖或删除历史。

### 完整迁移导出

```text
GET  /api/admin/exports/summary
POST /api/admin/exports/download
```

固定确认词：

```text
EXPORT_FULL_PUBLIC_DATABASE
```

导出ZIP不会被保存为公开地址，不修改公共版本；页面在下载后立即撤销临时Object URL。

## 正式投影检查

每个业务模块都要求服务端响应同时满足：

```text
capabilities.productionAdmin=true
capabilities.syntheticFixtureOnly=false
stablePromotionAuthorized=false
```

任一不满足，页面拒绝把响应当作正式管理员结果。

## CSP与外部资源

页面固定：

```text
default-src 'none'
connect-src 'self'
style-src 'unsafe-inline'
script-src 'unsafe-inline'
base-uri 'none'
form-action 'none'
object-src 'none'
frame-ancestors 'none'
```

页面只有一个内联样式和一个内联脚本，不加载第三方脚本、字体、图片、样式或框架。

## 自动测试

### Node静态与产物测试

`tests/stage8b-production-admin-console.test.mjs`验证：

- CSP、单脚本、单样式和无外部资源；
- 不调用浏览器持久化存储；
- 不包含临时访问令牌传播或预览页面路径；
- 全部正式API路径和冻结确认词；
- 脚本语法；
- 401、退出和页面隐藏的数据清理锚点；
- 正式投影与稳定晋升未授权；
- 普通用户和管理员两套产物双向隔离；
- 管理员发布清单的Hash、字节数和版本边界。

### Chromium浏览器回归

`tests/stage8b_browser_production_admin_console.py`使用同源API拦截验证：

- 初始无会话；
- 登录与密码输入清空；
- 精确价格批准；
- 普通资料批准；
- 敏感详情与批准；
- 设备详情与设为可信；
- 回滚固定确认词；
- 完整ZIP导出固定确认词和下载；
- 服务端会话失效后全部业务视图清空；
- 浏览器本地存储为空；
- 控制台无JavaScript错误。

### 继承回归

专项Actions同时运行正式管理员身份、普通审核、敏感审核、设备治理、回滚和完整导出服务端测试，并重新构建普通用户与管理员两套产物进行互斥检查。通用CI继续执行完整Node和阶段5A至6B浏览器矩阵。
