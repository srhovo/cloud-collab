# 阶段7S：正式管理员登录与会话

## 目标

在不创建真实Blob、不填写真实密钥、不部署管理员页面和不晋升稳定版的前提下，将阶段5A已验证的管理员登录、会话和退出能力接入正式生产运行时。

## 入口保持不变

```text
POST /api/admin/auth/login
GET  /api/admin/auth/session
POST /api/admin/auth/logout
```

Cloud Function入口只依赖`admin_auth_mode_dispatch_v1.js`：

```text
CLOUD_ADMIN_PRODUCTION_ENABLED=0或未配置 → 阶段5A隔离预览处理器
CLOUD_ADMIN_PRODUCTION_ENABLED=1          → 阶段7S正式管理员处理器
其他值                                    → 503失败关闭
```

## 正式配置

正式处理器通过`readProductionRuntimeConfig`读取并验证：

```text
管理员用户名：xiaxue
管理员Store：cloud-collab-admin-production-v1
管理员来源：CLOUD_ADMIN_PUBLIC_ORIGIN中的纯HTTPS来源
管理员密码：至少32字节随机值
会话密钥：至少32字节随机值
登录限流盐值：至少32字节随机值
```

上述密钥必须彼此不同，也不得复用公共客户端密钥、公共限流盐值、设备引用盐值、回滚盐值或导出审计盐值。

## 安全边界

- 错误或缺失生产配置在初始化Blob前返回503；
- 登录POST必须携带与管理员来源完全一致的`Origin`；
- 跨站请求在读取正文和创建Store前返回403；
- 登录每个用户名与客户端地址组合每10秒一个限流槽；
- 密码使用定长摘要与`timingSafeEqual`比较；
- 会话Cookie为`HttpOnly; Secure; SameSite=Strict`；
- 会话有效期15分钟；
- 会话端点只返回脱敏身份与能力，不返回密钥或完整会话ID；
- 退出只清除Cookie；
- 管理员Store与公共生产Store严格隔离；
- 普通用户三文件白名单不包含管理员页面；
- `stablePromotionAuthorized=false`。

## 兼容策略

阶段5A预览处理器、预览变量和浏览器回归保持不变。阶段7S只在显式生产管理员开关为1时进入正式处理器，因此现有候选部署和本地模式不会突然启用管理员能力。
