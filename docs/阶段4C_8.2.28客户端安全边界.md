# 阶段4C：码单器8.2.28客户端安全边界

## 唯一目标

在8.2.27只接收同步基础上，增加设备注册、本地设备凭据、`pendingCloudChanges`候选派发、幂等重试、错误分类、状态回写和断网降级。

## 强制门禁

- 客户端只允许 `group_fixture / lib_receive_fixture` 的普通精确价格 `upsert`。
- 只有本地绑定模式为 `collaborate` 时才允许派发；`local` 与 `receive` 永不发送POST。
- 页面不接收、显示或持久化预览密钥。部署环境未提供受控运行时门禁时，派发保持关闭。
- 设备令牌只保存在既有 `cloudDeviceCredential` 专用本地Store，仅通过 `Authorization: Bearer` 发送，不进入候选正文、日志或普通备份。
- 服务端返回 `publicMutationAllowed=true` 或 `autoApprovalEnabled=true` 时，客户端立即拒绝响应。
- `PREVIEW_WRITE_DISABLED` 等禁写状态为非自动重试错误，避免关闭门禁时形成重试循环。

## 隐私边界

候选正文只包含冻结协议允许的普通精确价格投影，不包含历史记录、订单、原始聊天、备注、比例、布局、使用次数、最近老板、老板记忆、原始文本或任何密钥。

## 清理工具状态

一次性清理工具仅在本地完成严格门禁实现和测试，未进入本分支。由于当前执行环境没有EdgeOne Blob管理权限、也不接收用户私密密钥，未声称已执行线上清理。正式执行时仍必须满足：预览写入关闭、固定命名空间、白名单对象、强一致预检、摘要确认、删除后强一致复核，并在完成后删除工具。

## 回滚

本分支不修改8.2.25稳定基线。回滚只需关闭或撤销本PR；运行时门禁默认关闭，断网、禁写或未配置时队列保持本地，正常码单与只接收同步不受影响。
