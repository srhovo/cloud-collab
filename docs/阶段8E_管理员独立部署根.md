# 阶段8E：管理员独立部署根

## 目标

阶段8D已完成管理员四文件构建与普通/管理员双产物互斥验证。阶段8E补齐未来独立管理员项目需要的同源Cloud Functions部署根，使控制台页面与`/api/admin/*`位于同一来源。

本阶段只提交配置、构建器、自动化和文档，不创建平台项目、不绑定域名、不填写环境变量、不部署页面。

## 项目Root Directory

未来独立管理员项目使用：

```text
deploy/admin
```

项目配置：

```text
deploy/admin/edgeone.json
```

固定设置：

```text
安装：复制仓库根package.json与package-lock.json后执行npm ci --ignore-scripts
构建：运行prepare-admin-deployment-root-v1.mjs
输出：./.edgeone-admin-artifact
Node构建版本：22.11.0
```

Root Directory只影响管理员项目；仓库根`edgeone.json`和普通用户三文件项目保持不变。

## 静态输出

```text
index.html
production-console.css
production-console.js
admin-release.json
```

部署脚本由阶段8C权威源生成前三个文件，并把初始会话检查从静默模式确定性改为可见模式。页面打开时，如果生产管理员能力仍关闭，会明确显示失败关闭状态，而不是永久停留在“正在检查”。

`admin-release.json`记录：

- 40位源提交；
- 控制台标题；
- 管理员项目根和配置路径；
- 三个静态文件的SHA-256、字节数和UTF-8内容类型；
- 管理员Cloud Functions与服务器模块统计；
- API范围仅为`/api/admin/*`；
- 匿名公开API未包含；
- 真实秘密未包含；
- 平台响应头必须配置；
- 生产能力默认关闭；
- 冻结候选8.2.31及其既有SHA；
- 稳定版8.2.25；
- 稳定晋升和生产写入启用均为false；
- 部署状态仍为代码完成、未部署。

## Cloud Functions范围

构建时只复制：

```text
cloud-functions/_shared
cloud-functions/api/admin
src/server
```

不会复制：

```text
/api/device/register
/api/submissions/create
/api/sensitive-submissions/create
/api/public/*
```

构建器递归检查全部相对导入必须在管理员项目根内解析，并限制生成的JavaScript源代码总量。专项运行时审计还会逐个导入所有管理员Cloud Function，要求每个模块具有默认处理器。

## 平台响应头

配置对四个静态文件显式设置UTF-8内容类型，并对全部路径设置：

```text
Cache-Control: no-store
严格CSP（含frame-ancestors 'none'）
HSTS
nosniff
DENY
no-referrer
COOP/CORP same-origin
受限Permissions-Policy
```

HTML元信息不是平台响应头的替代品；未来上线后仍需线上验证器读取实际响应头。

## 自动化

专项CI会：

1. 从最新源码生成管理员部署根；
2. 逐个导入管理员Cloud Functions；
3. 运行部署根、配置、阶段8C控制台和阶段8D隔离测试；
4. 在固定测试来源加载生成页面，验证生产能力关闭时显示明确错误；
5. 检查浏览器持久化存储为空；
6. 审计工作流只读、无部署命令、无真实网络目标；
7. 检查生成目录全部被忽略，仓库工作树保持干净；
8. 只上传清单、配置和测试日志，不上传或部署真实项目。

## 将来最少人工步骤

只有在另行授权管理员静态上线后，负责人需要：

1. 创建与普通用户项目分离的管理员项目；
2. 选择Root Directory为`deploy/admin`；
3. 核对安装、构建、输出和Node设置；
4. 选择合适的Cloud Functions区域；
5. 核对平台实际响应头；
6. 所有生产开关和私密变量先保持未启用；
7. 完成静态部署验证后，再按单独方案逐级录入私密变量、初始化资源和开启能力。

临时预览链接不能作为长期管理员来源。
