# 阶段4B.2：隔离预览写路由

## 1. 本阶段目标

在阶段4B服务层通过完整CI后，为真实EdgeOne预览环境增加两条受保护的Node.js Cloud Functions路由：

```text
POST /api/device/register
POST /api/submissions/create
```

本阶段只验证设备注册、Bearer鉴权、不可变候选持久化和幂等语义，不连接8.2.27用户页面，不开放正式公共库写入。

## 2. 为什么使用Cloud Functions

阶段4A和阶段4B服务模块依赖`node:crypto`。因此写路由位于：

```text
cloud-functions/api/
```

而不是V8 Edge Functions。阶段3B已有只读接口继续保留在`edge-functions/api/`，不做迁移。

## 3. 默认关闭

仓库样例配置固定为：

```text
CLOUD_COLLAB_WRITE_PREVIEW_ENABLED=false
```

未显式启用时，两条POST路由均返回：

```text
WRITE_PREVIEW_DISABLED
```

启用前必须在EdgeOne项目环境变量中配置：

```text
CLOUD_COLLAB_WRITE_PREVIEW_ENABLED=true
CLOUD_COLLAB_WRITE_PREVIEW_KEY=<至少32字节随机值>
CLOUD_COLLAB_PRIVATE_BLOB_STORE=cloud-collab-preview-private
```

环境变量修改只应用于之后的新部署，因此配置完成后必须重新部署。

真实预览密钥不得写入GitHub、HTML、普通备份、截图、日志或聊天消息。

## 4. 双重访问凭据

两条路由都必须携带：

```text
X-Cloud-Collab-Preview-Key: <隔离预览密钥>
```

候选提交还必须携带设备注册后获得的：

```text
Authorization: Bearer <deviceToken>
```

预览密钥用于隔离测试入口，设备令牌用于识别具体匿名设备。二者不能互相替代。

## 5. Blob隔离

预览写入使用独立Blob命名空间：

```text
cloud-collab-preview-private
```

首次调用SDK时由EdgeOne自动创建。所有状态读取采用强一致模式，设备档案、令牌索引和候选记录继续使用`onlyIfNew:true`。

该命名空间不得与未来正式公共数据命名空间共用。

## 6. 设备注册路由

请求上限为8KB，Content-Type必须为`application/json`。

成功时：

- 返回一次性明文设备令牌；
- Blob仅保存令牌Hash；
- 同一deviceId重复注册返回409；
- 不授予可信设备、管理员或发布权限；
- 固定返回`submissionEnabled:false`；
- 固定返回`publicMutationAllowed:false`；
- 固定返回`autoApprovalEnabled:false`。

## 7. 候选提交路由

请求上限沿用阶段4A冻结协议的16KB。

本阶段硬限制为：

```text
groupId=group_fixture
libraryId=lib_receive_fixture
```

任何其他团或价格库均返回：

```text
PREVIEW_SCOPE_REQUIRED
```

服务端继续执行：

1. Bearer设备鉴权；
2. 阶段4A完整原子提交校验；
3. Authorization设备与正文deviceId一致性检查；
4. 服务端重算Hash；
5. Blob强一致幂等读取；
6. `onlyIfNew`不可变候选写入；
7. 同正文重放返回第一次结果；
8. 同幂等键不同正文返回409。

候选状态仅为`waiting_confirmation`，不会写入公共快照、公共版本或批准事件。

## 8. 当前仍关闭

- 8.2.28用户页面身份注册；
- 本地待上传队列派发；
- 普通用户无预览密钥访问；
- 正式团和正式公共库提交；
- KV限流与异常计数；
- 两设备匹配；
- 可信设备自动资格；
- 自动批准；
- 公共事件发布；
- 公共版本增加；
- 管理员审核与回滚。

## 9. 验收顺序

1. GitHub CI完整通过。
2. 合并到main并等待EdgeOne部署。
3. 不配置环境变量，确认两条POST路由返回503和`WRITE_PREVIEW_DISABLED`。
4. 在EdgeOne配置随机预览密钥并重新部署。
5. 仅使用合成设备和合成测试库完成注册、首次提交、幂等重放与冲突验收。
6. 验收完成后把总开关恢复为false并重新部署。

在第5步完成前，不得把写路由接入用户端。
