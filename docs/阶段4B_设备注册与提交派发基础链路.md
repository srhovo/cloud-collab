# 阶段4B：设备注册、令牌与提交派发基础链路

## 目标

在不修改公共快照、不自动批准、不开放管理员写入的前提下，建立：

- 匿名设备注册
- 一次性随机设备令牌
- Blob强一致读取与 `onlyIfNew` 不可变写入
- 普通精确价格候选持久化
- 单设备分钟/小时限流
- 客户端待上传队列派发器
- 网络失败重试与永久错误阻断

## 单一权威实现

阶段4B路由只复用已经通过CI的服务层：

```text
src/server/blob_repository_v1.js
src/server/device_registration_v1.js
src/server/submission_acceptance_v1.js
src/server/submission_policy_v1.js
```

路由层不得复制第二套令牌、Blob或候选逻辑。新增的 `submission_rate_limit_v1.js` 只负责持久化限流标记。

## 路由

```text
POST /api/v1/device-register
POST /api/v1/submissions
```

两个路由位于 `cloud-functions/`，使用Node.js运行时。没有显式环境开关时返回503；Blob SDK或命名空间不可用时同样失败关闭。

## 环境变量

```text
CLOUD_COLLAB_DEVICE_REGISTRATION_ENABLED=false
CLOUD_COLLAB_SUBMISSION_INTAKE_ENABLED=false
CLOUD_COLLAB_BLOB_STORE=cloud-collab-v1
CLOUD_COLLAB_DEVICE_TOKEN_TTL_MS=7776000000
CLOUD_COLLAB_SUBMISSION_MINUTE_LIMIT=20
CLOUD_COLLAB_SUBMISSION_HOUR_LIMIT=200
```

阶段4B不使用仓库密钥。设备令牌由服务端生成32字节随机值，明文只在首次注册成功响应中返回一次。

## 设备令牌

客户端收到：

```text
dt_v1_<32字节随机值的Base64URL>
```

服务器持久化：

```text
dth_v1_<设备令牌SHA-256 Base64URL>
```

令牌只允许出现在 `Authorization: Bearer ...` 请求头和本地 `cloudDeviceCredential` 中；不得进入提交正文、待上传队列、错误详情、日志、普通备份或公共导出。

重复注册同一 `deviceId` 返回409，不静默签发第二枚令牌。令牌轮换与找回必须单独设计。

## 不可变幂等

1. 服务端按阶段4A冻结协议重新校验提交并重算三个Hash。
2. 候选路径由 `libraryId + idempotencyKey` 决定。
3. Blob使用 `onlyIfNew:true` 保存候选。
4. 同一幂等键、相同规范正文返回第一次结果。
5. 同一幂等键、不同规范正文返回409 `IDEMPOTENCY_CONFLICT`。
6. 候选保持 `publicMutationAllowed:false` 和 `autoApprovalEnabled:false`。

## 限流

提交路由在鉴权和冻结协议校验后，为设备写入分钟、小时两个Blob标记。相同幂等键重复请求复用同一个标记，不重复计数。

默认：

```text
每分钟20条
每小时200条
```

超过限制返回429 `RATE_LIMITED`，客户端进入 `retry_wait`。限流存储不可安全读写时返回503，不继续保存候选。

## 候选状态

当前只产生：

```text
waiting_confirmation
```

两设备匹配、可信设备资格、审核与公共版本发布在后续阶段接入。本阶段不会改写 `catalog.js`、公共快照、增量事件或 `publicVersion`。

## 客户端派发器

`src/cloud_collab_submission_client.js` 提供独立、尚未嵌入8.2.27页面的派发模块：

- 凭据缺失时惰性注册
- 令牌只放Authorization头
- `queued/retry_wait → sending → acknowledged`
- 网络、超时、429和5xx进入 `retry_wait`
- 400、401、403及幂等正文冲突等永久错误进入 `blocked`
- 页面启动和正常码单不等待派发结果

本阶段只验证接口与派发底座。8.2.28候选页面的用户语义入队钩子和自动派发开关在下一子阶段接入。

## 安全结论

- 两个写入开关默认均为false。
- 开关或Blob任一缺失都不会假成功。
- 不上传历史、订单、聊天、备注、金额、布局、自定义比例、使用统计或本地时间。
- 不监听底层localStorage写入。
- 不把令牌放入UI状态、提交正文或队列。
- 阶段3B只接收同步与8.2.25稳定基线保持不变。
