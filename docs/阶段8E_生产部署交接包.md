# 阶段8E：生产部署交接包与安全Blob初始化入口

## 目标

阶段8D已经建立独立管理员四文件产物和普通/管理员双产物互斥门禁。阶段8E复用该权威产物，把未来需要真实平台权限的操作固化为可重复构建、机器校验和默认零写入的交接包。

本阶段不部署EdgeOne、不填写真实环境变量、不访问真实Blob、不生成真实密钥、不修改8.2.31候选、不晋升8.3.0。

管理员四文件通过阶段8D的`prepareAdminConsole`重新构建后复制，摘要和源提交保持可核验。交接包根目录不包含普通用户`index.html`、`build-manifest.json`或`pages-release.json`。

## Blob正确操作方式

EdgeOne Blob命名空间不需要在控制台手工创建。Makers Function或外部SDK首次调用`getStore`时自动创建命名空间；控制台Blob页面只用于只读查看命名空间和对象目录。

阶段8E提供外部一次性初始化器：

```bash
npm run production:bootstrap:edgeone:plan
```

默认只输出计划，真实读写均为0。真实执行必须同时满足：

```text
--execute
EDGEONE_PROJECT_ID
EDGEONE_API_TOKEN
CLOUD_PRODUCTION_BOOTSTRAP_CONFIRMATION=INITIALIZE-see-see_cz-V1
```

初始化器使用强一致读取、`onlyIfNew`不可变写入和写后强一致复核。目标对象存在且完全一致时安全重放；不同内容时失败关闭。

## 手动GitHub Actions

工作流：

```text
stage8e-edgeone-production-bootstrap
```

只允许`workflow_dispatch`，不会因push或pull request自动执行。

### plan

默认操作：

- 不读取EdgeOne Secrets；
- 不连接Blob；
- 生成生产交接包；
- 生成10个初始化资源的零写入计划；
- 上传脱敏证据包。

### execute

只有选择`execute`并输入精确确认词时才进入真实写入任务。还必须提前配置：

```text
EDGEONE_PROJECT_ID
EDGEONE_API_TOKEN
```

Token不得写入Issue、PR、Actions输入、聊天或仓库文件。

## 负责人未来操作路径

### 1. 获取EdgeOne项目ID和API Token

在EdgeOne Makers控制台打开当前项目并记下项目ID，再创建API Token。API Token具有账户级操作能力，只保存到密码管理器或GitHub Secrets。

### 2. 写入GitHub仓库Secrets

```text
GitHub仓库
→ Settings
→ Secrets and variables
→ Actions
→ New repository secret
```

分别创建`EDGEONE_PROJECT_ID`和`EDGEONE_API_TOKEN`，不要把值发送到聊天。

### 3. 运行零写入计划

```text
GitHub仓库
→ Actions
→ stage8e-edgeone-production-bootstrap
→ Run workflow
→ operation: plan
```

预期：

```text
operation=plan
status=ready_not_executed
resourceCount=10
realBlobReadsPerformed=0
realBlobWritesPerformed=0
realBlobDeletesPerformed=0
```

### 4. 一次性初始化

确认计划后：

```text
GitHub仓库
→ Actions
→ stage8e-edgeone-production-bootstrap
→ Run workflow
→ operation: execute
→ confirmation: INITIALIZE-see-see_cz-V1
```

首次预期`status=initialized`、`createdCount=10`；精确重放预期`status=already_initialized_exact`、`createdCount=0`、`existingExactCount=10`。两种结果都必须保持`productionCapabilitiesEnabled=false`和`stablePromotionAuthorized=false`。

### 5. EdgeOne只读核对Blob

```text
EdgeOne Makers
→ 当前项目
→ Blob存储
```

应看到`cloud-collab-production-v1`和`cloud-collab-admin-production-v1`。控制台没有“新建Blob”按钮属于正常现象。

### 6. 配置正式环境变量

有长期HTTPS入口后：

```text
EdgeOne Makers
→ 当前项目
→ Settings
→ Environment Variables
```

复制`config/production.env.template`中的变量名。八项随机秘密必须彼此不同且至少32字节。环境变量修改只应用于后续部署，保存后必须重新部署`main`。

### 7. 自定义域名

```text
EdgeOne Makers
→ 当前项目
→ Domain Management
→ Add custom domain
```

含中国大陆加速区的自定义域名需要先完成ICP备案。正式公共域名关联production环境，并跟随生产分支最新成功部署。

### 8. 独立管理员项目

使用交接包`admin-artifact`四文件及`config/edgeone-admin.project.json`创建独立管理员来源，将其纯HTTPS Origin填入`CLOUD_ADMIN_PUBLIC_ORIGIN`。不得把管理员四文件复制到普通用户三文件项目。

### 9. 分阶段开启

严格按照`activation-phases.json`：

```text
disabled
→ bootstrap_once
→ read_sync_only
→ admin_foundation
→ ordinary_submission_small_cohort
→ ordinary_auto_approval
→ admin_operations
→ sensitive_manual_review
```

每次只进入一个阶段，保存变量后重新部署并完成验收，不得一次性全部开启。
