# 阶段8D：负责人未来操作清单

## 现在是否需要操作

当前不需要负责人操作。

在没有长期管理员HTTPS域名之前，请不要：

```text
不要创建管理员正式项目
不要填写真实环境变量
不要生成或发送真实密钥
不要开启任何生产开关
不要创建重复空Blob
不要重新部署普通用户8.2.31候选
不要授权8.3.0稳定晋升
```

以下步骤仅在你拥有可控制的长期管理员HTTPS来源后执行。

---

## 1. 准备管理员长期HTTPS来源

建议形式：

```text
https://admin.你的域名
```

要求：

- 你能控制DNS；
- 只使用HTTPS；
- 不带路径、查询参数、片段或`eo_token`；
- 中国大陆节点按EdgeOne届时要求完成备案。

预期结果：管理员域名能长期直接访问，不需要3小时临时令牌。

失败处理：没有长期域名时停止，不使用EdgeOne临时预览链接填写`CLOUD_ADMIN_PUBLIC_ORIGIN`。

## 2. 在EdgeOne中新建管理员项目

控制台路径：

```text
EdgeOne Pages控制台
→ 项目列表
→ 新建项目
→ 导入GitHub仓库
→ 选择 srhovo/cloud-collab
```

项目建议名称：

```text
cloud-collab-admin
```

生产分支：

```text
main
```

关键设置：

```text
Root Directory：deploy/admin
```

不要选择仓库根目录。根目录设为`deploy/admin`后，EdgeOne应自动读取：

```text
deploy/admin/edgeone.json
```

预期显示：

```text
Node.js：22.11.0
输出目录：.edgeone-admin-artifact
构建脚本：prepare-admin-deployment-root-v1.mjs
```

失败处理：若控制台仍显示`npm run edgeone:build`或`.edgeone-artifact`，说明Root Directory未生效。停止部署，返回项目设置检查根目录，不手工删除仓库根`edgeone.json`。

## 3. 确认构建产物

首次构建成功后，在部署详情中核对静态产物只能包含：

```text
index.html
production-console.css
production-console.js
admin-release.json
```

同时构建日志应显示管理员Cloud Functions只位于：

```text
/api/admin/*
```

预期结果：不出现普通用户候选主页，也不出现匿名提交API。

失败处理：若出现`build-manifest.json`、`pages-release.json`、普通码单器主页或`/api/submissions`等匿名入口，立即停止，不绑定管理员域名，保存构建日志。

## 4. 核对Blob可访问性

管理员项目必须使用既有具名Store：

```text
cloud-collab-production-v1
cloud-collab-admin-production-v1
```

控制台可能位于：

```text
管理员项目
→ 存储 / Blob
或
管理员项目
→ 设置
→ 存储绑定
```

具体文字可能随EdgeOne更新变化。

预期结果：管理员项目能关联或访问已经存在的两个Store，而不是创建新的空数据库。

失败处理：

```text
若只能创建新空Store：停止
若同名Store显示为空：停止
若控制台提示跨项目不可访问：停止
不要复制或手工重建正式数据
不要创建同名空库冒充正式库
```

请保留该页面截图和错误提示，再继续技术评估。

## 5. 生成真实密钥

在受信任电脑的仓库根目录执行：

```bash
npm ci --ignore-scripts
npm run production:secrets:generate -- --output /安全目录/cloud-collab-production-secrets.env
```

生成后：

1. 保存到密码管理器；
2. 不发送到聊天；
3. 不提交GitHub；
4. 不上传公开云盘；
5. 不截图展示；
6. 不使用此前聊天中出现过的旧管理员密码。

预期结果：8项彼此不同且长度合格的随机值。

失败处理：删除不完整文件后重新生成，不用重复字符串人工补长度。

## 6. 录入管理员项目环境变量

控制台路径：

```text
cloud-collab-admin项目
→ 设置
→ 环境变量
```

按`config/production.env.template`录入。首次全部保持关闭：

```text
CLOUD_PRODUCTION_ENABLED=0
CLOUD_PRODUCTION_READ_SYNC_ENABLED=0
CLOUD_PRODUCTION_ORDINARY_SUBMISSION_ENABLED=0
CLOUD_PRODUCTION_AUTO_APPROVAL_ENABLED=0
CLOUD_PRODUCTION_SENSITIVE_SUBMISSION_ENABLED=0
CLOUD_PRODUCTION_ADMIN_REVIEW_ENABLED=0
CLOUD_PRODUCTION_DEVICE_GOVERNANCE_ENABLED=0
CLOUD_PRODUCTION_EXPORT_ENABLED=0
CLOUD_ADMIN_PRODUCTION_ENABLED=0
CLOUD_PRODUCTION_BOOTSTRAP_ENABLED=0
```

固定非秘密值：

```text
CLOUD_PRODUCTION_EXTERNAL_CLUB_ID=see
CLOUD_PRODUCTION_EXTERNAL_LIBRARY_ID=see_cz
CLOUD_PRODUCTION_GROUP_ID=group_see
CLOUD_PRODUCTION_LIBRARY_ID=lib_see_cz
CLOUD_PRODUCTION_BLOB_STORE_NAME=cloud-collab-production-v1
CLOUD_ADMIN_PRODUCTION_BLOB_STORE_NAME=cloud-collab-admin-production-v1
CLOUD_ADMIN_USERNAME=xiaxue
CLOUD_ADMIN_PUBLIC_ORIGIN=https://admin.你的域名
```

注意：正式回滚没有独立`CLOUD_PRODUCTION_ROLLBACK_ENABLED`变量；它依赖只读同步、管理员身份和管理员审核门禁。

预期结果：所有变量存在，但页面仍不能执行生产操作。

失败处理：任一密钥长度不足、两个密钥相同、Origin不是纯HTTPS、Store名称错误时停止，不放宽代码校验。

## 7. 绑定管理员域名

控制台路径：

```text
cloud-collab-admin项目
→ 自定义域名
→ 添加域名
```

填写：

```text
admin.你的域名
```

按控制台提示设置DNS。

预期结果：HTTPS证书生效，访问首页能看到管理员登录页；未登录时明确显示“当前没有有效管理员会话”。

失败处理：证书未生效、域名跳转到普通码单器、需要临时令牌或响应不是管理员页面时停止。

## 8. 核对真实HTTP安全响应头

浏览器开发者工具路径：

```text
开发者工具
→ Network
→ 重新加载管理员首页
→ 选择Document请求
→ Response Headers
```

必须存在：

```text
Cache-Control: no-store, max-age=0, must-revalidate
Content-Security-Policy
Strict-Transport-Security
X-Content-Type-Options: nosniff
X-Frame-Options: DENY
Referrer-Policy: no-referrer
Permissions-Policy
Cross-Origin-Opener-Policy: same-origin
Cross-Origin-Resource-Policy: same-origin
```

预期结果：CSP仅允许同源CSS、JS和API，不允许第三方资源或框架嵌入。

失败处理：缺少任一关键Header时，不录入生产开启值；保留Network截图和原始响应头。

## 9. 再执行初始化和分阶段开启

只有以下全部通过后才进入已有生产流程：

```text
长期管理员域名已生效
四文件产物正确
仅管理员API正确
两个既有Blob可访问
真实密钥已安全录入
HTTP安全响应头完整
```

之后按已有顺序：

```text
一次性空库初始化
→ 生产总开关
→ 只读同步
→ 管理员身份
→ 完整导出
→ 普通提交
→ 自动审核
→ 人工审核
→ 设备治理
→ 回滚验收
→ 敏感提交与敏感审核
```

每次只开启一项并完成真实验收。
