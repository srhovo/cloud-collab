# 阶段8E：管理员线上部署验证器

## 目标

阶段8D已生成独立管理员四文件产物和安全响应头模板。阶段8E新增只读验证器，用于未来管理员静态来源部署后，自动确认线上文件、提交身份、逐文件摘要和平台响应头与构建证据一致。

本阶段只实现验证器、模拟网络测试和无部署门禁，不创建项目、不访问真实来源、不填写环境变量、不启用生产能力。

## 将来执行命令

```bash
npm run admin:verify:deployment -- \
  --url https://管理员长期域名 \
  --expected-commit 40位Git提交SHA
```

地址必须是无凭据、无路径、无查询参数、无片段的纯HTTPS来源。临时预览地址不能作为正式管理员来源。

## 验证内容

线上来源必须提供：

```text
admin-release.json
index.html
production-console.css
production-console.js
```

验证器检查：

- 发布清单类型、40位源提交和精确四文件范围；
- 三个权威源文件到输出文件的映射；
- 每个HTML、CSS、JS的SHA-256和字节数；
- 不含普通用户候选和真实秘密；
- 生产能力默认关闭；
- 冻结候选仍为8.2.31及既有SHA；
- 稳定版仍为8.2.25；
- 稳定晋升和生产写入启用均为false。

## 平台响应头

四个成功响应必须包含：

```text
Cache-Control: no-store且max-age=0
Strict-Transport-Security: max-age至少31536000
X-Content-Type-Options: nosniff
X-Frame-Options: DENY
Referrer-Policy: no-referrer
Cross-Origin-Opener-Policy: same-origin
Cross-Origin-Resource-Policy: same-origin
Permissions-Policy: 禁用相机、麦克风、定位、支付和USB
```

CSP必须精确限制为：默认拒绝，仅允许同源连接、样式和脚本；图片、字体、媒体、对象、框架、Worker和Manifest均拒绝；`base-uri`、`form-action`和`frame-ancestors`均为`none`。

验证器检查真实HTTP响应头，不把HTML元信息当成平台安全头证据。

## 内容类型

```text
admin-release.json       application/json; charset=utf-8
index.html               text/html; charset=utf-8
production-console.css   text/css; charset=utf-8
production-console.js    application/javascript; charset=utf-8
```

缺少UTF-8、空文件、超出安全大小或非200均失败关闭。

## 普通产物排除

管理员来源中的`build-manifest.json`和`pages-release.json`必须返回404或410。任何一个返回200，都判定普通用户候选文件泄漏。

管理员HTML不得出现普通候选标题或候选APP版本锚点；CSS不得加载外部资源；JS不得使用浏览器持久化存储、读取Cookie、使用`innerHTML`、引用外部地址或旧导出响应头。

## 重试与结果

默认最多验证6次，每次间隔5秒。请求固定使用无缓存、拒绝重定向和不发送来源页信息。

成功状态：

```text
verified_admin_static_deployment_not_runtime_activation
```

成功结果仍明确：

```text
runtimeActivationVerified=false
productionWriteEnablementIncluded=false
stablePromotionPerformed=false
```

## 边界

静态验证通过不代表真实管理员登录、会话Cookie、Blob、私密环境变量或控制面API已经启用，也不代表移动端验收或稳定晋升完成。这些必须后续单独授权并逐级验收。

## 自动化测试

内存网络测试覆盖：完整成功、首次失败后重试成功、无效地址、提交错配、清单边界放宽、CSP/HSTS/UTF-8缺失、文件篡改和普通清单泄漏。

专项Actions只运行模拟验证和本地构建；无部署审计确认仓库只读、仅使用允许的官方Actions、无部署命令、无真实网络目标。
