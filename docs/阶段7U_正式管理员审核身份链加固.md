# 阶段7U：正式管理员审核身份链加固

## 背景

阶段7T已经接入正式精确价格、陪玩名字和老板资料的审核队列、详情与人工动作。随后对已合并实现进行独立安全审计，确认两项身份链缺陷：

1. 审核模式分发器只读取`CLOUD_ADMIN_PRODUCTION_ENABLED`。生产项目中管理员子开关关闭时，若历史预览变量仍存在，路由可能回退阶段5预览处理器；
2. 正式审核HTTP读取正式管理员配置，却调用阶段5A预览会话验证器。阶段7T已把正式会话issuer隔离为`cloud-collab-admin-production`，预览验证器会按`cloud-collab-admin-preview`校验，因此正式会话可能被错误拒绝。

本阶段只修复这两项缺陷，不新增审核能力、不修改协议、不部署EdgeOne、不访问真实Blob、不修改候选版本或稳定版。

## 模式分发修复

`src/server/admin_review_mode_dispatch_v1.js`不再自行读取管理员子开关，而是复用经过阶段7T加固的：

```text
resolveAdminAuthMode(env)
```

分发规则统一为：

```text
CLOUD_PRODUCTION_ENABLED=0或未配置 → 阶段5预览审核处理器
CLOUD_PRODUCTION_ENABLED=1          → 正式审核处理器
其他值                              → 503 PRODUCTION_FLAG_INVALID
```

进入正式处理器后，再由生产运行时验证：

```text
CLOUD_ADMIN_PRODUCTION_ENABLED=1
CLOUD_PRODUCTION_ADMIN_REVIEW_ENABLED=1
```

因此生产总开关为1、管理员子开关为0时只会失败关闭，不会回退预览队列或预览写入动作。

## 会话验证修复

`src/server/production_admin_review_http_v1.js`改为直接复用正式身份核心：

```text
readProductionAdminAuthConfig
verifyProductionAdminSessionToken
```

正式审核只接受issuer为：

```text
cloud-collab-admin-production
```

的15分钟HttpOnly会话。阶段5A预览令牌不能用于正式审核，正式令牌也不能用于预览审核。

## 保持不变

正式审核已有功能不变：

- 精确价格队列、详情、批准、拒绝、编辑后批准；
- 陪玩名字和老板资料队列、详情、批准、拒绝、编辑后批准；
- 正式公共Store与协议作用域；
- POST精确同源校验；
- 不可变决策、审计、完成和归档；
- 幂等重放、基线冲突与脱敏投影；
- 所有生产开关默认关闭；
- 稳定晋升授权为false。

## 回归门禁

测试改为使用`createProductionAdminSessionToken`签发正式会话，并新增：

- 正式issuer会话可访问正式队列和执行正式人工动作；
- 生产总开关为1、管理员子开关为0、预览变量仍存在时，返回正式503且不创建Store；
- 非法生产总开关在进入任何审核处理器前返回`PRODUCTION_FLAG_INVALID`；
- 静态门禁禁止正式审核HTTP调用`verifyAdminSessionToken`；
- 静态门禁禁止审核分发器读取`CLOUD_ADMIN_PRODUCTION_ENABLED`；
- 阶段5B、5C、5G预览审核回归继续运行；
- 完整Node与浏览器矩阵继续运行。
