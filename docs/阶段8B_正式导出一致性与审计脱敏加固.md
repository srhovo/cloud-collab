# 阶段8B：正式导出一致性与审计脱敏加固

## 背景

阶段8A已经接入正式完整公共数据库导出。独立审计确认两个需要在真实启用前加固的边界：

1. 普通事件、敏感事件和统一快照由多个Blob读取组成。导出期间若追加新公共事件，不能生成“快照已更新、审计历史仍旧”的混合包。
2. 正式导出审计身份必须真正使用独立导出审计盐，不应只对固定管理员用户名做无盐摘要。

## 公共状态一致性

阶段8B把导出读取顺序固定为：

```text
第一次读取普通事件链
→ 第一次读取敏感事件链
→ 构建统一公共快照
→ 第二次读取普通事件链
→ 第二次读取敏感事件链
→ 比较前后完整事件指纹
→ 核对快照普通基线、公共版本与快照版本
→ 生成ZIP
```

只有以下条件全部成立才允许生成ZIP：

- 前后普通事件链完全一致；
- 前后敏感事件链完全一致；
- 快照`groupId/libraryId`与正式作用域一致；
- 快照`baseOrdinaryVersion`等于普通事件链末版本；
- 快照`publicVersion`等于敏感事件末版本，或无敏感事件时等于普通事件末版本；
- `snapshotVersion === publicVersion`。

任何并发变化统一返回：

```text
PRODUCTION_EXPORT_PUBLIC_STATE_MOVED
HTTP 409
```

管理员应使用新的请求重新导出。底层公共基线移动类409也统一映射为该可重试错误，不伪装成500/503。

## 审计身份脱敏

正式管理员会话仍向管理员页面显示真实用户名`xiaxue`，但下载审计核心不再接收该明文身份。

HTTP适配器先执行：

```text
HMAC-SHA-256(CLOUD_ADMIN_EXPORT_AUDIT_SALT, normalized username)
```

然后把结果作为导出专用伪名传入不可变请求、决策和审计链。后续`actorTag`和请求摘要因此随审计盐变化，无法仅凭固定用户名在不同环境间直接关联。

性质：

- 同一用户名、同一审计盐：伪名稳定，支持精确重放；
- 同一用户名、不同审计盐：伪名不同；
- 伪名不包含真实用户名；
- 页面会话身份、登录流程和Cookie不变；
- 审计盐本身不写入Blob、不进入响应、不进入ZIP。

## 响应边界

正式下载额外明确返回：

```text
X-Cloud-Collab-Stable-Promotion-Authorized: 0
```

该阶段不构成8.3.0稳定晋升授权。

## 自动测试

`tests/stage8b-export-consistency-hardening.test.mjs`覆盖：

- 稳定状态执行两次普通事件读取、两次敏感事件读取和一次快照读取；
- 前后事件链变化时返回409；
- 快照版本与事件链不一致时返回409；
- 底层公共基线409映射为统一可重试错误；
- HMAC审计伪名稳定、可轮换且不包含真实用户名；
- 正式HTTP只把伪名传给下载审计核心；
- 下载响应继续声明稳定晋升未授权。

专项工作流同时回归阶段8A正式导出、阶段5F预览导出、正式管理员身份、敏感审核、公共回滚和Cloud Function相对导入。
