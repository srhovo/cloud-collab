# 阶段4A：普通价格提交协议 v1

## 本阶段目标

建立普通精确价格的服务端提交协议、字段白名单、标准化、Hash、幂等键和候选资格判断。

本阶段**不开放真实写入接口**，不注册设备、不签发令牌、不保存候选、不修改公共版本，也不执行自动批准。

## 冻结原子提交结构

```json
{
  "schemaVersion": 1,
  "payloadSchemaVersion": 1,
  "submissionId": "sub_01JABCDEF0123456789XYZABCD",
  "deviceId": "dev_01JABCDEF0123456789XYZABCD",
  "groupId": "group_fixture",
  "libraryId": "lib_receive_fixture",
  "bossId": null,
  "dataType": "exact_price",
  "operation": "upsert",
  "origin": "user",
  "clientCreatedAt": 1784376000000,
  "businessKey": "bk_v1_...",
  "contentHash": "ch_v1_...",
  "idempotencyKey": "ik_v1_...",
  "payload": {
    "serviceName": "测试服务A",
    "settleType": "round",
    "unitPrice": 110
  },
  "clientContext": {
    "appVersion": "8.2.28",
    "projectionSpecVersion": 1,
    "queueSchemaVersion": 1
  }
}
```

顶层、`payload` 与 `clientContext` 均采用严格白名单。多字段、少字段、未来版本、私人字段或禁止来源都拒绝。

## 当前开放范围

- `dataType`：仅 `exact_price`
- `operation`：仅 `upsert`
- `origin`：仅 `user` 或 `initialBinding`
- `libraryId`：普通价格必须存在
- `bossId`：普通价格必须为 `null`
- `settleType`：`round` 或 `hour`
- `unitPrice`：大于0、不超过1000000、最多3位小数
- `serviceName`：Unicode NFKC、合并重复空格、长度1～50、禁止控制字符
- 单次请求体：最大 **16384字节（16KB）**

## 服务端重算

客户端上报的三个Hash均不可信，服务端必须用冻结算法重新计算并逐项比对：

```text
businessKey = bk_v1_ + base64url(SHA-256(canonical identity))
contentHash = ch_v1_ + base64url(SHA-256(canonical content))
idempotencyKey = ik_v1_ + base64url(SHA-256(canonical {
  schemaVersion: 1,
  deviceId,
  submissionId
}))
```

普通价格身份对象与阶段3B保持一致：

```text
groupId + libraryId + normalizedServiceName + settleType + ruleType=exact + variant=standard
```

测试服务A（按局，110）的冻结结果：

```text
businessKey: bk_v1_ja06mv-cCqOze_uiSIK4YjKoixcrewF-NIQXmAiTyTQ
contentHash: ch_v1_8cLsoYwgKnmAXyDgNjXMkDLlD2t2JjdXyKrde_KhoqA
```

## 候选资格判断

- 与正式公共内容完全相同：`duplicate_noop`
- 新内容且只有一个普通设备：`waiting_confirmation`
- 两个不同设备提交相同内容：`eligible_auto_approval`
- 可信设备提交普通安全内容：`eligible_auto_approval`
- 正式公共值不同：`pending_review`
- 同一业务键存在冲突候选：`pending_review`

阶段4A无论资格结果为何，都必须保持：

```text
publicMutationAllowed = false
autoApprovalEnabled = false
```

## 未编码为硬规则的内容

“价格变化±10%”没有获得最终业务确认，因此不得写入不可变提交协议。价格幅度判断只能在后续可版本化审核策略中实现；当前已有公共价格发生变化时一律进入审核资格。
