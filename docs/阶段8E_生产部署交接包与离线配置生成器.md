冻结结论：

1. 固定项目域名指向最新成功部署；当前含中国大陆区域通过平台域名访问仍使用三小时预览链接。
2. 主分支推送会触发生产部署，也可在部署记录中手动重新部署。
3. 自定义域名关联生产环境后，跟随该环境最新成功部署。
4. 当前含中国大陆区域绑定自定义域名前需要ICP备案。
5. 环境变量修改只影响新部署，保存后必须重新部署。
6. Blob命名空间不需要在控制台手工创建；首次SDK `getStore("名称")`调用并触发请求后自动创建。控制台主要用于只读查看命名空间和对象目录。

## 离线配置工具

```text
tools/production-secret-generator.html
tools/production-secret-generator.css
tools/production-secret-generator.js
```

工具安全边界：

```text
每项使用Web Crypto生成48随机字节
八项随机值彼此独立
不联网
不使用浏览器持久化存储
不读取Cookie
不申请剪贴板权限
关闭页面或pagehide时清空内存
所有生产开关保持0
初始化确认词保持空
```

工具不会进入普通用户三文件产物，也不会进入管理员四文件产物。专项Actions只上传工具源文件，不上传运行中生成的值。

## 未来需要负责人亲自完成的6步

当前都不需要执行。拥有可控制域名后再按顺序操作。

### 1. 准备域名与ICP备案

路径：域名注册商控制台；需要时进入工信部ICP备案系统。

建议规划：

```text
app.<你的域名>    普通用户入口
admin.<你的域名>  管理员入口
```

### 2. 绑定两个正式域名

路径：

```text
EdgeOne Makers
→ cloud-collab项目
→ 域名管理
→ 添加自定义域名
```

分别把普通用户域名和管理员域名关联到生产环境，按平台提示完成DNS归属验证和CNAME。

### 3. 在可信设备生成并导入配置

路径：

```text
本地打开 tools/production-secret-generator.html
EdgeOne Makers → cloud-collab项目 → 项目设置 → 环境变量
```

操作：

1. 本地打开HTML。
2. 填入两个正式HTTPS来源。
3. 点击“生成全部”。
4. 将八项值分别保存到密码管理器。
5. 使用系统复制菜单复制批量文本并导入EdgeOne。
6. 确认所有生产开关仍为0。
7. 保存后触发新部署。

不得把生成值发到聊天、GitHub、Actions日志、普通用户页面或截图中。

### 4. 一次性初始化

触发部署路径二选一：

```text
向main推送已验收提交
或
EdgeOne Makers → cloud-collab项目 → 部署记录 → 重新部署
```

初始化时仅设置：

```text
CLOUD_PRODUCTION_BOOTSTRAP_ENABLED=1
CLOUD_PRODUCTION_BOOTSTRAP_CONFIRMATION=INITIALIZE-see-see_cz-V1
```

其他生产开关全部保持0。初始化请求首次调用`getStore`后自动创建：

```text
cloud-collab-production-v1
cloud-collab-admin-production-v1
```

成功复核后立即把初始化开关恢复为0、确认词清空，并再次部署。

### 5. 分阶段开放

每次只改一组开关并重新部署：

```text
总开关与只读同步
→ 普通提交小流量
→ 普通自动审核
→ 管理员身份与人工审核
→ 敏感提交
→ 设备治理
→ 完整导出
```

敏感提交可独立暂停，暂停后管理员仍能处理存量敏感队列。

```bash
npm run production:handoff
```

输出：

```text
dist/production-handoff-v1.json
dist/production-owner-actions-v1.md
dist/production-edgeone-env-template-v1.txt
```

这些文件只包含变量名、固定非敏感参数、关闭状态和操作路径，不包含真实随机值。
