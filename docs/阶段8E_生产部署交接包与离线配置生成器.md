# 阶段8E：生产部署交接包与离线配置生成器

## 当前结论

阶段8E提供负责人操作清单和可信设备离线配置工具；阶段8H把真实Blob初始化加固为手动`main`触发、审批环境、双确认和独立解锁值；阶段8I固定`xiaxue.site`并发现跨项目Blob边界；阶段8J已通过单项目双域名Host隔离解决该边界。

当前不部署EdgeOne、不修改环境变量、不创建真实Blob对象、不生成真实私密值、不启用生产能力，也不授权8.3.0稳定晋升。

## 已选正式域名与平台拓扑

```text
主域名：xiaxue.site
普通入口：https://app.xiaxue.site
管理员入口：https://admin.xiaxue.site
EdgeOne项目数量：1
```

两个域名绑定同一个EdgeOne项目。普通与管理员静态文件和API通过`middleware.js`按Host严格隔离；两个Blob命名空间属于同一项目，因此不需要在长期运行环境中保存平台账户级API Token。

在域名注册、实名、DNS和HTTPS获得实际证据前，`config/production.env.template`中的两个Origin继续保持空白。

## 离线配置工具

```text
tools/production-secret-generator.html
tools/production-secret-generator.css
tools/production-secret-generator.js
```

工具使用Web Crypto为八项私密变量分别生成48随机字节，并保持：不联网、不使用浏览器持久化存储、不读取Cookie、不申请剪贴板权限、关闭页面时清空内存。工具不会进入普通用户或管理员线上产物。

## 当前负责人操作

现在只执行：

```text
腾讯云控制台
→ 域名注册
→ 我的域名
→ xiaxue.site
```

确认域名状态正常、实名认证完成、到期时间正确，并建议开启自动续费。不要购买轻量服务器、付费DNS或单独SSL证书，也不要提前猜测EdgeOne要求的TXT或CNAME记录。

## 后续顺序

1. 选择EdgeOne加速区域与ICP备案路线；
2. 创建或复用一个EdgeOne项目；
3. 把`app.xiaxue.site`和`admin.xiaxue.site`都绑定到该项目；
4. 按域名向导完成所有权验证和CNAME；
5. 为两个域名申请免费HTTPS证书并开启强制HTTPS；
6. 实际验证后填写两个正式Origin；
7. 在可信设备生成私密配置，全部开关仍保持0；
8. 完成单项目双域名Host隔离与响应头核验；
9. 单独审查是否允许阶段8H真实初始化；
10. 按只读、普通提交、自动审核、管理员、敏感提交、设备治理、导出的顺序逐级验收；
11. L4通过后另行授权8.3.0晋升。

不要创建第二个管理员EdgeOne项目。旧的`deploy/admin`独立部署根继续保留为历史审计材料和回退参考，不属于当前正式拓扑。
