# 阶段8E：生产部署交接包与离线配置生成器

## 当前结论

阶段8E提供负责人操作清单和可信设备离线配置工具。阶段8H已经把真实Blob初始化加固为手动`main`触发、审批环境、双确认和独立解锁值。阶段8I进一步固化正式域名`xiaxue.site`，并发现独立管理员项目的跨项目Blob边界尚未解决。

当前不部署EdgeOne、不修改环境变量、不创建真实Blob对象、不生成真实私密值、不启用生产能力、不修改8.2.31候选，也不授权8.3.0稳定晋升。

## 跨项目Blob阻断

当前运行时通过当前EdgeOne项目调用命名Store。若普通项目与管理员项目完全分离，仅使用相同Store名称不能自动共享数据。

平台支持通过目标项目ID和API Token外部访问Blob，但平台API Token属于账户级凭据。项目安全边界明确禁止把账户级访问令牌长期放入管理员Functions环境变量。

阶段8I完成以下方案之一的设计、代码和自动化验证前：

```text
管理员项目创建：禁止
管理员域名绑定：禁止
真实Blob初始化execute：禁止
GitHub初始化Environment Secrets配置：禁止
生产能力启用：禁止
```

待评估路线：

1. 管理员项目只负责控制台和会话，公共数据操作通过专用签名网关进入普通数据项目；
2. 单项目多域名，并按Host严格隔离普通与管理员静态产物、API和响应头。

## 离线配置工具

```text
tools/production-secret-generator.html
tools/production-secret-generator.css
tools/production-secret-generator.js
```

工具使用Web Crypto为八项私密变量分别生成48随机字节，并保持：不联网、不使用浏览器持久化存储、不读取Cookie、不申请剪贴板权限、关闭页面时清空内存。工具不会进入普通用户或管理员线上产物。

## 当前负责人操作

现在只执行：

```text
腾讯云控制台
→ 域名注册
→ 我的域名
→ xiaxue.site
```

确认域名状态正常、实名认证完成、到期时间正确，并建议开启自动续费。不要购买轻量服务器、付费DNS或单独SSL证书，也不要提前猜测EdgeOne要求的TXT或CNAME记录。

## 阻断解除后的顺序

1. 在EdgeOne普通项目添加`app.xiaxue.site`；
2. 按域名向导完成所有权验证和CNAME；
3. 申请免费HTTPS证书并开启强制HTTPS；
4. 实际验证后填写`CLOUD_PRODUCTION_PUBLIC_ORIGIN=https://app.xiaxue.site`；
5. 按最终管理员架构配置`admin.xiaxue.site`；
6. 实际验证后填写`CLOUD_ADMIN_PUBLIC_ORIGIN=https://admin.xiaxue.site`；
7. 在可信设备生成私密配置，全部开关仍保持0；
8. 完成双来源零开关部署验证；
9. 单独审查是否允许阶段8H真实初始化；
10. 按只读、普通提交、自动审核、管理员、敏感提交、设备治理、导出的顺序逐级验收；
11. L4通过后另行授权8.3.0晋升。
