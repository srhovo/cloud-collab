# 阶段7T：正式管理员普通审核队列与动作

## 目标

在阶段7S正式管理员登录/会话和正式敏感候选入队已经完成后，将既有管理员普通审核能力接入生产运行时：

- 精确价格审核队列和详情；
- 陪玩名字、老板普通资料审核队列和详情；
- 批准；
- 拒绝；
- 修改后批准；
- 不可变决策、审计、完成和归档记录；
- 公共事件与公共快照更新。

## API保持不变

精确价格：

```text
GET  /api/admin/reviews
GET  /api/admin/reviews/detail?id=...
POST /api/admin/reviews/approve
POST /api/admin/reviews/reject
POST /api/admin/reviews/edit-and-approve
```

陪玩名字和老板资料：

```text
GET  /api/admin/ordinary-reviews
GET  /api/admin/ordinary-reviews/detail?id=...
POST /api/admin/ordinary-reviews/approve
POST /api/admin/ordinary-reviews/reject
POST /api/admin/ordinary-reviews/edit-and-approve
```

`CLOUD_ADMIN_PRODUCTION_ENABLED=0`或未配置时继续使用阶段5B/5C/5G预览处理器；显式为`1`时使用正式处理器；其他值返回503。

## 正式隔离

```text
管理员会话与登录限流Store：cloud-collab-admin-production-v1
审核队列、候选、公共事件和审计Store：cloud-collab-production-v1
协议作用域：group_see / lib_see_cz
管理员HTTPS来源：CLOUD_ADMIN_PUBLIC_ORIGIN
```

生产处理器不会将管理员会话对象写入公共Store，也不会从管理员Store读取公共候选。

## 通用归档Key

阶段5G普通审核模块原本复用一个只允许`lib_receive_fixture`的预览归档Key函数。阶段7T将普通审核模块切换到`admin_review_key_v1.js`：

- 只接受`lib_`开头的严格ID；
- 只接受格式正确的审核ID；
- 通过统一Blob Key规范化；
- 阶段5B/5C预览配置本身仍锁定`lib_receive_fixture`；
- 正式配置允许`lib_see_cz`。

该变更只泛化安全Key生成，不放宽预览环境变量或合成作用域门禁。

## 身份与同源边界

- 所有端点必须持有阶段7S签发的有效HttpOnly会话；
- POST动作要求精确匹配管理员`Origin`；
- 跨站请求在创建公共Store前被拒绝；
- 会话过期或伪造返回401并清除Cookie；
- 所有响应继续脱敏设备ID、提交ID、幂等键和内部Blob路径；
- 普通用户三文件公开白名单不包含管理员页面。

## 写入语义

批准和修改后批准复用既有阶段5C/5G公共事件引擎：

- 公共事件、决策、审计、完成、归档均为不可变写入；
- 同一审核决策精确重放返回已有结果；
- 不同决策抢占同一审核项返回冲突；
- 公共基线变化时批准失败关闭；
- 拒绝不修改公共版本；
- 精确价格与普通类型继续使用各自已验证的修改确认词和安全规则。
