# 阶段8D：独立管理员产物与部署前检查

## 当前结论

阶段8D只建立管理员独立构建、响应头模板和产物互斥验证，**不部署任何管理员页面**。

当前负责人无需手动操作，也不要：

```text
不要重新部署普通用户8.2.31候选
不要创建临时管理员公开入口
不要填写真实环境变量
不要开启生产开关
不要创建或写入真实Blob
不要授权8.3.0稳定晋升
```

## 两套严格分离的产物

普通用户候选仍为：

```text
.edgeone-artifact/
  index.html
  build-manifest.json
  pages-release.json
```

管理员产物为：

```text
.edgeone-admin-artifact/
  index.html
  production-console.css
  production-console.js
  admin-release.json
```

管理员构建直接复制阶段8C权威源：

```text
admin/production-console.html -> index.html
admin/production-console.css  -> production-console.css
admin/production-console.js   -> production-console.js
```

不再维护或提交第二份`dist/admin-production-console.html`，避免控制台实现分叉。

## 构建命令

```bash
npm run admin:prepare -- --output .edgeone-admin-artifact
npm run admin:verify:isolation
npm run edgeone:admin:build
```

`edgeone:admin:build`先运行完整Node CI，再生成管理员四文件产物。普通项目原命令保持不变：

```bash
npm run edgeone:build
```

生成目录均在`.gitignore`中，不提交GitHub。

## admin-release.json

管理员发布清单记录：

- 40位源提交SHA；
- 三个权威源文件及对应输出文件；
- 每个源文件SHA-256和字节数；
- 精确四文件输出白名单；
- 独立管理员来源和正式管理员会话要求；
- 平台响应头必须另行配置；
- 普通用户候选未包含；
- 真实秘密未包含；
- 所有生产能力默认关闭；
- 冻结候选仍为8.2.31及其既有摘要；
- 稳定版仍为8.2.25；
- 稳定晋升与生产写入启用均为false；
- 部署状态为`code_complete_not_deployed`。

该清单不包含真实域名、访问令牌、密码、会话密钥或盐值。

## 管理员项目配置模板

```text
config/edgeone-admin.project.json
```

模板与普通用户根`edgeone.json`相互独立。它冻结：

```text
安装命令：npm ci --ignore-scripts
构建命令：npm run edgeone:admin:build
输出目录：./.edgeone-admin-artifact
Node：22.11.0
```

平台响应头至少包含：

```text
Cache-Control: no-store, max-age=0
Content-Security-Policy:
  default-src 'none'
  connect-src 'self'
  style-src 'self'
  script-src 'self'
  object-src 'none'
  frame-src 'none'
  worker-src 'none'
  base-uri 'none'
  form-action 'none'
  frame-ancestors 'none'
Strict-Transport-Security: max-age=31536000
X-Content-Type-Options: nosniff
X-Frame-Options: DENY
Referrer-Policy: no-referrer
Cross-Origin-Opener-Policy: same-origin
Cross-Origin-Resource-Policy: same-origin
Permissions-Policy: camera=(), microphone=(), geolocation=(), payment=(), usb=()
```

HTML中的CSP元信息只是浏览器内第二道限制；`frame-ancestors`和HSTS必须由平台实际响应头提供。因此本阶段不会把“HTML中出现某个字符串”当成真实部署安全证据。

## 自动互斥验证

`scripts/verify-admin-public-artifact-isolation-v1.mjs`要求：

1. 管理员目录精确四文件；
2. 普通目录精确三文件；
3. 所有文件为普通文件且不是符号链接；
4. 管理员源文件、输出文件、SHA-256和字节数完全一致；
5. 两个`index.html`内容和摘要不同；
6. 管理员产物不包含普通候选标题、APP_VERSION或公开清单；
7. 普通产物不包含管理员CSS、JS或发布清单；
8. 普通候选仍为8.2.31、稳定版仍为8.2.25且未晋升；
9. 管理员配置模板的构建字段、CSP、HSTS和各文件内容类型完整。

任一条件不满足，构建或CI失败关闭。

## 将来必须人工完成的最少事项

只有在另行决定真实上线后，才需要负责人完成：

1. 准备可长期控制的管理员HTTPS来源；
2. 在平台创建与普通用户项目分离的管理员项目；
3. 按配置模板录入构建设置和响应头；
4. 确认部署产物只含管理员四文件；
5. 所有生产开关先保持0；
6. 再按单独授权和逐级验收方案创建资源、录入私密变量、初始化和开启能力。

临时令牌链接、短期预览地址或聊天中出现过的凭据都不能作为正式管理员来源或真实密钥。
