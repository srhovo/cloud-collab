# 阶段7Q：正式普通陪玩名字与老板资料候选

## 目标

在阶段7P正式设备注册与精确价格候选入队基础上，把阶段5G已验收的普通共享类型接入同一个正式提交入口：

```text
playable_name
boss_profile
```

本阶段仍然只接收不可变候选，不执行普通自动审核，不修改公共库。

## 复用范围

正式普通候选复用：

- `normalizeOrdinarySubmission`严格白名单与隐私校验；
- 陪玩名字、老板名、派单字段长度与联系方式拦截；
- 老板折数范围与精度校验；
- `deriveBossId`稳定老板身份；
- 普通业务Key、内容Hash和幂等Key；
- `acceptOrdinarySubmission`不可变候选写入；
- 设备令牌鉴权；
- 生产作用域与生产限流。

## 统一正式入口

```text
POST /api/submissions/create
```

分发：

- `exact_price` → 精确价格候选接收器；
- `playable_name` → 普通共享候选接收器；
- `boss_profile` → 普通共享候选接收器；
- 删除、区间、加价、礼物和其他敏感变化 → `PRODUCTION_SENSITIVE_HANDLER_REQUIRED`；
- 未知类型 → 400并返回普通类型白名单。

设备注册响应同步声明三个普通候选类型。

## 安全边界

- 正式提交必须使用协议作用域`group_see / lib_see_cz`；
- 用户可见作用域仍为`see / see_cz`；
- 联系方式、邮箱、电话、链接和永不上传字段在入队前拒绝；
- `bossId`必须等于服务端按`groupId + bossName`派生结果；
- Authorization设备必须等于正文deviceId；
- 新普通候选使用5秒生产限流；
- 精确幂等重放不消耗新限流槽；
- 候选保存后`publicMutationAllowed=false`；
- `autoApprovalEnabled=false`；
- `stablePromotionAuthorized=false`。

## 自动审核和敏感边界

若普通自动审核开关提前开启，正式提交入口在读取正文和创建Store前返回503：

```text
PRODUCTION_AUTO_APPROVAL_HANDLER_REQUIRED
```

删除与敏感类型不会回落到普通接收器，必须等待独立的敏感人工审核处理器。
