# 阶段8D：生产部署交接包与离线配置生成器

## 目标

阶段8C后，正式运行时、管理员控制台、设备治理、回滚和完整导出已经具备代码与自动测试。当前真正阻断正式上线的是负责人可控制的域名、平台私密环境变量、一次性Blob初始化和真实环境L4验收。

阶段8D把这些未来人工动作压缩成可下载、可机器复核的交接包；本阶段不部署EdgeOne、不创建真实Blob对象、不填写真实私密值、不启用生产开关、不修改8.2.31候选或8.2.25稳定版。

## 已核实的EdgeOne行为

官方文档：

- 域名概览：`https://pages.edgeone.ai/zh/document/domain-overview`
- 自定义域名：`https://pages.edgeone.ai/zh/document/custom-domain`
- 部署触发：`https://pages.edgeone.ai/zh/document/create-deploys`
- 构建与环境变量：`https://pages.edgeone.ai/zh/document/build-guide`
- Blob：`https://pages.edgeone.ai/zh/document/blob-storage`

冻结结论：

1. 固定项目域名会指向最新成功部署；当前含中国大陆区域通过平台域名访问仍需要三小时预览链接。
2. Git仓库主分支推送会自动触发生产部署；也可在部署记录中手动重新部署。
3. 自定义域名关联生产环境后，自动指向该环境最新成功部署。
4. 当前含中国大陆区域添加自定义域名前需要ICP备案。
5. 环境变量修改不会影响旧部署，只对新部署生效，因此修改后必须触发新部署。
6. Blob命名空间无需在控制台手工创建。代码首次调用`getStore("名称")`并触发请求后自动创建；控制台Blob页面用于只读查看命名空间和对象目录。

## 离线配置生成器

源文件：

```text
tools/production-secret-generator.html
tools/production-secret-generator.css
tools/production-secret-generator-config.js
tools/production-secret-generator.js
```

安全边界：

```text
Web Crypto每项生成48随机字节
八项值彼此独立
不联网
不调用fetch/XHR/WebSocket
不使用localStorage/sessionStorage/IndexedDB
不读取document.cookie
不申请剪贴板权限
关闭页面或pagehide时清空内存
所有生产开关保持0
初始化确认词保持空
```

工具不会进入普通用户EdgeOne或GitHub Pages三文件产物。CI会将四个文件作为独立交接产物上传；以后在可信设备解压到同一文件夹，本地打开HTML即可。

## 负责人未来必须亲自执行的操作

当前无需执行。拥有可控制域名后按以下顺序操作。

### 1. 准备域名与备案

路径：域名注册商控制台，以及需要时的工信部ICP备案系统。

操作：

1. 购买或使用已有域名。
2. 当前项目加速区域包含中国大陆，因此绑定前完成ICP备案。
3. 建议规划两个子域名：

```text
app.<你的域名>    普通用户入口
admin.<你的域名>  管理员控制台入口
```

不要把带`eo_token`的临时预览链接填入正式来源。

### 2. 绑定两个生产域名

路径：

```text
EdgeOne Makers控制台
→ cloud-collab项目
→ 域名管理
→ 添加自定义域名
```

操作：

1. 添加普通用户子域名并关联生产环境。
2. 添加管理员子域名并关联生产环境。
3. 按弹窗指引在域名注册商添加归属验证记录和CNAME。
4. 等待平台显示已生效和HTTPS可用。

应填写的来源：

```text
CLOUD_PRODUCTION_PUBLIC_ORIGIN=https://app.<你的域名>
CLOUD_ADMIN_PUBLIC_ORIGIN=https://admin.<你的域名>
```

### 3. 离线生成并导入配置

路径：

```text
可信设备本地：tools/production-secret-generator.html
EdgeOne Makers → cloud-collab项目 → 项目设置 → 环境变量
```

操作：

1. 在本地打开离线工具。
2. 填入两个HTTPS来源。
3. 点击“生成全部”。
4. 把八项随机值分别保存到密码管理器。
5. 选中批量导入文本并用系统复制菜单复制。
6. 在EdgeOne环境变量页面批量导入。
7. 确认所有功能开关仍为`0`后保存。
8. 环境变量变更仅对新部署生效，保存后必须重新部署。

不得把生成值发到聊天、GitHub、Actions日志、普通用户HTML或截图中。

### 4. 触发部署与一次性初始化

触发路径二选一：

```text
向main推送已验收提交
或
EdgeOne Makers → cloud-collab项目 → 部署记录 → 重新部署
```

首次初始化前：

```text
CLOUD_PRODUCTION_ENABLED=0
全部业务能力开关=0
CLOUD_ADMIN_PRODUCTION_ENABLED=0
CLOUD_PRODUCTION_BOOTSTRAP_ENABLED=1
CLOUD_PRODUCTION_BOOTSTRAP_CONFIRMATION=INITIALIZE-see-see_cz-V1
```

触发初始化请求后，代码首次调用`getStore`自动创建：

```text
cloud-collab-production-v1
cloud-collab-admin-production-v1
```

初始化成功并复核对象后，立即把：

```text
CLOUD_PRODUCTION_BOOTSTRAP_ENABLED=0
CLOUD_PRODUCTION_BOOTSTRAP_CONFIRMATION=
```

并再次部署。初始化执行必须由负责人在操作前再次确认，不能由CI自动运行。

### 5. 分阶段开放

每次只改一组开关，重新部署并完成对应验收：

```text
生产总开关 + 只读同步
→ 普通提交小流量
→ 普通自动审核
→ 正式管理员身份与人工审核
→ 敏感提交
→ 设备治理
→ 完整导出
```

敏感提交入口可以独立关闭；关闭新候选后管理员仍可处理存量敏感队列。

### 6. L4与稳定晋升

完成桌面、Android、iPhone、离线、故障、回滚、完整导出和管理员控制台真实验收后，项目负责人必须另行明确授权8.3.0稳定晋升。

当前：

目标稳定版：8.3.0
稳定晋升授权：false
稳定晋升执行：false
```

```

```text

```
