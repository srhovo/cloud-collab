# 阶段3B：只接收同步规则

## 1. 触发条件

只有价格库绑定模式为 `receive` 或 `collaborate` 时允许读取公共数据。

触发入口：

- 保存绑定后的异步检查
- 页面启动后的异步检查
- 页面可见状态下每5分钟轮询
- 用户点击“检查并接收更新”

页面启动不等待网络，服务器失败不阻塞码单。

## 2. 当前接收范围

阶段3B只接收 `exact_price`，投影字段固定为：

```json
{
  "serviceName": "服务名称",
  "settleType": "round 或 hour",
  "unitPrice": 100
}
```

区间、加价、礼物、陪玩名字和老板资料本阶段均不会写入本地。

## 3. 读取路径

- 本地版本为0、强制检查或增量失败：读取完整快照。
- 本地版本大于0且远端版本更高：优先读取增量事件。
- 增量分页每页最多100条，最多循环10页。
- 响应正文最大512KB。
- 快照记录总数最多5000条。

## 4. Hash验证

客户端按冻结协议重新规范化并重新计算：

- `businessKey`
- `contentHash`

Web Crypto 可用时优先调用 `crypto.subtle.digest('SHA-256')`；本地文件等环境没有 Web Crypto 时使用同算法的纯 JavaScript 回退实现。任一 Hash 不一致时整次同步停止，价格库保持原状。

## 5. 三方比较

三方分别为：

- `baseHash`：上次成功接收时的公共内容
- `localHash`：当前本地内容
- `remoteHash`：本次公共内容

规则：

1. 本地与远端相同：确认基础 Hash。
2. 本地仍等于基础：安全应用远端。
3. 远端仍等于基础：保留本地修改。
4. 本地与远端都偏离基础：记录冲突，不覆盖本地。
5. 远端删除：仅当本地仍等于基础时删除；否则记录冲突。
6. 没有基础且本地不存在：新增远端数据。
7. 没有基础但本地已存在不同内容：记录冲突。

## 6. 持久化与回滚

- 价格内容必须通过现有 `PriceLibraryStore.persist()` 写入。
- 活动价格库兼容镜像 `priceMemory` 继续由该 Store 维护。
- 同步状态和绑定版本使用本地跨 Store 事务。
- 任一步失败时恢复价格库、兼容镜像、同步状态和绑定版本。
- 回滚不完整时停用云功能，但旧码单功能继续运行。

## 7. 冲突数据

本地只保存：

- businessKey
- base/local/remote Hash
- 数据类型
- 发现时间
- 状态

不保存完整远端价格副本、订单、历史、聊天或个人数据。再次强制检查且本地已经与远端一致时，对应冲突摘要自动清除。

## 8. 上传边界

云端拉取来源不会调用待上传队列。`pendingCloudChanges` 在本阶段始终为0；用户提交与管理员写入能力仍关闭。
