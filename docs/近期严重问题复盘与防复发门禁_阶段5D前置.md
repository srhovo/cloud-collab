# 近期严重问题复盘与防复发门禁

适用起点：`main@82ec83a4c57abdf5df3a232ea733bda73d1b465d`

## 1. EdgeOne HTTPS与同源误判

真实公网为HTTPS，但Cloud Function运行时可能看到内部HTTP。最终方案不是放宽HTTPS，而是：

- `CLOUD_ADMIN_PUBLIC_ORIGIN`只接受纯HTTPS来源，不带路径或查询参数；
- 从公开来源识别EdgeOne项目，允许同项目轮换部署域名；
- 内部HTTP仅在host、HTTPS Origin、同项目和`Sec-Fetch-Site=same-origin`同时成立时认可；
- 所有管理员路由必须传递`publicOrigin`，不能只修登录而漏掉5B、5C、清理或5D；
- 单测必须模拟“内部HTTP URL + 公网HTTPS Origin”。

## 2. 环境变量已改但旧部署仍失败

环境变量更新不会改变旧部署。固定顺序为：保存环境变量、生成新部署、打开新URL、核对API版本，再删除旧部署。部署问题优先核对head和URL，不先改业务代码；真实验收Blob默认保留，不重做提交。

## 3. 笼统错误与页面内存残留

清理页曾只显示“暂时不可用”，且失败后继续复用页面内存中的旧输入。最终方案为：

- 保留稳定、非敏感错误码；
- 页面按错误码区分路由、开关、来源、作用域和输入问题；
- 网络不可达使用独立页面错误码；
- 失败后立即清除页面内存；
- 不暴露设备令牌、Hash、Blob Key、盐值或异常栈。

## 4. 浏览器回归误报

预期401曾被当成控制台故障，两个相近按钮也曾被模糊选择器混淆。最终方案为：

- 用精确可访问名称定位操作；
- 对预期401单独计数，其他控制台错误仍必须为0；
- 先区分真实bug、测试过时、目标版本错误和环境限制；
- 测试有误只改测试，不改业务语义迎合测试。

## 5. 保护真实失败现场

代理或部署问题只重新部署，不重造两设备与审核数据。销毁严格遵循：关闭能力、强一致检查、摘要删除、二次复查、销毁临时资源。一次性验收PR保持Draft和`DO NOT MERGE`，验收后关闭而不合并。

## 6. 阶段5D长期门禁

- 从新main独立分支推进，不修改8.2.25；
- 设备投影不得返回令牌、tokenHash、Blob Key或可逆身份；
- 封禁后认证失败关闭，并同时撤销可信；
- 解封不自动恢复可信；
- 撤销可信后不得继续单设备自动批准；
- 治理请求幂等，事件与审计不可变；
- 新写接口继承阶段5A的同源HTTPS实现；
- 新开关默认0，回滚、导出和敏感规则不进入5D；
- 若与后续阶段合并真实EdgeOne验收，则不再重复单独验收，但代码、CI、PR和合并授权仍独立。

## 7. 固定排障顺序

1. 路由与404；
2. 部署head、API版本和当前URL；
3. 环境变量是否属于该新部署；
4. 开关和互斥门禁；
5. public origin、代理URL、Origin和Sec-Fetch-Site；
6. 会话或访问输入；
7. Blob作用域与对象结构；
8. 最后才排业务状态机。
