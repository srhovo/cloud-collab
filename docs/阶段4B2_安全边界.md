# 阶段4B.2安全边界

本阶段只在隔离测试作用域验证设备注册和候选接收。

## 三重启用门禁

真实预览写入必须同时满足：

1. `CLOUD_WRITE_PREVIEW_ENABLED=1`；
2. EdgeOne环境变量中存在至少32字节的`CLOUD_WRITE_PREVIEW_KEY`；
3. 请求头携带完全相同的`X-Cloud-Collab-Preview-Key`。

预览密钥使用恒定时间比较。缺失或错误时，在解析正文和初始化Blob之前返回403。真实密钥不得写入GitHub、HTML、截图、日志或聊天记录。

限流盐`CLOUD_RATE_LIMIT_SALT`必须是另一份至少32字节的随机值，不得复用预览密钥。

## 硬编码隔离作用域

代码只允许：

```text
group_fixture
lib_receive_fixture
```

环境变量必须仍填写这两个合成测试值。即使配置成语法合法的正式团或正式价格库，服务也会返回`PREVIEW_SCOPE_MISCONFIGURED`并拒绝启动。

## 其他边界

- 写入功能默认关闭。
- 用户页面仍不发送数据。
- Blob命名空间必须与未来正式公共数据分离。
- 设备注册响应明确保持`submissionEnabled:false`。
- 候选仅保存为`waiting_confirmation`。
- `publicMutationAllowed:false`。
- `autoApprovalEnabled:false`。
- 正式公共快照与公共版本不修改。
- 自动批准与管理员操作保持关闭。
- 验收结束后关闭预览写入开关并重新部署。
