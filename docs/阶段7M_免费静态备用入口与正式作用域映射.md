# 阶段7M：免费静态备用入口与正式作用域映射

## 负责人决策

- 当前没有可控制的自定义域名；
- 先完成所有不依赖域名、真实Blob和真实密钥的工作；
- 用户可见首个club ID为`see`；
- 用户可见首个library ID为`see_cz`；
- 正式稳定目标仍为`8.3.0`，尚未授权晋升。

## 作用域兼容方案

用户界面与协议层分离：

```text
用户可见club ID：see
用户可见library ID：see_cz
协议groupId：group_see
协议libraryId：lib_see_cz
```

原因：现有设备提交、业务Hash、幂等键、审核链、Blob目录和迁移工具均已经围绕`group_* / lib_*`协议完成自动化及真实EdgeOne验收。通过确定性映射保留这些协议，不需要重新设计或迁移已验证的数据身份。

映射规则：

- 普通ID自动添加协议前缀；
- 既有`group_* / lib_*`不会重复添加前缀；
- 映射可以无损投影回用户可见ID；
- 中文、空格、连字符和过短ID失败关闭；
- 正式初始化只允许协议作用域`group_see / lib_see_cz`。

## EdgeOne平台域名边界

EdgeOne项目域名在项目存在期间固定，但当前“全球可用区（含中国大陆）”的项目域名需要临时预览访问令牌。令牌有效期约3小时，因此可用于技术验收，但不能提供无需令牌的永久公开入口。

阶段7M不要求购买域名，也不阻断代码、CI、静态备用入口和生产初始化预演。永久匿名主入口继续记录为待自定义域名。

## GitHub Pages免费静态备用

新增`.github/workflows/pages-static-backup.yml`：

- `main`每次更新后自动执行完整CI；
- 重新构建并核对冻结8.2.31候选；
- 只发布`index.html`、`build-manifest.json`、`pages-release.json`；
- 明确校验稳定晋升为false；
- 明确校验正式公共写入为false；
- 部署后核对提交、标题、版本、SHA-256、字节数和管理员页面未公开。

GitHub Pages只提供本地模式和离线静态备用，不承载：

- EdgeOne Cloud Functions；
- Blob数据库；
- 设备注册；
- 公共数据写入；
- 自动审核；
- 管理员会话或审核控制面。
