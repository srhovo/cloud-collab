# 阶段7G：候选8.2.30打包与最终发布清单

## 一、项目负责人授权

2026-07-21，项目负责人明确授权：

- 候选版本采用`8.2.30`；
- 接受清理精确删除数量及两次独立零对象数字缺失的单独证据豁免；
- 继续生成候选单文件和最终发布清单；
- 暂不晋升稳定版。

该授权只允许候选打包，不允许修改冻结稳定基线`码单器8.2.25_现.html`，不允许开启正式公共写入，也不允许自动执行稳定版晋升。

## 二、候选构建契约

阶段7G继续复用经过阶段6B与阶段7A至7F验证的构建链路，但将最终候选外壳统一为：

```text
APP_VERSION：8.2.30
页面标题：码单器8.2.30（公共协作发布候选版）
构建输出：dist/index.html
候选文件：release-output/码单器8.2.30_候选.html
```

候选页面标题、`APP_VERSION`、构建清单版本、项目负责人版本决策和最终发布清单必须全部一致。

## 三、清理证据豁免边界

账本继续如实保留：

```text
exactDeletionCountsRecorded = false
independentZeroCountEvidenceRecorded = false
```

同时记录：

```text
exactEvidenceWaiverAcceptedByOwner = true
```

这表示项目负责人接受无法恢复的精确数字缺口，不表示这些数字已经补录，也不把用户报告伪装成平台导出的机器证据。

## 四、生成物

运行：

```bash
npm run ci
npm run release:package
```

会生成三个文件：

1. `release-output/码单器8.2.30_候选.html`
2. `release-output/最终发布清单_8.2.30.json`
3. `release-output/发布审计_8.2.30.json`

最终发布清单记录：

- 稳定基线8.2.25的冻结元数据；
- 候选8.2.30的标题、字节数和SHA-256；
- 候选版本授权及两项证据豁免；
- 默认关闭环境核查；
- 稳定晋升未授权、未执行；
- 正式公共写入未授权。

同一构建输入重复打包时，候选文件和最终发布清单必须保持一致。

## 五、自动化门禁

新增独立GitHub Actions工作流`stage7g-release-candidate`：

- 使用Node.js 24；
- 运行完整`npm run ci`；
- 运行`npm run release:package`；
- 检查三个生成物存在；
- 输出候选SHA-256；
- 上传`mdq-8.2.30-candidate-package`构建产物。

现有完整浏览器回归工作流继续独立运行。本阶段只有两个工作流均通过后才能合并。

## 六、失败关闭条件

以下任一情况必须失败：

- 候选版本不是8.2.30；
- 页面标题、`APP_VERSION`、构建清单或负责人决策不一致；
- 候选摘要或字节数与构建清单不一致；
- 清理数字既未补证，也未获得明确豁免；
- 任一预览或写入能力默认开启；
- 示例环境包含私密值或临时部署来源；
- 稳定晋升授权被设置为`true`；
- 稳定晋升或正式公共写入被执行。
