# 阶段7W：正式管理员设备治理

## 目标

在阶段7V完成正式普通与敏感审核链后，把阶段5D已经验证的设备治理核心接到正式管理员会话、生产公共Blob和固定正式作用域，不修改设备令牌格式或治理事件语义。

## 正式API

```text
GET  /api/admin/devices
GET  /api/admin/devices/detail?id=<deviceRef>
POST /api/admin/devices/trust
POST /api/admin/devices/revoke-trust
POST /api/admin/devices/block
POST /api/admin/devices/unblock
```

URL保持不变。六个Cloud Function入口通过`admin_device_governance_mode_dispatch_v1.js`选择正式或阶段5D隔离预览处理器，并保留原处理器名称作为兼容锚点。

## 模式分发

```text
CLOUD_PRODUCTION_ENABLED=0或未配置 → 阶段5D隔离预览处理器
CLOUD_PRODUCTION_ENABLED=1          → 正式设备治理处理器
其他值                              → 503失败关闭
```

生产总开关为1后，不论管理员或审核子开关是否关闭，都不会回退预览；正式处理器会在创建Store前返回503。

## 正式配置

正式设备治理要求：

```text
生产总开关：开启
只读同步：开启
管理员身份：开启
管理员人工审核：开启
公共Blob：cloud-collab-production-v1
管理员会话issuer：cloud-collab-admin-production
设备脱敏盐值：CLOUD_ADMIN_DEVICE_REF_SALT
用户可见作用域：see / see_cz
协议作用域：group_see / lib_see_cz
```

设备档案、治理事件、治理状态投影和设备令牌都位于公共生产Blob。管理员登录限流与会话数据仍位于独立管理员Blob。正式响应只暴露不可逆`deviceRef`，不暴露`deviceId`、令牌Hash、请求Hash、内部Blob Key或真实密钥。

## 治理动作

### 设为可信

- 仅未封禁、当前非可信设备可设为可信；
- 可信设备可参与普通自动审核的单设备可信批准路径；
- 敏感提交仍不能绕过人工审核。

### 撤销可信

- 立即取消可信资格；
- 后续普通自动审核不再把该设备视为可信；
- 不撤销现有设备令牌。

### 封禁

- 封禁同时撤销可信；
- `authenticateDevice`立即返回`DEVICE_BLOCKED`；
- 不删除设备档案、令牌索引、历史候选、审核或审计；
- 治理事件、请求索引和审计保持不可变。

### 解除封禁

- 恢复设备令牌认证资格；
- 不自动恢复可信状态；
- 如需重新可信，必须执行新的显式`trust`动作。

## 幂等与并发

- 每个动作要求`requestId`、`deviceRef`和固定原因码；
- 相同请求精确重放返回原治理结果；
- 同一治理基线被不同请求占用时冲突失败；
- 治理事件和审计使用不可变写入；
- 可变head仅作为强一致状态投影，不删除历史事件。

## HTTP安全边界

- 正式接口只接受`cloud-collab-admin-production` issuer会话；
- POST必须精确匹配`CLOUD_ADMIN_PUBLIC_ORIGIN`；
- 跨站请求在正文读取和公共Blob创建前阻断；
- 会话无效或过期返回401并清理Cookie；
- 响应使用`no-store`、同源资源策略和`nosniff`；
- 不设置通配CORS。
