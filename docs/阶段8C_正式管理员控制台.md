# 阶段8C：正式管理员控制台

## 目标

阶段7S至8B已经完成正式管理员身份、普通审核、敏感审核、设备治理、公共回滚和完整数据库导出API。本阶段提供一个同源、独立、无浏览器持久化存储的正式管理员控制台源实现，使这些能力可以通过统一页面操作和验证。

本阶段只提交代码、自动化测试和文档，不部署管理员页面，不配置真实管理员域名，不启用任何生产开关。

## 文件

```text
admin/production-console.html
admin/production-console.css
admin/production-console.js
```

控制台采用三文件同源结构：

- HTML不含内联脚本和内联样式；
- CSP只允许`connect-src 'self'`、`script-src 'self'`和`style-src 'self'`；
- 不加载图片、字体、媒体、框架、Worker或第三方资源；
- 不使用表单提交，所有写入都通过同源`fetch`调用已有正式API；
- 动态内容只通过`textContent`和DOM节点创建，不使用`innerHTML`。

## 功能范围

### 管理员身份

```text
POST /api/admin/auth/login
GET  /api/admin/auth/session
POST /api/admin/auth/logout
```

密码在构造登录请求后立即从输入框和临时变量清理。页面不读取Cookie；会话只由服务端HttpOnly Cookie维护。

### 人工审核

```text
精确价格：/api/admin/reviews
普通资料：/api/admin/ordinary-reviews
敏感变更：/api/admin/sensitive-reviews
```

控制台使用服务器冻结确认词：

```text
APPROVE
REJECT
EDIT_AND_APPROVE
APPROVE_ORDINARY
REJECT_ORDINARY
EDIT_AND_APPROVE_ORDINARY
APPROVE_SENSITIVE
REJECT_SENSITIVE
EDIT_AND_APPROVE_SENSITIVE
```

敏感队列显示`sensitiveSubmissionIntakeEnabled`。关闭新敏感候选入口时，页面明确提示仍可处理存量队列。

### 设备治理

```text
GET  /api/admin/devices
GET  /api/admin/devices/detail?id=...
POST /api/admin/devices/trust
POST /api/admin/devices/revoke-trust
POST /api/admin/devices/block
POST /api/admin/devices/unblock
```

页面只显示不可逆`deviceRef`，不接收或显示真实`deviceId`和设备令牌。

### 公共数据回滚

```text
GET  /api/admin/rollbacks
POST /api/admin/rollbacks/execute
```

使用确认词`ROLLBACK_TO_PREVIOUS_APPROVED_VALUE`。控制台说明回滚会追加补偿事件，不删除或改写历史。

### 完整迁移导出

```text
GET  /api/admin/exports/summary
POST /api/admin/exports/download
```

控制台严格使用阶段8B契约：

- 摘要读取`data.summary`；
- ZIP大小读取`summary.byteLength`；
- 下载读取`X-Cloud-Collab-Package-Id`和`X-Cloud-Collab-Public-Version`；
- 必须验证`X-Cloud-Collab-Stable-Promotion-Authorized: 0`；
- ZIP生成与下载不会修改公共数据，只写入私有不可变导出审计。

## 浏览器隐私边界

```text
localStorage：不使用
sessionStorage：不使用
IndexedDB：不使用
document.cookie：不读取
管理员密码：请求后清空输入框
页面退出或pagehide：清空队列、详情、导出摘要和Object URL
```

所有API请求统一使用：

```text
credentials=same-origin
cache=no-store
redirect=error
referrerPolicy=no-referrer
```

每个正式业务响应必须继续声明：

```text
productionAdmin=true
syntheticFixtureOnly=false
stablePromotionAuthorized=false
```

否则控制台失败关闭。

## 普通用户候选隔离

普通用户候选发布白名单仍然只有：

```text
index.html
build-manifest.json
pages-release.json
```

`admin/`目录不进入`.edgeone-artifact`，也不进入GitHub Pages普通用户候选入口。本阶段不修改`prepare-public-candidate-v1.mjs`的白名单。

管理员页面未来只能部署到负责人控制的独立管理员来源，并必须由平台响应头补充实际有效的CSP、HSTS、`frame-ancestors 'none'`和其他同源安全头；本阶段不生成该部署。
