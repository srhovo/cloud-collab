# 阶段7V：正式敏感审核与墓碑发布

## 目标

阶段7S已经把区间、加价、礼物、老板敏感变化和显式删除作为不可变候选写入生产公共Blob；阶段7T与7U已经建立并加固正式管理员身份和普通审核链。

阶段7V把阶段6B已经验证的敏感审核核心接到正式管理员身份与生产作用域，提供队列、详情、批准、拒绝、编辑后批准以及显式删除墓碑发布。

本阶段不启用EdgeOne生产开关、不填写真实密钥、不部署新版本、不修改8.2.31候选或8.2.25稳定版。

## 路由

```text
GET  /api/admin/sensitive-reviews
GET  /api/admin/sensitive-reviews/detail?id=<review-id>
POST /api/admin/sensitive-reviews/approve
POST /api/admin/sensitive-reviews/reject
POST /api/admin/sensitive-reviews/edit-and-approve
```

五个Cloud Function统一调用`admin_sensitive_review_mode_dispatch_v1.js`。

模式分发复用阶段7T加固后的生产总开关：

```text
CLOUD_PRODUCTION_ENABLED=0或未配置 → 阶段6B隔离预览处理器
CLOUD_PRODUCTION_ENABLED=1          → 正式处理器
其他值                              → 503 PRODUCTION_FLAG_INVALID
```

生产模式不会因管理员、人工审核或敏感提交子开关关闭而回退预览。

## 正式门禁

正式处理器复用`readProductionRuntimeConfig`，要求：

```text
CLOUD_PRODUCTION_ENABLED=1
CLOUD_PRODUCTION_READ_SYNC_ENABLED=1
CLOUD_ADMIN_PRODUCTION_ENABLED=1
CLOUD_PRODUCTION_ADMIN_REVIEW_ENABLED=1
CLOUD_PRODUCTION_SENSITIVE_SUBMISSION_ENABLED=1
```

同时固定：

```text
公共Store：cloud-collab-production-v1
协议groupId：group_see
协议libraryId：lib_see_cz
管理员来源：CLOUD_ADMIN_PUBLIC_ORIGIN
```

任一门禁关闭时在创建Blob Store前返回503。

## 身份与同源

正式敏感审核只接受issuer为：

```text
cloud-collab-admin-production
```

的阶段7T会话。GET要求同站请求；所有POST还要求精确`Origin`。跨站请求在解析正文和创建Store前阻断。

## 审核动作

复用阶段6B冻结确认词：

```text
批准：APPROVE_SENSITIVE
拒绝：REJECT_SENSITIVE
编辑后批准：EDIT_AND_APPROVE_SENSITIVE
```

敏感候选不能自动批准，可信设备和双设备一致也不能绕过人工审核。

批准或编辑后批准：

- 校验候选捕获的公共基线仍然有效；
- 追加不可变敏感公共事件；
- 更新统一公共快照；
- 显式删除发布不可变墓碑；
- 写入不可变决定、审计、归档和完成标记；
- 精确重放返回原决定，不产生第二个公共版本。

拒绝：

- 写入不可变决定、审计、归档和完成标记；
- 不修改公共数据。

编辑后批准不得改变业务身份，删除候选不得编辑后批准。

## 响应边界

正式响应使用脱敏投影，并声明：

```text
productionAdmin=true
syntheticFixtureOnly=false
manualReviewRequired=true
automaticApproval=false
trustedDeviceBypass=false
twoDeviceBypass=false
tombstonePublish=true
publicMutationAllowed=true
stablePromotionAuthorized=false
realSecretValuesExposed=false
```

响应不返回设备令牌、管理员令牌、授权头、原始聊天、订单、备注或真实秘密。

## 自动化验证

专项工作流运行：

- 阶段7V生产HTTP与模式分发测试；
- 阶段6B敏感审核核心、公共事件、墓碑、不可变审计与幂等回归；
- 阶段7S正式敏感候选入队回归；
- 阶段7T正式管理员身份和普通审核身份链回归；
- 生产运行时门禁和Cloud Function导入回归；
- 完整Node与浏览器矩阵。
